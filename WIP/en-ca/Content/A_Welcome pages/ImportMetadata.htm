Importing GCPM metadata to online help

Steps to importing GCPM metadata into the online help:

  1. Create a metadata Flare project (e.g. GCPMMetadata).
  2. Create export files for each feature set (e.g. “Performance” includes all entities included in this feature.
    1. Use XRM toolbox to generate the file. Select entities to export, and use these settings: Filter custom attributes by prefix egcs; Include attribute location in forms, gather entities list in a summary
    2. Enter file path

      e.g. C:\Users\mortimek\Documents\My Projects\GCPMMetadata\MetadataFiles\Stakeholder.xlsx

    3. Save these settings (This PC > Documents) as a *.msettings file

      e.g. Stakeholder.msettings

    4. Generate the metadata file. Open the *.xls file and remove these columns:

      Schema name

      Plural names

      Custom attribute

      Empty columns (with formatted headers)

    5. Note select the entities that you want to group together, these are displayed in the “Entities” list which is the first page of the worksheet created.
  3. Import the metadata into the GCPM Metadata project. Note the options:
    • Choose to import worksheets as Snippets.
    • The first time the import is done, create a topic with snippets. Once this is completed, deselect this option.
    • In flare, modify the topic with snippets and remove link to import file
    • Styles: Choose Don’t preserve…, leave and Leave blank for none
  4. Review the metadata files, move all generated topics into the Metadata files folder. Update the "Date of import".
  5. Clean up unused files, move things into folders appropriately.

  6. Update the MetadataTOC file to include all imported files.

  7. In the GCPMDoc flare project, import the GCPM metadata.
  8. Apply the metadata table style to all imported files.
  9. Verify the import has worked by generating the new project and using the link under Developer Corner / GCPM Metadata.
