Release notes for GCPM documentation

Finalized: June 28, 2019

This document describes the tools and files provide the GCPM online help (English and French) located at https://rda-help-app.egcs-sesc.ca/wip/en-ca/Content and facilitate translation of the GCPM interface (e.g. form names, entity names, field names and descriptions).

Key tools used to produce online help

Madcap Flare

  • Single source authoring Write content once, publish multi-channel (HTML, Word, PDF, etc.)
  • Publishing templates – use CSS to quickly change look and feel, with print, web and multi-device options
  • Integrate video, images, readily update
  • Chatbots, micro content
  • Multi-language support
  • Review cycles with experts – send files for review and easily integrate comments
  • MadCap Flare is Section 508 and WCAG (Web Content Accessibility Guidelines) compliant.

Madcap Lingo

  • Generate a French help site – to enable users to switch from one topic to another online (Demo)
  • Advanced translation management for technical authors, documentation teams and translators.
  • Package files for translation with tool that automates some aspects of translation management.
  • Merge translated files received from translator into help

Tools used to do onscreen help and translation

  • GCPM team members provided onscreen translations. These translations (e.g. view names, field names, etc.) are stored in an Excel file.
  • A short video provides step-by-step instructions.
  • Online help translation was mostly completed by contract. The translators used translation tools and translation exchange file format to readily exchange translated content. Translations that already existed, (e.g. in XLS file or the translation memory file), were done by the GCPM team.

Training content

  • Not a focus of GCPM project, done on ad hoc basis.
  • Tools: Snagit and Camtasia to generate screenshots and video.
  • Approach:
  • Mixed recorded demos with narration (using script, so it could be translated) and Powerpoint slides
  • Videos stored in online help and integrated to relevant topics.
  • Future options: WalkMe, ClickLearn. Others?

Change management

Factors that will impact change management:

  • Skills on the team, level of flexibility (in both the implementation team and the users experiencing change), use of new processes (e.g. agile) have huge impact. Be honest in assessing what is possible for your group. We use many out-dated approaches and technologies; this might be the opportunity to leap forward.
  • Tools available – using modern tools and approaches make a big difference! But they require training so people understand how to use them
