Create a custom SSRS report

Disclaimer: 

This content was written early in the development of GCPM while testing out the capabilities of SSRS for reporting purposes. This content will remain here for as it is useful for getting up to speed on how to implement a custom SSRS report in the Dynamics CRM system; however, it has not been tested against Dynamics CRM v9.0 or Dynamics CRM Online, and has not been vetted since late 2017.

This help topic describes how to create custom reports using Visual Studio to overcome the limitations of Dynamics 365 reports. Reports using the report wizard in Dynamics 365 can be limited (Tablix or Matrix only). 

An SSRS (SQL Server Reporting Services) report provides more flexibility.

Prerequisites: 
  • Install Visual Studio 2017
  • Know how to write SQL queries
  • Install SSRS templates (insert link) 

To create a report using VisualStudio:

  1. Create a blank report in CRM using the report wizard. The report wizard ensures the report includes all necessary back-end parameters used by CRM at run time.
  2. Download a copy of the newly created report template locally.
  3. Open VS 2012 and create a new BI Report.
    Notes: 
    1. You can use CRM Advanced Find or a tool such as XRMToolBox (Fetch XML Builder plugin) to generate a FetchXML query expression.
    2. At the time of writing, there was an issue when using VS2015 or newer version. It appears that VS2012 had native SSRS support enabled and works, while newer versions can have problems. This may not always be the case going forward.
    3. If you don't start with a template document generated by CRM, you cannot use a Shared data set. You must instead embed the data set within the report. Otherwise you will get error when trying to upload the *.RDL file back into CRM.

  4. Choose a data source for a report

  5. Replicate the screen when defining your data source.

    Notes:

    • Connection String: the first part is the URL of your Org. CRM automatically replaces the first part of the URL past https:// with “dev”. If you did not use a templated report, then you will need to adjust the URL.
    • The part after the semi-colon is the Unique Org Name.
    • If you don’t know what that is, you can find it by going to the following in Dynamics 365: Settings >Customizations >Developer Resources.

    Changing connection options for the data source.

  6. Now create a custom report as required.

  7. When completed, build the project and browse to the Bin\Debug folder and copy the *.rdl file located there.

  8. Browse to the CRM report you created in step 3. Select the report and click Edit. A dialog box opens. Choose a report type.
  9. Select Existing File for the Report Type. Browse to the *.rdl file you created in the previous step and upload it.
  10. Chose the appropriate location you want the report to be displayed in and Save.
  11. Click OK to the Update Prompt.
  12. Run the report to verify its accuracy.
