Plugin development guide

To standardize plugin development across the solution, GCPM has a base plugin structure that all plugins must follow to ensure adherence to GCPM requirements. Please follow this guide when developing new plugins for GCPM.

Notes: 

  • CRM caches plugins
  • CRM does not initialize plugins every time
  • Executes plugins on multiple threads using thread agility.

  • Maximum Call Depth - Optional

    • Sets the maximum call depth allowed by the plugin. Call depth refers to where in the stack the plugin is called. For example, the first plugin that triggers will have a call depth of 1. If the first plugin calls another plugin, the second plugin will have a call depth of 2.

    A plugin can call another plugin multiple times, and each plugin call has the same call depth. For example, if the first plugin creates 5 new records, and each record created triggers a plugin, those 5 plugin calls will all have a call depth of 2.
    By default, the max call depth set by CRM is 8, which can be changed.

Best practices: 

  • You may wish to only trigger your plugin up to a certain call depth, e.g. to stop recursive plugins up to a certain point.
  • Set max depth to 1 to ensure a plugin is only triggered directly by a record and not by another plugin.

Plugin Extensions - Optional

You can register plugin extensions to be run separately or before or after your plugin execution. Any object that implements the IPluginExtension interface is a valid plugin extension.

Currently, the only plugin extension is the Plugin Step Registration extension.

Bypass CRM's Plugin Caching Mechanism

Since CRM caches the plugin object, instead of creating a new plugin object every time, simply move all your plugin logic to a different class and create a new object of that class during execution. This way, you can have all the global class variables you want!

Plugin Extensions - Plugin Step Registrator

The purpose of this extension is to save the user the hassle from registering plugin steps manually. When configured, this extension will create, update, and delete plugin steps as configuration records are created, updated, and deleted.
