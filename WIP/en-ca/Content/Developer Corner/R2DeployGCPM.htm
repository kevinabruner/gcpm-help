Key steps to deploy GCPM 2.1

GCPM deployment has been tested in this environment (the GCPM development environment):

  • Microsoft SQL Server 2016 (SP2-GDR) (KB4293802) - 13.0.5081.1 (X64) Jul 20 2018 22:12:40 Copyright (c) Microsoft Corporation Developer Edition (64-bit)
  • Microsoft Dynamics 365 Version 1612 (9.0.2.3041) (DB 9.0.2.3041) on-premises
  • Microsoft Dynamics 365 French Language Pack

Note on executing scripts

You must update scripts to connect to different environments (On-prem or On-line). For example: 

  • Example of function to start deployment – on prem
    Import-CrmMultiplePackagesAsync -OrgUrl '<org name>' -ServerUrl ‘<server url>’ -IsCrmOnline $false -path '<path to release folder location>' -CsvImportList '\SCRIPTS\package_list_v2.csv'
  • Example of function to start deployment – on line

    Import-CrmMultiplePackagesAsync -OrgUrl '<org name>' -ServerUrl '<server url>’ -IsCrmOnline $true -path '<path to release folder location>’ -CsvImportList '\SCRIPTS\package_list_v2.csv'

Folder structure

/GCPM

/FINAL_PACKAGE

Set of folders / ddls that are readable by PackageDeployer tool from SDK.

As an example:

/SCRIPTS
  • Org_Settings_ONLINE_ONPREM.ps1 - PowerShell file that set up org settings.
  • Load_Packages_ONLINE_ONPREM.ps1 - PowerShell file that will execute set of PackageDeployer files from FINAL_PACKAGE folder
  • package_list_v2.csv - Order of execution of different PackageDeployer projects
/UTILITY
  • Enable_BusinessRules_ONPREM_ONLINE.ps1 and businessrulelist.csv PowerShell script and input file with list of Business Rules that need to be disabled after init installation of GCPM System
  • CleanUP-Views-ONPREM_ONLINE.ps1 and viewlist.csvPowerShell script and input file with list of OOB views that need to be disabled after init installation of GCPM System
  • Enable_Connections_ONPREM_ONLINE.ps1 and connectionrolelist.csv PowerShell script and input file with list of OOB Connections that need to be disabled after init installation of GCPM System
  • Import-CRMUsers_ONPREM_ONLINE.ps1 and users.csv PowerShell script and example of input file to add users.
  • TeamToRole-ONPREM_ONLINE.ps1 and teamroles.csv PowerShell script and example of input file with N:N relationships between teams and roles that need to be enabled after init installation of GCPM System
  • EntityMaps-ONPREM_ONLINE.ps1 and mapping.csv PowerShell script and example of input file with mapping information for relationships between different entities that need to be imported after init installation of GCPM System
/LOG

Folder to hold log files generated by PackageDeployer tool

Key steps to deploy GCPM Release 2.0

  1. Create an organization.
    Note: Use Canadian Dollar as default currency.
  2. Configure organization settings by running this script:

    Org_Setting_ONLINE_ONPREM.ps1
    OR

  3. Manually update organization settings using Dynamics 365 interface. Navigate to
    Settings > Administration > Languages or Settings > Administration > System Settings 
    • Enable French Language Pack
    • Enable Language Pack
    • Enable Global help and Set URL to Help file
    • Change Format to English Canadian
    • Disable Tracing
    • Disable AutoSave
    • Disable Presents
    • Disable Bing Map integration
  4. Deploy Solutions / Data by executing the Load_Packages_ONLINE_ONPREM.ps1 file.
    Note:  Deploying the GCPM solution can take some time.
  5. Configure CroweGrids. Navigate to Settings > EditAble CRM Grid Settings and click Apply New Licence.
    NOTE: A CroweGrid license is not provided. You must procure a license for your organization.
  6. Export CroweGrids: navigate to Settings >EditAble CRM Grids and click EXPORT EDITABE GRIDS.

    NOTE: CroweGrids configuration is stored in following XML webresources.


    Note: After importing all Crowe grids, you should have 100 records.

    Note: Some CroweGrids cannot be published (see screenshot below). Please delete these records: 

    See Crowe Grid Help for further information.

  7. Ensure that all processes are activated. Use Advanced Find to generate a list of inactive processes:
    Note: Workflows related to Snapshot functionality need to be re-opened manually and re-activated.
  8. Execute the following PowerShell scripts:

    • Enable_BusinessRules_ONPREM_ONLINE.ps1
    • CleanUP-Views-ONPREM_ONLINE.ps1
    • Enable_Connections_ONPREM_ONLINE.ps1
    • TeamToRole-ONPREM_ONLINE.ps1
  9. Add users. Do this manually, or execute Import-CRMUsers_ONPREM_ONLINE.ps1
  10. Add users to teams - this must be done manually.
  11. Configure the document manager.
    See Key steps to configuring the document manager
    • (If using Sharepoint) Re-configure document management for SharePoint: update links in CFG Document Location and Document Location entities to point to your SharePoint site and document libraries.
    • Note: GCPM has been configured with a sample SharePoint configuration, to provide the deployment team with an example. These sample configuration records are located in (CFG Document Location, CFG Document Entities and Document Location). To disable this sample configuration:
      • Disable all records in the CFG Document entity. Use this query in Advanced Find to locate all workflows that generate snapshots:
      • Disable all workflows that generate snapshots.
  12. Set up financial integration.
    If you choose to configure financial integration at a later time, disable all workflows in the GCPM_FI_AUTOMATION solution.
