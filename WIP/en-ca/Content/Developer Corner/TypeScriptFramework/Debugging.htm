Debugging

Setting Fiddler AutoResponder

AutoResponder in Telerik Fiddler can be used to replace the content of a web resource with content from a local file rather than uploading it in CRM each time code is changed. This way a TypeScript file can be debugged using browser debugging tools.

See this article Streamline JavaScript web resource development using Fiddler AutoResponder for information on activating AutoResponder.

Another way to have this functionality is to modify Fiddler Script. These are the steps to accomplish it:

  1. Select FiddlerScript tab
  2. Add snippet to OnBeforeRequest method. Specify your local path.

  3. Snippet

    var index = oSession.url.indexOf("/WebResources/egcs_/");

            if (index > 0) {

                var sub = oSession.url.substr(index + "/WebResources/egcs_/".length);

                oSession["x-replywithfile"] = "C:\\GCPM\\EGCS\\EGCS.CRM\\EGCS.CRM.Framework\\EGCS.CRM.WebResources\\egcs_\\" + sub;

            }
  4. Save Fiddler Script.
