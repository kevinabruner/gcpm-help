TypeScript

Information on TypeScript can be found on its official site: https://www.typescriptlang.org/

Some of the main syntax differences with JavaScript are listed here.

TypeScript Syntax

Static Types

One of the most important things when working with TypeScript is always declare a variable’s type.

Type

Note
Declaring types

var n: number = 5;

function add (a: number, b: number): number { return a + b; }
Multiple types

function(shape: Square | Circle): void { }
Dynamic type

Declaring the type as “any” makes it accept any value.
Type casting

Syntax: <string>variableName or variableName as string

Interfaces

There is a TypeScript specific syntax that is similar to interfaces in Java/C#.

Interfaces are used to create strongly-typed and statically-typed objects.

Interfaces in TypeScript are only used to provide strong type checking at compile time and have no inherit meaning in JavaScript itself.

Namespace vs. Module

This is a TypeScript specific distinction.

Namespaces live in the global namespace as global variables.

Modules are self-contained JS files that are loaded into another module and lives in the local namespace.

Enumerations

Unlike other languages, enums can be numeric or strings.

There are two types of enums:

  • Regular enums
  • Are transpiled to regular object literals in the JS file.
  • Const enums
  • Are not transpiled in the JS file; think of it as string const in C#, which replaces all references to the string const with the actual string when compiled.

TypeScript Package Installation

In order to work with TypeScript in Visual Studio, @types/xrm and @types/jquery packages must be installed. To install them, follow these steps:

  1. Install Package Installer in Visual Studio
  2. Install @types/xrm and @types/jquery Packages
  3. Right click the project and choose Quick Install Package
  4. Install packages.
  5. Include node_modules folder in project.

    If node_modules folder is not visible, use “Show All Files”. Right click node_modules folder and include it in the project.
