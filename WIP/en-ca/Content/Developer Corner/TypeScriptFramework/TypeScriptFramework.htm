Framework (for TypeScript)

main.js

Analogous to the “static main” method in other languages, this file is the main entry point for all JS scripts.

There are three places in CRM where JavaScript can be used:

  • Form
  • Ribbon Bar
  • Form, Grid, Subgrid, Associated View, Dashboard, etc.
  • HTML Web Resources

Thus, there are three “main” entry points, listed below. These entry points can be seen as the “base” class that set up everything for the script.

Main Entry Point for Form Scripts – EGCS.Main.Form.load()

Setup:

  • Script: egcs_/shared/js/main.js
  • Form Event: On Load
  • Form Function: EGCS.CRM.Form.load
  • Parameters:
  • Path to script file
  • Other required parameters

A form module should only export a default function, which will receive the following input values:

  • FormContext object
  • Events object
  • Any values specified in the form editor as separate input parameters
Main Entry Point for Ribbon Bar Commands – EGCS.Main.Command.load()

Setup:

  • Create a DisplayRule or EnableRule to load the command script.
  • Web Resource: egcs_/shared/js/main.js
  • Function: EGCS.Main.Command.load
  • First input parameter: “path to command script”
  • Optionally include function to call as query string
  • Pass in any other parameter values as desired.
  • This will load all the exports into the EGCS.Main.Command namespace
  • Be aware of duplicates
  • Create a ribbon Action to call corresponding function in the command script.
  • Web Resource: egcs_/shared/js/main.js
  • Function: EGCS.Main.Command.<function_name>
  • Must pass PrimaryControl as the first input parameter.
  • Pass in any other input parameter values as desired.
Main Entry Point for HTML Web Resources – EGCS.Main.Page.load()

In the html, load, in order:

  • <script src=“ClientGlobalContext.js.aspx” />
  • <script src=“egcs_/shared/js/main.js” onload=“EGCS.Main.Page.load(‘path to module’)” />

A page module should only export a default function, which will receive the following input values:

  • PageContext object

All script dependencies should be loaded within the script file using import/export.

context.js

This is the context object that gets passed to event handlers, such as onload, onsave, onchange, etc.

Just like the server-side framework, there are different types of context objects:

  • Context
  • This is the base context class, which is an abstract class.
  • FormContext
  • This is the context object for form scripts.
  • Contains wrapper objects for some XRM objects.
  • CommandContext
  • This is the context object for ribbon bar commands.
  • Also contains the form context if it is the form ribbon bar
  • PageContext
  • This is the context object for html web resources.
  • Also contains the form context if it is loaded from a form

events.js

The events object is only used in entity form scripts.

Similar to the PluginExecution class in the server-side framework.

Contains all the possible form events to which you can attach an event handler.

Contains custom events that do not exist in CRM, e.g.: oncreate, onupdate, etc.

core.js

A core set of classes used by the client-side framework.

Framework will not work without these classes.

common.js

Includes all the common/shared code. It is used by most JavaScript web resource.

Eventually, we will split it up into appropriate modules. However, the modules will still be bundled together into one single .js file with the name “common.js” or “shared.js” or something like that.
