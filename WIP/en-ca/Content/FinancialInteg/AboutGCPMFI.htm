About financial system integration with GCPM

Financial Integration (FI) is a series of components that can be used to integrate financial system operations into your grants and contributions processes. These components have a flexible design, which can be used in a fully automated, fully connected scenario as well as a fully manual, fully disconnected scenario, or a combination of these.

This ensures that departments with varying degrees of automation and different grants and contributions systems can configure financial system integration to meet their requirements.

For departments and agencies that plan a fully connected scenario: 

  • The FI components can be seen as a reference implementation.
  • The list of entities will not change, but you may wish to incorporate new fields. Incorporating new fields will require changes to the code.
  • The REST API is currently submitting to a mock financial system. You need to point the API to your financial system, either directly, or by making calls to your financial system's API (using web services or otherwise).

  • You need to implement each endpoint in the API that is responsible to read and write to your financial system for seamless integration with GCPM. GCPM relies on these endpoints in the form of workflow steps.

  • It is imperative to ensure that the response from your financial system call is captured in the transaction logic in the API to ensure that administrators have access to a log of what is returned from your financial system in GCPM.

See FI prerequisites

GCPM Integration Points

While working through the FI components you must identify the points within GCPM from which you will make calls to the FI components. Whether your organization chooses a fully manual approach or fully automated approach, there are options for automation within GCPM that do not involve the financial system. These options will make the lives of your financial officers easier.

Note: Coding blocks are not populated with values. This could be done using a workflow.

For example, there are workflows to automate the generation of FI records. Once these records are created, you can decide if the data will be submitted automatically or manually.

Connect the calls out to REST service to your actual financial system. Implement your own version following of REST service following this endpoint structure: EGCS.CRM.Services.GCPMFinancialSystemAPI

Transitions

In the sample transition templates in GCPM, there are many examples of calling the financial system during the project life cycle. The main tool is the CRM workflow. All calls to the financial system, or to generate FI records, are made with Custom Workflow Steps. These workflow steps reduce complexity to provide a simple way to trigger aspects of financial integration.

Reports (word templates)

Word templates have been created for the vendor customer, payment, and commitment entity to allow GCPM officers to print or send a request to finance if their instance of GCPM is not leveraging the financial system API.

In certain scenarios, a paper version of the request may still be required even if GCPM is integrated with your financial system. As a result, these basic reports have been created to support this use case. The reports use the out of the box word template feature provided by Dynamics 365.

Financial Integration help topics

The financial integration help topics are designed to provide GCPM administrators and developers with the information required to configure, implement, and expand the financial system integration components within the GCPM platform.
