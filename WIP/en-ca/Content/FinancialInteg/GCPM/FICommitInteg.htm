Commitment integration

FI Commitment stores information in GCPM that is related to a commitment in the financial system. It has a child entity FI Commitment Detail that stores one or more detail lines under the commitment. GCPM has several points at which it automates the creation and submission of a commitment to the financial system.

Prerequisites
  • Financial Integration solutions
  • Workflows
  • Custom workflow steps
  • Entity models

Automating generating a soft commitment during the project lifecycle

  • The workflow Generate Soft Commitment creates an FI Commitment and sends it to finance.
    See Workflows to manage commitments.
  • This workflow can be called from any action set using the Trigger Workflow action definition.
    See About the transition framework
  • In the GCPM sample program, this workflow is configured to trigger in the transition Officer assessment complete - prepare recommendation.
  • The workflow Send Hard Commitment on Project is configured to trigger in the transition Offer Accepted.
  • The workflow Update Commitment – Amendment is configured to trigger in any amendment transition that references the Copy Amendment to Project action set.

To update FI objects on a periodic basis

In some cases you may wish to update the GCPM FI objects with the most recent financial system data. There are many ways that this could be accomplished using custom code, but there is also an out-of-the-box way to accomplish this: with a Bulk Delete system job.

  1. Create a new entity: CFG Financial System Synchronization
  2. Add a lookup to FI Commitment.
  3. Create a new workflow Call FI Commitment Synchronization on entity CFG Financial System Synchronization.
  4. Set the workflow to trigger on delete.
    You will need to create a new Expand Lookup custom action to expand the commitment's project lookup. See the other custom actions with the label Expand Lookup - <entity name> for an example.
  5. Set the workflow to trigger on delete. Create a new Expand Lookup custom action to expand the commitment’s project lookup.
  6. Add any conditional logic you desire, such as If Project decision equals Approved, Project not closed, FI Commitment has a financial system id.
  7. Call child workflow Update Commitment from Finance.
  8. Create a new record: CFG Financial System Synchronization and populate the commitment lookup.
  9. Navigate to Settings > System Jobs > More Actions > Bulk Delete. Look for CFG Financial System Synchronization.
  10. Set the scheduled time and any other settings you desire.
  11. Review, then submit.
