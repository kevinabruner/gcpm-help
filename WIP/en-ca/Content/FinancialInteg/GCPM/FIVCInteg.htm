Vendor customer integration

Vendor customer stores information in GCPM that is related to a Vendor and Customer in the financial system.

GCPM has several points that reference / interact with a vendor customer.

Prerequisites
  • Financial integration solutions
  • Workflows for financial integration
  • Financial integration custom workflow steps
  • Financial system entity models

Associate Vendor and / or customer with GC Client

To associate an existing vendor and / or customer with a GC Client:

  1. In a GC Client record, navigate to Financial System Data and click + to create a new VendorCustomer record.

  2. Change Vendor customer type to Mapped.

  3. Fill in the fields using onscreen help and the notes below: 
    • Name: Automatically populated with account legal name. Your organization can choose if you wish to lock this field.
    • Is Vendor: Yes / no. Yes will display the vendor-related fields.
    • Is customer:  Yes / no. Yes will display the customer-related fields.
    • Vendor customer map type: Indicates if this is a new or mapped (existing) VendorCustomer.
    • FI user: The user that is submitting the data to the financial system. Will be automatically populated if your organization decides to implement automatic population.
  4. Depending on the Is vendor and Is customer flags, you will see either Vendor code, Vendor customer code, or Customer code.
  5. Enter your financial system's unique identifier in the appropriate field.

  6. Save the record to pull in the data.

Automate FI Vendor Customer creation

To automate FI Vendor Customer creation during project life cycle:

  1. Navigate to Settings > Processes and open the workflow Generate VendorCustomer from Project (or another workflow you intend to use to generate the FI VendorCustomer record from a project).
  2. Expand the window such that you can see its ID in the address bar.
  3. Copy the ID somewhere for later use. Exclude %7b and %7d when copying.
  4. Navigate to your desired transition template: Programs > and select a program.
  5. Expand Project Lifecycle Templates and click Transition template lookup.
  6. Find the applicable transition and open its action set.
  7. If you need to create an action:
    • Add a new action, select Trigger Workflow definition, and fill in any other desired fields.
    • Scroll down and enter the workflow ID you copied in the step above and click Save Parameters.
    • Save and close the action.

Importing Vendors and Customers

If you choose to do a full import of Vendors and Customers from your financial system then you should keep the following in mind:

  • There is a lookup to FI Vendor Customer at the Account, GC Client, and Project levels that should be populated once the FI Vendor Customer records are created
  • Each account and GC Client only has one FI Vendor Customer. Create one record with both the Vendor and Customer data within it. Do not create two records to hold a vendor and a customer separately.

If your organization does not do a full import of VendorCustomer objects:

  • The Generate VendorCustomer from Project workflow is not smart enough to know if there is an existing VendorCustomer in your financial system that should be mapped to a project's GC Client rather than creating a new one.
  • To mitigate this, a validation action Verify Vendor does not already exist in Financial Systemcreates a warning in the event that the FI Vendor Customer field on GC Client is blank.
