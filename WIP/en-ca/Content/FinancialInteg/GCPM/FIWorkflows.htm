Workflows for financial integration

A series of workflows facilitate financial system transactions in the system and automatically create GCPM records and transitioning between different states for FI integration records (see state machine section).

See all FI workflows in this Excel file.

Commitment workflows
Workflows to manage commitments

Workflow

Description

Generate Soft Commitment
  • Generates an FI Commitment record.
  • Generates an FI Commitment Detail record for each cash flow record on the project
  • Triggers child workflow Sync and Send Commitment to Finance

Send Hard Commitment on Project
  • If a project does not have an FI Commitment this workflow makes a call to the child workflow Generate Soft Commitment
  • Updates project’s FI Commitment field to Type = Hard
  • Triggers child workflow Sync and Send Commitment to Finance

Sync and Send Commitment to Finance

Ensures this project has an associated commitment, otherwise it fails.

  • Changes associated FI Commitment to draft.
    See FI entity states (a.k.a state machine).
  • If this FI Commitment has a financial system ID then a call to Update Commitment from Finance is made (see below).
  • Updates FI Commitment’s FI User field to tell the financial system who is submitting the request. See for more information on setting up FI User records.
  • Submits the FI Commitment to finance:
    • If the submission is successful then a success transaction is generated and the workflow is complete.
    • If the submission fails then an error transaction is generated, the FI Commitment is set to an error state, and the workflow completes.

Sync Cash Flow with Commitment Detail – Create

Ensures FI Commitment Detail does not have a cash flow reference:

  • Ensures FI Commitment Detail has an associated project, otherwise fails.
  • Grabs a fiscal year record, based on the fiscal year specified on new FI Commitment Detail.
  • Stores the current state of the project’s Assessment Locked flag.
  • Sets the project’s “Assessment Locked” flag to No.
  • Creates a cash flow record, populating these fields: Project, Fiscal year, Amount, Expended amount, and Commitment detail reference.
  • Sets the project’s Assessment Locked flag back to original state.
  • Sets the “Cash flow” field on the FI Commitment Detail to the new created cash flow.
For these workflows: Sync Cash Flow with Commitment Detail – Create, Update, Delete
  • To improve performance, we recommend removing the steps that change the Assessment Locked flag. Instead, make a call to the action Enable Lock Plugin Override. This will allow the commitment detail to be created / updated / deleted without the expensive recursive action of unlocking the project.
  • Check for error status on the associated FI Commitment Detail so as not to update a cash flow record with invalid data.

Sync Cash Flow with Commitment Detail – Delete

Ensures FI Commitment Detail has a cash flow reference.

  • Stores the current state of the project’s “Assessment Locked” flag.
  • Sets the project’s Assessment Locked flag to No.
  • Deactivates the associated Cash Flow record.
  • Sets the project’s Assessment Locked flag back to original state.
For these workflows: Sync Cash Flow with Commitment Detail – Create, Update, Delete
  • To improve performance, we recommend removing the steps that change the Assessment Locked flag. Instead, make a call to the action Enable Lock Plugin Override. This will allow the commitment detail to be created / updated / deleted without the expensive recursive action of unlocking the project.
  • Check for error status on the associated FI Commitment Detail so as not to update a cash flow record with invalid data.

Sync Cash Flow with Commitment Detail – Update

Ensures FI Commitment Detail has a cash flow reference.

  • Stores the current state of the project’s Assessment Locked flag.
  • Sets the project’s Assessment Locked flag to No.
  • Updates the associated cash flow record’s following fields: Amount, Expended amount.
  • Sets the project’s Assessment locked flag back to original state.
For these workflows: Sync Cash Flow with Commitment Detail – Create, Update, Delete
  • To improve performance, we recommend removing the steps that change the Assessment Locked flag. Instead, make a call to the action Enable Lock Plugin Override. This will allow the commitment detail to be created / updated / deleted without the expensive recursive action of unlocking the project.
  • Check for error status on the associated FI Commitment Detail so as not to update a cash flow record with invalid data.

Update Commitment – Amendment

Ensures this amendment’s project has an associated commitment, otherwise it fails.

  • Updates FI Commitment and its details based on amendments to cash flow.
    • If a cash flow was added, it triggers Sync Cash Flow with Commitment Detail – Create.
    • If a cash flow was updated, it triggers Sync Cash Flow with Commitment Detail – Update.
    • If a cash flow was deleted, it triggers Sync Cash Flow with Commitment Detail – Delete.
  • Triggers child workflow Sync and Send Commitment to Finance.

Update Commitment from Finance
  • If the FI Commitment state is not Draft, the workflow fails.
  • Makes a call to retrieve the latest commitment and details from the financial system which updates the FI Commitment and its FI Commitment Details.
    • If the submission is successful, a success transaction is generated and the workflow is complete.
    • If the submission fails, an error transaction is generated, the FI Commitment is set to an error state, and the workflow completes.
  • The FI Commitment must have Status = Draft to edit. If you implement a periodic update of all objects from finance, changing the status to draft for all of them may take a while. If this is the case you could consider updating only FI objects related to "active" projects ("active" would be defined by your organization).
PAYE workflows
Workflows to manage PAYE

Workflow

Description

Generate FI PAYE for PAYE
  • Creates an FI PAYE record if one was not already created.
  • Pre-populates the Name, Amount, and Project reference.
  • Triggers child workflow: Sync and send PAYE to finance.

Sync and send PAYE to finance

  • Ensures the FI PAYE is at draft status. See FI entity states (a.k.a state machine).

  • Ensures the amount is greater than 0 so as not to submit an empty PAYE.

  • Sets the FI User field on the FI PAYE to tell the financial system who is submitting the request. See

  • Submits to financial system:

    • If the submission is successful, a success transaction is generated and the workflow is complete.

    • If the submission fails, an error transaction is generated, the FI PAYE is set to an error state, and the workflow completes.
Vendor Customer workflows

Workflows to manage Vendor Customers

Workflow

Description

Generate VendorCustomer from project
  • Creates VendorCustomer record.
  • Populates the record’s data from the current project’s account.
  • Sends the record to the financial system.
  • Associates record to this project’s account and GC Client records.
  • Associates record to this project.

Update account from VendorCustomer
  • Called on-demand by officer from account.
  • Updates this account’s VendorCustomer record from the financial system.
  • Updates bank account information on this account from newly updated FI VendorCustomer record.

  • Might want to check for error state here as an error may cause some of the FI VendorCustomer information to be empty (depending on how you implement) which could end up unintentionally clearing the values in your account

Update VendorCustomer from account
  • Called on-demand by officer from an account record.
  • Updates the account’s VendorCustomer record with data from the account.
  • Submits changes to the financial system.

  • Sets FI VendorCustomer to draft state. See FI entity states (a.k.a state machine).

  • Sets the FI User field to tell the financial system who is submitting the request. See .

Receivable integration workflows
Workflows to manage Receivable Integration

Workflow

Description

Sync and Send Receivables to Finance - Project
  • If this project already has a receivable then set its status to draft (see FI entity states (a.k.a state machine) for explanation of FI entity states)
  • If this project’s FI Receivable already lives in the financial system then trigger child workflow “Update Receivable From Finance” (described below)
  • Create or Update the FI Receivable with most recent repayment schedule installments
  • Trigger child workflow “Send Receivables to Finance” (described below)

Sync and Send Receivables to Finance – Amendment
  • If this amendment’s project has an FI Receivable then set its status to draft
    • If this amendment’s project’s FI Receivable already lives in the financial system then trigger child workflow “Update Receivable From Finance” (described below)
  • Create or Update the FI Receivable with amended repayment schedule installments
  • Trigger child workflow “Send Receivables to Finance” (described below)

Send Receivables to Finance
  • Ensure this FI Receivable has a project reference, otherwise fail
  • Change the FI Receivable status to draft
  • Update the FI Receivable’s FI User field to tell the financial system who is submitting the request.
  • Send the receivable to finance
  • If successful then set the FI Receivable status to Complete
  • Otherwise set the FI Receivable status to Error

Update Receivable from Finance
  • If this FI Receivable is not in draft state then fail
  • Unlock the associated project
  • Retrieve the latest FI Receivable details from the financial system
  • Re-lock the project
  • If the retrieve was successful then there is nothing left to do
  • Otherwise set the FI Receivable status to error

  • Note: this is another area where you may wish to override the lock rather than unlocking the project assessment
    See Override a lock for more information.
Payment integration workflows
Workflows to manage Payment Integration

Workflow

Description

Generate Payment
  • Creates an FI Payment
  • Creates FI Payment Details based on dollar amounts in the disbursement review:
    • If Adjusted Payment Amount > 0
      • Create FI Payment Detail with associated project’s coding for “Incurred Costs”
    • If New Advance > 0
      • Create FI Payment Detail with associated project’s coding for “Advance”
    • If Advance Settlement > 0
      • Create FI Payment Detail with associated project’s coding for “Advance Settlement – Incurred Costs” (positive amount)
      • Create FI Payment Detail with associated project’s coding for “Advance Settlement - Advance” (negative amount)
    • If PAYE Reduction > 0
      • Create FI Payment Detail with associated project’s coding for “PAYE”

      If Hold back > 0

      • Create FI Payment Detail with associated project’s coding for “Holdback” (negative amount)

      If Release Hold back > 0

      • Create FI Payment Detail with associated project’s coding for “Hold back” (positive amount)

Send Payment to Finance
  • Ensure this FI Payment has a project and disbursement review reference, otherwise fail
  • Change the FI Payment status to draft
  • Update the FI Payment’s FI User field to tell the financial system who is submitting the request
  • Send the payment to finance
  • If successful then set the FI Payment status to Complete
  • Otherwise set the FI Payment status to Error
