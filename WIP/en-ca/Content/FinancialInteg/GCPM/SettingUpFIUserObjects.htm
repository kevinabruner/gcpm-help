Setting up FI user objects

The FI User entity exists to provide financial system-related information about the user triggering a given request. The most common piece of information is the user's financial system user name. As of Spring 2019 this entity was not integrated into the data sent to the financial system API.

This entity can be expanded to include as much additional data as needed. Once expanded to include desired fields, adjustments to the workflow steps and financial system API will be necessary to include the data.

Note: The classes between the Financial System API and the workflow steps need to be kept in sync.

Scenarios for submitting data to the financial system

There are two common scenarios:

  1. Only financial officers submit data to the financial system.
    • Here the setup of FI Users will be remain mostly stagnant.
    • Create FI User records for each financial officer, then select appropriate records from within the CRM User object
    • As financial officers change, you must update the list of FI Users.
  2. Any user who has access to the financial system can submit data to the financial system.
    • Create an FI User object for each user with access to the financial system. As user access to the financial system changes, so to will this list need to be maintained. Each CRM user with access to the financial system will need their FI User lookup populated
    • This level of maintenance should be used only to clarify the GCPM user interface.
    • For example, if the user account does not have the FI User lookup populated, don't display functions that would result in submission of data to the financial system.
    • The financial system permissions should be configured to reject submissions from users without appropriate rights. This ensures that if the FI User list is not maintained, users who have had their rights revoked will not have the inappropriate right to submit data.
