Generic application questions

Generic questions that appear on portal applications must be configured in two places: 

  • In the application entity. See below.
  • in the portal Alphinat form. See Configure forms for the portal.

To create generic application questions, open the Application entity in the CRM interface: 

  1. Navigate to Settings > Solutions and choose the solution the application entity is stored in.
    For example, GCPM_Base.
  2. Navigate to Application > Fields and create a new field.
  3. Configure the field to suit the needs of the question. Note the options available will change, depending on the data type selected.
     
    • Display name: the displayed text for the question.
    • Field requirement: indicate if it is optional, business recommended or business required.
    • Name: this field is auto-populated.
    • Data type: This data type should match the type selected on the Alphinat form. If using option sets, the options should be identical on both the application and the Alphinat form.
  4. In the solutions view for the application, navigate to Forms. Select the form for the application.
  5. All the new fields you have created appear in the Field Explorer.
  6. Create a tab on the Application entity. The column view of information can be changed by selecting the Column type found on the Insert menu tab.
    Note all GCPM generic fields are captured in the General Information tab.
  7. To change the displayed tab heading text, double-click the new tab and edit the properties. Change the Label field to the text to be displayed on the form for that tab.
  8. To name the column sections, double click on the section within the tab to open the properties menu.

    • Label:Enter an appropriate display name.
    • Show the label of this section on the form: Check this box.
  9. Drag and drop fields into the desired tab and section on the form.
  10. Double-click fields to edit properties, such as Read only, lock field, visible.
  11. Save the form. To preview the form before publishing, select Preview > Desktop client > Create form.
  12. When the form is finalized, click Publish.
