Configure a clause template

Clause templates ensure the same clauses are copied to every project in a program. Templates allow standard clauses to be reused in different ways for different programs.

To configure a clause template: 

  1. Using Advanced Find, search for Clause Template | A. Create a new record.

  2. Name: Type a name for the template.
  3. Add clauses to the template. In the Clauses grid, click + NEW to add a new clause.

  4. Use the onscreen help and tips below to complete this record: 
    • Name: Type a short name for the clause
    • Clause template: This field is auto-populated with the template name.
    • Clause: Choose a clause from the available list, or create a new one.
    • Parent clause: If this clause is a sub-clause, choose the parent clause from the list.
    • Order: If this clause is a parent clause, it must have a unique whole number. If this is a child or grandchild clause, the number must be unique within the parent clause.
      See this note on clause order.
    • Mandatory: Indicate if this clause must be included.
      If mandatory is not checked, the clause may be removed from the project CA by an officer.
    • Editable: Indicate if the clause can be edited by officers.
    • Visible: Indicate if the clause is visible to officers.
  5. Save and close the record.

Tips:

  • You may configure clauses to be automatically inserted into a CA based on the response to an assessment question.
  • Mandatory + not editable + not visible = the clause is inserted into the CA based on logic configured in the document template.
    • Example: an officer manages sub-clauses, where the parent clause is standard text (e.g. a heading). In this case the parent clause can be mandatory + not editable + not visible. See Ordering clauses for more detail.
  • Mandatory + editable = officers can modify a required clause.
  • It does not make sense to have a clause be editable + not visible.
  • Test all options for the clauses you have configured in the template before deploying the template to a production environment.
Clauses not managed with clause templates 

Conditional clauses can be managed in the document template. For example, when an officer chooses which official language act requirements apply to a project, the appropriate clause can be inserted into the CA, if the document template for the CA has been appropriately configured. Conditional clauses such as these are not managed using clause templates.
