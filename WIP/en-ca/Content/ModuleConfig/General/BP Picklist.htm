Best practices

Modifying option sets

When configuring option sets in GCPM, it is recommended that existing (default) values are not changed. Existing values may be overwritten with GCPM updates. This could have a negative impact on usability, as well as existing data records. Option set values may be configured to trigger automation in business process flows and other features.

Recommended practice: 

If you encounter option set values in GCPM that are not appropriate for your organization, you have these options:

  1. Create new items in the option set, do not rename existing ones. Instruct users to ignore the options that are not applicable to your organization.
  2. Use lookup fields instead of option sets. Lookup values have the advantages listed below.

Risk of changing the name of an option set: ; It would not be recommended to rename existing ones -> due to possibility of overwrite with new GCPM Updates;

Modifying lookup fields vs. option sets
  • Lookup field values can be set to "inactive."
    • Users cannot apply inactive values to new records.
    • Existing records with the "inactive" value will not be modified.
    • For example, the Account Tag value "Clean Tech" is no longer applicable, but records with this value
  • Deleted option sets removes the data on existing records.
    For example, if an option set "Approve / Reject" is modified to "Accept / Reject", all records that previously had the value "Approve" will now have a blank field.
Using the Lock plugin
  • If you want to lock a new field on an entity, and there is an existing lock configuration for the entity, you must enable that field in the lock configuration record.
  • Best practice: wait until your localization efforts are completed to finalize lock configurations.
Modifying fields

Best practice: Do not delete fields, and do not change the purpose of a field. Create new fields instead, and either create new forms that display desired fields, or hide unwanted fields from the forms.

Using calculated fields

If a calculated field should be included into an existing formula, then all applicable calculation field configurations must be updated.

Quick views
  • Do not modify the display names of fields with the prefix qv_. These are quick view fields, wich are CRM "object names."

Best practices when working in the French interface

What "NOT" to translate

In the Scoring Rollup entity, the Rollup field is an option set with 4 functions: Sum, Avg, Min and Max. Do not translate these options! These are CRM functions and will not work if they are modified.
