GCPM security roles

A security role in Dynamics CRM is a granular set of permissions that controls what a user can do in the system. For example, a role may have access to Accounts, but no access to the Account’s credit rating. Individual users are assigned to one or more user roles to ensure they have access they need, while preventing them access to information they should not have. You can also grant a role to a group of users called a team instead of managing permissions on a per user basis.

For more information on configuring security roles, see the Microsoft documentation on security roles.*

GCPM ships with several example roles prefixed with "GCPM". It is not expected that you will use these roles directly; rather, they are there to denote which entities are associated to which major functions and plugins. For example, "GCPM Action Framework - Read" shows which custom entities are involved in granting a user / team read access to the action framework functionality. A user requires read access at a minimum in order to use most of the functionality in GCPM.

Go to Settings >Security >Security Roles to view them:

Stakeholder security roles

A user role is a set of permissions that controls what a user can do in the system. For example, a role may have no access to the credit rating of an account. Individual users are assigned to one or more user roles to ensure they have access they need, while preventing them access to information they should not have.

Best practice: 

  • To ensure easier integration with future GCPM releases, do not modify existing roles.
  • Use the “Copy Role…” feature to create you local copy of a role.
  • Read more about copying security roles*

The GCPM default roles are:

SH GCPM Regular User (e.g. officers, managers) 

Users with this role can create/modify the following data in Release 1.0: 

  • Create / modify accounts
  • Create / modify any attributes related to accounts: NAICS codes, tags-account, other addresses, connections, bank accounts, accreditations, etc.
  • Create / modify contacts
  • Create / modify any attributes related to contacts: NAICS codes, groups-contacts, other addresses, connections, bank accounts, accreditations, etc.
  • Create / modify G&C accounts
  • Create / modify any attributes related to G&C accounts: credit ratings , etc.
SH GCPM Bus Admin Users

Business administrators can do everything that end users can do, as well as:

  • Create / modify / delete lookup data (e.g. business types, account tags, contact groups, salutations, info request types, cities, etc.)
SH GCPM Config Users (Configurators) 

GCPM configurators can do everything that business administrators can do, and:

  • Create / modify records for CFG Bilingual
  • Create / modify records for CFG Calculation
  • Create / modify records for CFG Sequence
