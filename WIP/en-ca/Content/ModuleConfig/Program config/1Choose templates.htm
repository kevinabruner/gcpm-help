Associate templates with a program

There are several types of assessments typically done in the project life cycle. These templates work in a similar way:

  • Eligibility: a one-time assessment to determine if the applicant and proposed activities are eligible for the program.
  • Priority: a one-time assessment to score applications based on how well they meet priority criteria.
  • Project assessment: a one-time assessment to track due diligence and project-specific factors.
  • Risk: assessments over time of the project (or client) risk based on template-specific risk criteria.

When you select templates for a program, you establish a relationship between the program and a specific template. This relationships is stored as a record in an intersect entity ( .. | PG). Configuring templates for a program ensures that all projects in the program have consistent data captured and assessed throughout the life of the project.

One template can be used in multiple programs. For example, if the same risk assessment is conducted for projects in multiple programs, the same risk template would be chosen in each of the programs.

To associate a template with a program: 

  1. Open the program you wish to configure.
  2. Navigate to the section for a template. Click in the template field, and select the template you have configured for this program.
  3. Save and close the program.

Instructions to associate templates are also provided for each module. For example, see Choose a clause template for a program, Associate an application template to a funding opportunity, Choose a disbursement condition template for a program, etc.

Most GCPM templates features have these common characteristics:
  • Templates facilitate creating standard, reusable content in GCPM (e.g. questions, clauses, risk criteria, etc.).
  • A template that is not associated with a program has | A name (e.g. Clause Template | A).
  • When a template is associated with a program, a record is created in an intersect entity. (e.g. Assessment Template | PG) While users do not generally look at the records in an intersect entity, they are useful for reporting.
  • Program templates identify the data (template elements) that will be copied to new projects in the program.
    • Template elements are questions / categories and elements / clauses etc. depending on the feature.
    • Template elements are copied to a project at a point in time. Once they have been copied, changes to the template are not updated on the projects.
    • The clone plugin is the mechanism to copy template elements to projects.
    • A template can be associated with one or more programs.
    • Template elements are a type of intersect entity.
  • A template can be associated with one or more programs.
  • The start and end date on templates is for information only, no business logic is associated with these fields.

Note: 

  • The template features are different from document templates, transition templates and email templates.
  • This visual identifies how template features entities are designed.
Template best practice: 
  • If you need to modify a template, set the status of the template to inactive, and create a new template (use the SAVE AS button) with the desired changes. This ensures no data is lost for risk assessments already completed. Do not delete templates that are no longer used.
