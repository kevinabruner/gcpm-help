Program configuration

Programs that are structured hierarchically (parent-child relationships) facilitate reporting project data at different levels of the hierarchy. For example, reporting project data from a single stream, from multiple streams, at the sub-program or program level. Each bottom-level program (e.g. sub-program, stream, etc.) can have multiple intakes.

GCPM includes two fully configured sample programs to provide examples on ways to configure programs:

GCPM Core Program

This program was used by the development team as GCPM was developed and tested. It is fully configured, with examples for all the program-driven tools for project management. This includes:

  • Eligibility criteria: a basic template of seven eligibility criteria for projects requiring yes / no responses is applied to each project created in this program. Yes = 1, No = 0. Projects can readily be sorted by their eligibility score, and a score less than 7 (or whatever your organization would require) can result in rejecting the application.
  • Assessment questions: A template with 14 assessment questions is applied to each project created in this program. The assessment questions capture simple responses from officers' qualitative review of the clients' application. Unlike eligibility criteria, questions are not limited to binary responses. Two questions related to advances and change of control are used to dynamically provide content to the contribution agreement. Contribution agreements must be configured with the document generator module. Assessment questions can be dynamically displayed in different areas of their assessment forms using the "section title." For example, assessment questions about risk appear with risk assessments, without having to configure several forms.
  • Priority:  the priority assessment template can generate an overall score, composed of seven categories and 27 criteria. These criteria help to rank /order a list of assessed projects to inform pipeline and recommendation decisions.
  • Risk: the initial risk assessment is created for each project in the program. Risk assessments can be completed separately by the officer, or as demonstrated on the GCPM assessment form they can be dynamically displayed in various sections right on the assessment form and completed by the officer as they progress through their various assessment tasks. The GCPM Core program does not include additional risk templates. These other risk templates, while not used during a project assessment, would be the valid templates for officers to use for other subsequent re-assessments of risk during the project lifecycle.
  • Business process flows and transition templates applied to GCPM core program fully demonstrate all the features that can be configured by an RDA to leverage using changes in status. The project lifecycle transition template (GCPM core repayable transition template) is configured to demonstrate all the standard required statuses and movement of a project from application to file closure. The transition template includes examples of transitions and stages that dynamically show / hide and lock / unlock parts of the project assessment form. This increases the flexibility for your organization to maintain a single project assessment form in the solution, while having specialized assessment processes for various programs. The core repayable transition template also provides examples of flexible action sets used to create records, manipulate statuses for the locking / unlocking of features (e.g. disbursement or amendment), and creating a snapshot of information for “moment in time” reporting or information management requirements. Similarly, it includes examples of decisions and approval pathways / authority types for the recommendation on whether to fund a project and move an agreement through the negotiation and execution.
  • Transition templates for disbursement status changes and amendment transitions are also included.
  • Checklists: The core program includes other templates to facilitate project lifecycle management, and ensure a proper audit trail or due diligence is followed. These include disbursement, amendment, and monitoring checklists.
  • Clause template: Different parts of the agreement can be included in a template and managed based on project coding. Unique contribution agreement templates that interact with the program assessment questions, special conditions as well as separate conditions for disbursement / pre-disbursement that officers can maintain throughout the lifecycle.
  • Performance measurement: Demonstrates how indicators can be associated to agency level outcomes and objectives and intersected across multiple or single programs to facilitate consistency in collection / coding of indicators and reporting on results.

Women Entrepreneurship Strategy

This program, with an associated Ecosystem sub-program and Regional Ecosystem stream are examples of how a larger program can be structured. GCPM includes inactive records for the Entrepreneurship Fund and National Ecosystem that can be enabled if your organization wishes to reference a more complex model.

Many of the same templates or features in the GCPM Core program are re-used in the Regional Ecosystem program to ensure it works for complete project lifecycle. The additional functionality included with this program is the application integration with the portal. A configured application form is available on the portal, similar to the WES application developed by ISED (Innovation, Science and Economic Development) for the regional development agencies. The priority ranking template for WES is configured to automatically run on applications submitted by clients on the portal. This provides an example of automation that can be configured at your organization.
