GCPM portal components

GCPM contains several custom components specifically for the portal to interact with GCPM. Components include custom actions that provide a public API that is consumed by the portal, custom fields to store portal-specific data, as well as portal-specific business processes within GCPM.

Custom Plugin

The plugin EGCS.CRM.Plugins.Portal contains the following entity business logic related to the portal that is executed server side.

 
Name

Action

Description

egcs_document

 

 

On Update

When a new GCPM portal document is uploaded and successfully makes its way to the internal document repository, assign a document id to the record and fill in the portal metadata. Synced status is set to false and sync type is set to Update.

On State Change

If a document is deleted (record is deactivated), set the document sync status to false and sync type to Delete.

On Delete

Checks whether the document record can be deleted or not. If the document has not been deleted on the portal side, the document record cannot be deleted.

egcs_sg_client

On Update

When a verified GC Client has the field “Client Portal Site” set to yes, the plugin will initiate the client portal site creation on the portal side.

egcs_pt_portaluser

On Create

Fill in the record name which is a concatenation of first, middle, and last name.

On Update

Fill in the record name which is a concatenation of first, middle, and last name.

egcs_pt_securemessage

On Create

When a new message is created, whether by portal or GCPM, first validate the message to ensure it contains proper data, then set the message id if it doesn’t exist, and set other metadata.

On Update

Prevent the message from being updated after creation.

On State Change

Prevent the message from being updated after creation.

On Delete

Prevent the message from being updated after creation.
