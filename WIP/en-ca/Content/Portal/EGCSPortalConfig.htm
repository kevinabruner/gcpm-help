EGCS.Portal configuration

Configuration settings for the EGCS.Portal project are stored as .NET settings in the EGCS.PortalSharePoint.Library project. Any environment which loads EGCS.Portal.SharePoint.Library.dll must also load configuration settings for the DLL, otherwise the portal code will revert to compiled-in defaults, which will almost certainly be wrong.

  • Configuration is typically loaded from web.config, or a .config file for the executable which references the EGCS.Portal.SharePoint.Library.dll.
  • In all cases, the config file will contain a reference to EGCS.Portal.SharePoint.Library.Properties.Settings:

<configSections>

    <sectionGroup name="applicationSettings" type="System.Configuration.ApplicationSettingsGroup, System, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" >

       <section name="EGCS.Portal.SharePoint.Library.Properties.Settings" type="System.Configuration.ClientSettingsSection, System, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" requirePermission="false" />

    </sectionGroup>

</configSections>

The settings are stored as XML. They are considered static. They are loaded when a runtime environment using EGCS.Portal.SharePoint.Library.dll is initialized, and do not change over the lifetime of execution. The runtime environment must be reloaded to read new settings. This means an IISRESET, application pool reload, or stopping / restarting a service or executable.

Installation

EGCS.Portal configuration settings are deployed by a PowerShell script. The script will copy the settings to all locations on the current server which require it, and reset/reload any dependant applications.

Installing EGCS.Portal configuration requires a build of the EGCS.Portal solution, which is assumed to be available in a directory “_Build”.

  1. Install EGCS.Portal configuration, specifying the configuration file to be loaded

    .\_Build\Scripts\Install-Egcs_Config.ps1 .\_Build\Config\<portal name>.egcs.config

  2. Deploys to:
  • SharePoint web application for the portal. Modifications are written to web.config for the web application, all zones. The portal web application is recycled in IIS.
    Files: C:\inetpub\wwwroot\wss\VirtualDirectories\<folder>\web.config
  • The Document Management Windows Service. Restarts the service.
    File: C:\Program
    Files\EGCS\EGCS.Portal.Service.DocumentManagement\egcs.config
  • The Site Creation Windows Service. Restarts the service.
    File: C:\Program Files\EGCS\EGCS.Portal.Service.SiteCreation\egcs.config
  • The portal web service. Recycles the application pool.
    File: C:\inetpub\wwwroot-webservice-http\EGCS.Portal.WebService.Portal\egcs.config

Configuration

The configuration file is found in the EGCS.Portal.SharePoint.Library project.

EGCS.Portal.SharePoint.Library\Config\<portal name>.egcs.config

Settings

The configuration file is divided into several sections:

Token

The “Token” section contains name / value pairs which are substituted into the rest of the file. The substitution pattern is %Token%. Use of tokens is not required, it is simply a shortcut to avoid repeating the same text multiple times.

New tokens can be added to the Token section as desired.

Application Settings

Application settings are the .NET settings for the EGCS.Portal.SharePoint.Library application. Effectively this is the contents of app.config. This configuration data is copied into SharePoint web.config, or becomes the egcs.config file for other executables or web applications.

These settings define the runtime values used by portal code.
