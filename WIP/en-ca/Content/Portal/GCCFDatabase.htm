GCCF database

The GCCF database is a SQL Server database which acts as a repository for the portal.

  • Not a SharePoint database, this database is only used by components of the portal.
  • Should be backed up at the same time as SharePoint databases.
  • Does not store the actual Client, Project, or Personal zone contents. Those zones are SharePoint site collections (SPSite) and sites (SPWeb) which are stored in SharePoint content databases.
  • A full list of tables is documented below. At a high level, the GCCF database stores:
    • Location of Client, Project, and Personal Zones
    • Mappings of GCCF Users between their GCCF identity (PAI) and their portal identity (PortalUserId)
    • GCCF user profiles
    • Two-factor authentication tokens
    • Terms and Conditions
    • Invitations
    • Master page configuration
    • Registry of documents (the actual documents are stored in SharePoint)
    • Registry of secure messages (the actual secure messages are stored in SharePoint)
    • Log files

Security and roles

Access to read and write GCCF database tables is performed exclusively through stored procedures. Users should belong to the GCCFAdfs or GCCFUser roles in the GCCF database to execute portal stored procedures. These roles are created by the installation script GCCF Database.security.sql in the GC.GCCF solution.

Service accounts used by the portal which should belong to the GCCFAdfs role:

  • ADFS Service account
  • Service accounts for GCCF Common Domain and ADFS Extensions web applications (this is usually the ADFS service account, but that is not required)

Service accounts used by the portal which should belong to the GCCFUser role:

  • SharePoint service account
  • Service account for portal web service (usually the SharePoint service account)
  • Service accounts for Site Creation Windows and Document Management Window services (usually the SharePoint service account)

Submission

Submissions (forms) known to the portal.

Includes both draft and submitted forms.

TermsAndConditions

Text of the Terms and Conditions for the portal (French and English).

Must be populated by the department, if they wish to use Terms and Conditions.

TermsAndConditionsAccepted

Record of which GCCF users have accepted which Terms and Conditions.

This table is populated and managed automatically.

TwoFactorAuthenticationRetry

Count of retry attempts for a user performing two-factor authentication.

If a user’s retry attempts are exhausted, they will be locked out of two-factor authentication.

Two-factor authentication tokens which have been issued to users.

A user’s browser must supply a valid two-factor authentication token in order to use the portal (if Two-Factor authentication is enabled).

History of two-factor authentication activity, such as codes being issued, and tokens being extended.

Code table.

White list of mobile phone numbers that the portal is allowed to send SMS messages to.

Intended for use on development or test portals.

Prerequisites

  • A build of the GC.GCCF solution, which is assumed to be available in a directory _Build.
  • SQL Server. The SQL Server used by SharePoint is acceptable.
  • A list of service accounts which need access to the GCCF database. See the section “Security and Roles” above.

Installation

  1. Log onto SQL Server with a DBA account.
  2. Create the database.
    • Edit file _Build\Database\GCCF Database.sql
    • Edit the FILENAME for the mdf and ldf file to place the files where desired, on the server.
    • Adjust size and growth parameters for the database if required.
    • Execute _Build\Database\GCCF Database.sql to create the database.
  3. Populate code tables.
    • Execute _Build\Database\GCCF Database.codetables.sql
    • If site definitions used by the department don’t match site definitions used by the Integration portal, the rows in MasterPageTab will need to be updated.
  4. Configure security.
    • Not scripted: grant all service accounts access to SQL Server if they don’t have it (SharePoint and ADFS service accounts will have access via their own installers). No specific server role is required, other than public (don’t change any roles for ADFS or SharePoint).
    • Edit file _Build\Database\GCCF Database.security.sql
    • Create users in the GCCF database for all service accounts.
    • Assign service accounts to the roles GCCFAdfs or GCCFUser.
    • Execute _Build\Database\GCCF Database.security.sql
  5. (Optional) Load Terms and Conditions.
    • This step is required if the portal will use Terms and Conditions.
    • English and French HTML text must be loaded into table TermsAndConditions.
    • See example in GCCF Database.demodata.sql in the GCCF Library project.
  6. The contents of the FormDefinition table need to be updated to match forms offered by the department. FormName must match the SmartGuide name of the form.
