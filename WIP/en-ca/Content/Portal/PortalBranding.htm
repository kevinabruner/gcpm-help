Element 8 - Tabs

Tab bar (configurable options, displayed are: Recent activity, messages, shared documents, site members, invitation, summary). In order to update the tabs that appear for a specific portal zone, update a row in GCCF.dbo.MasterPageTabs

  • Scope: Indicates which Portal Zone the tab appears in.
  • TabOrder: The order the tabs are rendered on the portal, from left to right
  • IsHiddenTab: Indicates if the tab is hidden.
  • PageUrlScope: Metadata for rendering proper links in sub-zones of a Portal Scope.
  • PageUrl: When clicking on the tab, provides the redirect link for the page.
  • Label_<lang>: The labels that the tab displays for the user’s respective language choice

Where to find portal elements

  • Almost all assets, including an image source URL, text displayed to the user, or URL redirect values are configured in the EGCS.SharePoint.Portal.Library project’s EgcsTranslation files (EgcsTranslation.resx and EgcsTranslation.fr.resx).
  • Any labels added to support the MasterPage that were not originally from the WET theme are denoted by a prefix of MasterPage_.
  • Other locations that the portal pulls assets from include: the GCCF database (for instance, a portal zone’s tab bar [8] is a set of URLs and labels that are selected from a table).
  • Image source URLs are stored in EgcsTranslation files, and the images themselves are stored in the MasterPage Layouts Folder.
  • Textual values that do not need to support translation can be found in the EGCS.Portal.SharePoint.Library project’s Settings file.
  • Other constant values can be found in EGCS.Portal.SharePoint.Library project’s Constants class.
  • At the bottom of the WET4Canada.master and WET4Search.master files is functionality to set the value of multiple literal controls. Some of these literal controls exist purely for accessibility purposes (meaning, they are picked up by screen readers but not noticeable to the average user), others are values that are almost always displayed and available on the portal.

CSS

Many colour and size changes are done using CSS. Note: Because the master page is using the WET Theme, there are some attributes that are explicitly being marked as ‘!important’ and thus although more specific attribute changes are being made, they are being overwritten. If you are attempting to update a an elements CSS and noticing the changes are not being reflected after clearing your cache, please view your development tools to ensure that a WET !important notation is not causing the issue.

Most of the header elements can be changed by updating or adding their id element along with the appropriate CSS attributes in header.css. Other master page changes can be made in override.css.

SmartGuide

Branding changes for SmartGuide are handled by updating the ‘WET_THEME’ theme. The most common files that may need to be updated to reflect your branding are:

  • custom.css (located in the resources folder)
  • Theme files translation spreadsheet – which can be imported and exported by navigating to the Themes tab, and using the export and import buttons1 on specific Theme folder rows.

Any changes to the look and feel of the PDF snapshot can be made in the ‘PrintOut’ folder. Information on designing SmartGuide themes, including PDF templates, is described on page 76, SmartGuide_7.0 AuthorGuide.
