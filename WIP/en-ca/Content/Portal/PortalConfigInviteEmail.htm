Portal configuration - invitation email

Invitation emails can be fully localized by the RDA to change the text, or other content of the email. The email system used by the portal allows flexible substitution of tokens into the text of an email before the email is sent. This allows the RDA to fill in arbitrary information like user names, site name, or URLs. Any information to which the portal has access could potentially be used in an email message.

  • Localizing the Invitation email required building a new version of the EGCS.Portal.SharePoint.Library project. The invitation text is deployed as a resource in the translation DLL for the project.
  • The email template is found in the EGCS.Portal solution:

    $\EGCS.Portal\EGCS.Portal.SharePoint\EGCS.Portal.SharePoint.Library\Translation\invitationEmail_For_ExternalClient.txt

  • The email template uses iso-8859-1 character encoding.
  • The contents of the email template is a HTML document. The invitation email is sent as HTML.
  • The <title> element of the email template will become the subject of the email.
  • There is only a single email template, which contains bilingual text.
  • An email template can contain any number of tokens. A token is defined as a string surrounded by ‘%’. E.g. %TokenName%. New tokens can be added as desired by the RDA.

The default set of tokens for the invitation email are:

  • SITE_NAME_EN – English name of the Client site.
  • SITE_NAME_FR – French name of the Client site.
  • INVITED_BY – Name of the person who is sending the Invitation.
  • INVITATION_URL_EN – URL to accept the Invitation. Sets the user’s language to English.
  • INVITATION_URL_FR – URL to accept the Invitation. Sets the user’s language to French.

Note: While it would be technically possible to include the shared secret in an email, it is not advisable to do so. The shared secret should be provided separately to the person being invited using a different communications channel, such as phone.

  • Class EGCS.Portal.SharePoint.Library.Communications method InvitationEmailForExternalClientCreate() takes an Invitation object and constructs an Invitation email. This method looks up all of the token values to be substituted into the email.
  • The use of shared secrets for an Invitation is controlled by a SharePoint Property Bag value named InvitationEnabledSharedSecret which is configured at the SPWeb level. This is an optional property and only applies to SPWebs where is property is defined as a Boolean.

    If InvitationEnabledSharedSecret is not present, or boolean true, a Shared Secret will be generated for an Invitation.

    If InvitationEnabledSharedSecret is boolean false, a Shared Secret will not be generated for an Invitation.

  • egcs.config property InvitationSharedSecretLength controls the length of a Shared Secret. Default 8 characters. This length applies to all SPWebs which send Invitation emails.
