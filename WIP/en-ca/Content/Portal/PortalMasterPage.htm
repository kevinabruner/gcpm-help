Portal - master page overview

The EGCSPortalMasterPage Project consists of two Master Pages; a series of Web Controls; a Layouts Folder to hold site assets including images, JavaScript, and CSS files; and a generic C# Help Class.

Translations are handled using the EGCS.Portal.SharePoint.Library project’s resource manager.

Master Files

The two Master Files included are WET4Canada.master and WET4Search.master. Both of these master pages have the same code behind class, WET4PublishingMaster.cs. The differentiator is that the WET4Search master page has a left navigator added to it in order to support search result refinement in the Portal Search Center.

Web Controls

For rendering content that depends heavily on logic, the Master Pages rely on WebControls. The portal MasterPage WebControls include the following:

Renders the Language Toggle Link, adds JavaScript function to the master page which sets the LCID cookie to the language value being toggled to (LCID value is a configuration that can be changed)

*MessageDropdown
  • Generates Message drop down Letter Icon in the top white row of the master page header.
  • Drop down contents include Personal Message, Client Message if in a client or project zone, and project message unread counts and redirect URLs to each respective message centre.
  • The Message Icon itself also has a rollup that tallies the total unread messages from each zone.

*Midbar
  • Renders the middle, dark blue bar for the portal master page.
  • The main functionality of the MidBar is navigation through the portal’s Zones.
  • Midbar always contains a link to the welcome centre, as well as a Create dropdown to create messages and forms (if the user is logged in).
  • If the user is signed in as a GCCF user, they also see a list of their available clients.
  • If a user is in a client zone, they also see a list of the available projects to that client.

Layouts folder

The Layouts folder contains static assets that are used by the master page. These include images, CSS, and JavaScript. They are categorized into further subfolders for organization purposes.

For branding purposes, most label / textual changes can be done by finding the appropriate label in the library project’s EgcsTranslation file. A few labels (such as error messages in dialogs), and images can be found in the MasterPage Layouts folder. For more information see Portal branding
