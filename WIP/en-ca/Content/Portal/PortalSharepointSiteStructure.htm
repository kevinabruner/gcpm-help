SharePoint - portal site structure

The portal is a SharePoint web application, and therefore all content in the portal is composed of SharePoint site collections (SPSite) or sites (SPWeb). Some of these sites are create automatically by portal services, and some of these sites are created by a SharePoint administrator at the time the portal is installed.

Note: 

Two items in the property bag for a site that drives MasterPage functionality:

  • PortalScope depends on the property bag for the current web it is being accessed in. Only one of these property values will exist for a given SPWeb.
    • <SPWeb>.AllProperties[“egcs_PortalUserId”] (Personal zone)
    • <SPWeb>.AllProperties[“egcs_AccountNumber”] (Client zone)
    • <SPWeb>.AllProperties[“egcs_FCProfileID”] (Project zone)

  • To determine header information:

    • SPWeb>.AllProperties["MasterPageHeaderScope"] = ‘Welcome’, ‘Client’, or ‘Project’

The set of portal sites includes:

Anonymous root
Anonymous root portal site

Item

Description

Purpose / contents
  • Allow internet users to discover the portal.
  • The splash page for the portal, where users initially arrive and select their language of choice.
  • Help content for the portal.
  • Pages to support login, Terms and Conditions, two-factor authentication, and user registration.
  • Any other content on the portal which should be available to the general public without authentication.

Typical URL
  • /

Permissions
  • Anonymous access must be enabled – this is the only section of the portal which should permit Anonymous access
  • “Read” access to the Everyone claim c:0(.s|true

SharePoint object
  • Site collection (SPSite)

How is it created?
  • By a SharePoint administrator when the portal is being installed and configured.

Apps

  • spfx-terms-solution
  • spfx-portal-user-solution
  • egcs-portal-share-point-spfx-factor-auth

Pages and web parts

URLs for the various pages are configured in gccf.config.

  • A page for Terms and Conditions which hosts the web part
    TermsandConditions.
    On the Integration Portal, this is /Conditions.aspx
  • A page for two-factor authentication with hosts the web part
    TwoFactorAuth.
    On the Integration Portal, this is /2FA.aspx
  • A page for registering a user profile which hosts the web part
    UserProfileInfo.
    On the Integration Portal, this is /ManageAccount-GererCompte.aspx

Notes
  • A department may wish to create SharePoint variations sites under the anonymous root to host help, or other language-specific content. e.g.
    /en, /fr

  • Static pages that report “Error”, “Login Success”, “Logout failed”, etc., may also be hosted in Variations sites. Those page URLs are configured in gccf.config.

  • If the department wishes to implement their own Chooser page, it can be hosted under Variations sites.

 /Welcome-Bienvenue

SharePoint object

Site collection (SPSite)

How is it created?

By a SharePoint administrator when the portal is being installed and configured.

Apps

spfx-portal-user-solution

Pages and web parts
  • default.aspx page should host the web part Welcome Centre Table of Contents or something similar.
  • A page for updating a user profile which hosts the web part
    |UserProfileInfo.
  • On the Integration Portal this is, /Welcome-Bienvenue/manageaccount-gerercompte.aspx.

Notes
  • The location of the Welcome Centre is configured in EGCS.Portal configuration – setting UrlWelcome

  • Page names must match the Welcome scope definition in GCCF.dbo.MasterPageTab
    (where PageUrlScope = ‘Welcome’)

Client Zones
Client Zone portal site

Item

Description

Purpose / contents
  • Represents a GC Client published to the portal from GCPM.
  • View profile information about a GC Client.
  • Submit applications for the client.
  • Send and receive secure messages for the GC Client.
  • Share documents with the RDA.
  • Invite and manage other users’ access to the client.
  • Contains all Project Zones for the client.

Typical URL

 /client/<Account Number>

Permissions
  • Restricted to GCCF users who have been specifically invited to the client.
  • The department may choose to automatically give access to groups of staff members (RDA users).
  • Security is implemented via SharePoint security groups and roles.

SharePoint object

Site collection (SPSite)

How is it created?
  • Automatically, by the site creation Windows service.
  • When an RDA staff member chooses to publish a client to the portal, the portal is notified (via the portal web service).
  • A row is added to GCCF.dbo.SiteQueueAccount.
  • The site creation service periodically creates any new GC Client sites in the queue.

Apps

  • egcs-portal-share-point-spfx-project-client
  • spfx-portal-user-solution
  • spfx-message-centre-solution

Pages and web parts
  • default.aspx hosts DraftSub and AppStatus web parts.
    Allows users to review draft submissions and applications which have been submitted.
  • message.aspx hosts SecureMessageCentre web part.
    secure messaging control.
  • members.aspx hosts UserManagement web part.
    Review users who have access to the Client. Users with admin privilege can change the role of other users, or remove them from the site.
  • invitation.aspx hosts siteInvitation web part.
    Invite new GCCF Users to the client.
  • profile-profil.aspx hosts ClientProfile web part.
    View profile information about the client.

Notes
  • The root location for client sites is configured in EGCS.Portal Configuration – UrlClientSiteRoot
  • Pages names must match the client scope definition in GCCF.dbo.MasterPageTab
Project Zones
Project Zone portal site

Item

Description

Purpose / contents
  • Represents a project (FCProfile) published to the portal.
  • View profile information about a project.
  • Submit claims, advances and performance reports for a project.
  • Send and receive secure messages for a project.
  • Share documents with the RDA.
  • View summary disbursement and budget information about a project.

Typical URL

/client/<Account Number>/<FCProfileID>

Permissions
  • Inherited from the Client Zone.
  • All project zones for a client share the client zone permissions (all users invited to a client have the same level of access to all Projects for the client).

SharePoint object

Site (SPWeb)

How is it created?
  • Automatically, by the site creation Windows service.
  • A new project zone is added to a client when a new project (FCProfile) is created in GCPM.

Apps

  • egcs-portal-share-point-spfx-project-client
  • spfx-message-centre-solution

Pages and web parts
  • default.aspx hosts DraftSub web part.
    Allows users to review draft submissions.
  • message.aspx hosts SecureMessageCentre web part.
    Secure messaging control.
  • summary-sommaire.aspx hosts web parts BudgetSummaryand DisbursSummary. Displays budget summary line items, and disbursement summary line items, respectively.
  • repaymentsched-calendrierderemb.aspx hosts Repayment Schedule.
    The repayment schedule for the project.

Notes

Pages names must match the project scope definition in GCCF.dbo.MasterPageTab
Personal Zones
Personal Zone portal site

Item

Description

Purpose / contents
  • A user’s Personal Zone on the portal.
  • Submit applications to the RDA.
  • Send and receive secure messages for the project.
  • Share documents with the RDA.

Typical URL

 /my-moi/<Guid>

Permissions
  • Restricted to a single GCCF User.
  • The department may choose to automatically give access to groups of staff members (RDA Users).
  • Security is implemented via SharePoint security groups and roles.
  • There is no ability to invite other GCCF users to this zone.

SharePoint object

 Site (SPWeb)

How is it created?
  • Automatically, by the site creation Windows service.
  • The service will maintain a queue of unassigned Personal zones in the database table GCCF.dbo.SiteQueueUser.

  • New Personal Zones will automatically be created as zones are removed from the queue.

  • The size of the queue is set in EGCS.Portal Configuration – SiteCreationUserSiteQueueSize.

Apps

  • egcs-portal-share-point-spfx-project-client
  • spfx-message-centre-solution

Pages and web parts
  • default.aspx hosts DraftSub and AppStatus web parts.
    Allows users to review draft submissions and applications which have been submitted.
  • message.aspx hosts SecureMessageCentre web part.
    secure messaging control.

Notes
  • The root location for personal sites is configured in EGCS.Portal Configuration – UrlUserSiteRoot
  • Pages names must match the Welcome scope definition in GCCF.dbo.MasterPageTab (where PageUrlScope = ‘Personal’)
  • <Guid> for a Personal Zone is randomly chosen.
