Sharepoint installation

The portal requires SharePoint 2016 on-premises version. While the design and installation of a SharePoint farm is out of the scope of this documentation, the Integration Portal for the GCPM project can be used as a rough guide towards creating a working environment to host the portal. Microsoft provides extensive documentation about SharePoint.

Factors to consider

When planning a portal installation, consider the following:

  • Does the portal require high availability?

    Do you expect to have very large numbers of users hitting the server simultaneously?

    If so, plan for a farm with multiple Web Front Ends.

  • Plan adequate disk space for future growth. You can estimate some Portal size requirements by creating hundreds or thousands of Client / Project sites in a content database. However, that estimate may be misleading.

    Keep in mind that portal users will be uploading arbitrary files to their sites. Those files are also stored in SharePoint content databases. Your disk space requirements will depend on your number of projected users, and what sorts of files you expect to be uploaded.

  • Plan for multiple content databases. SharePoint will balance the creation of site collections across all available content databases. Use multiple databases to spread the load out. Ensure you monitor the size of your content databases, and add new databases to the farm if the old ones start approaching the limits for a SharePoint content database*.
  • Consider automating SharePoint installation using AutoSPInstaller*, Microsoft’s Desired State Configuration*, or some other form of scripted setup. These tools provide highly repeatable configurations when moving from development to testing to production environments.
  • How will RDA users access the portal?
    • If the Portal is hosted on-premises at the RDA, then Windows authentication against a domain controller at the RDA is an option. If Windows Authentication is used, care must be taken to ensure that Internet users cannot use Windows authentication against the Portal from the Internet.
    • If the portal is hosted in a remote data center, an ADFS server will probably be required.

Required services

The portal requires the following SharePoint services:

  • App Management Service
  • Application Discovery and Load Balancer Service Application
  • Subscription Settings Service
  • Search service, if the Portal is to provide a Search interface.
  • State Service
  • Usage and Health Data Collection (not required, but desirable)

Manually Created Site Collections

The Portal automatically created Personal zones, Client zones, and Project zones. However the root site collection (/), the Welcome Centre (/welcome-bienvenue), and the Search Centre (/search-recherche) must be created manually, but a SharePoint administrator. See the document “Portal site structure” for more information.

Recommendations

  • Disable SharePoint’s “Minimal Download Strategy” on all sites. Ensure this is done for root, Welcome Centre, and Search Centre.

    $web = Get-SPWeb https://<portal>/site

    $web.EnableMinimalDownload = $false

    $web.Update()

  • Disable SharePoint’s mobile view on all sites

    _Build\Scripts\Disable-MobileView.ps1
