Configure forms for the portal

The portal is configured to display application forms designed in GCPM to portal users. Portal application forms can be:

  • Static: a common set of fields and questions is displayed. In this case, each program / initiative may have a separate application form.
  • Interactive: as users enter information about their project and organization, the data presented to the user dynamically changes. Form design can tailor questions based on input from the client

Portal accounts interact with GCPM account records. The portal can facilitate creating new accounts, and the management of client information, flagging where necessary when requested changes / updates are required.

When a client "Submits" an online application, the data is transferred to GCPM using web services. There are several configuration tasks related to the portal: 

  • Design portal application forms using the SmartGuide Designer software. Portal fields are mapped to GCPM fields.
  • Create static or dynamic portal forms, and other options.
  • Modify or build your own web services to transfer data from the portal to GCPM.
