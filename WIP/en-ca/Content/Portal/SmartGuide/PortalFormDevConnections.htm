Developing portal forms - connections

There are two types of connections configured for GCPM: API extension functions and SharePoint web service calls.

Extension functions

Extension functions are used across multiple forms are included in the table below.

API extension functions for portal connections

Extension function

Description
SmartletSerializer

 

This function is called on form save or submit.

Iterates through the form and serializes the input fields and metadata.

The root node is named after the internal name of the Smartlet, with its children nodes being Metadata and any entities that exist on the form.

The Metadata children are fields that are indicated as being serializable but do not follow the entity.field naming convention.

The entity nodes have their fields as direct children. The serialized XML string is stored in the configuration field named ‘xml’

GeneratePDF

This function is called on form Submit. Generates the based64 encoded string of the PDF representation of the form and user input. The contents of the PDF are determined by what is in the data recap field on the Review page.

You can use the following naming convention to change which page(s) render in the PDF:

byte[] bytes = smartlet.generateFile(Constants.FileType.PDF, $"P{page1Id}}:PrintOut |P{page2Id}:PrintOut");

The naming convention is ‘P’ + IdOfPageToAdd + ‘:’ + ‘NameOfTemplate’. Any additional pages follow the same naming convention, with a pipe ‘|’ character in between the template name and ‘P’ character.
InitializeRequestParameter

Pulls data from request parameters and sets their corresponding config fields.

Called on Smartlet initialization.
InitSmartlet

Called on form load, calls the LoadDraft webservice and SmartletDeserializer extension function.

Other extension function and web service calls may be made that are more form dependent.
ImportCsv

Allows the user to download a provided template, populate it with data, and upload it to the form in order to create rows in a repeatable group.

Two parameters, the name of the SmartGuide upload field and repeatable group need to be provided.

Note:

  • The user must upload a csv with headers that match the labels of the fields in the repeatable group.
  • The form does not care if the template language matches the current SmartGuide language (can upload English template in French form context and vice versa) however it does assume that input will match the CSV language – meaning currencies will follow French currency formats in the French template if they use any currency formatting.
  • Repeat groups that have dropdown / selection fields or calculated fields will also be ignored and not needed in the CSV template.
SharePoint web service calls
  • SharePointConnections: Contains all of the web service calls the form makes.
  • Any web service call to CRM is made through a SharePoint web service. For more information, see Dynamic portal forms

When adding new web services, it is important to include the following headers along with their default values:

Header Value

Content-Type

application/json;odata=verbose

Cookie

${requestHeader("Cookie")}$

Referer

${"https://" + requestHeader("Host") + "/"}$

When adding a URL to the web service, include the parameter {Client-Site} set to the configuration variable site.
