Invite a GC Client to the portal

You can a link to a GC Client's portal account via email. First create (or validate) the portal link, then send the email.

To create a portal link on a GC Client Account:

  1. Navigate to Stakeholders > GC Clients and open a GC client record.
  2. Verify the following fields are completed: 
    • GC Client primary contact: Select a valid contact.
    • Lead officer on record: Select the lead officer.
    • GC Client portal = Yes
    • Hold= No
    • Recovery state = No
    • Default state = No
  3. Navigate to Client Transitions and select Set client ready for funding.

    Note: In order to set a client "Ready for funding", these fields must be completed:
    Account: Legal name, GC Client type = Yes, Business ID validated = Yes
    GC Client: Lead office
  4. Allow a few minutes for the creation of the portal site. Then click the Portal site hyperlink to launch the sign-in page.
  5. Log into portal, select Continue with RDA Authentication.

To send a portal access invitation to a GC Client:

  1. In the GC Client record, click Portal hyperlink. This launches the portal.
  2. Use the RDA Authentication secure sign-in method to access the portal.
    Your organization may tailor this option - check with your portal administrator for details.
  3. Verify the information on the Client Profile tab is correct.
  4. Navigate to Invitation tab, and enter the client's name and an email address.
    Note the address cannot be a *@canada.ca address.
  5. Select appropriate user rights for the client:
    • Read: Allows a user to read messages, documents, and forms
    • Edit: Allows a user to create and send messages, upload documents, and create and submit forms.
    • Admin: Allows a user to create and send messages, upload documents, and create and submit forms. An Admin user may also invite new users to the client web site.
  6. Click Invite to send the invitation to the client.
    A message displays indicating success that includes the name, email address, and Shared Secret Code.

Note: 

  • For security reasons, the shared secret code must be communicated to the client using a different mechanism than the email address used for the invitation.
  • For example, send an SMS message, make a phone call, or use a secondary email address.
