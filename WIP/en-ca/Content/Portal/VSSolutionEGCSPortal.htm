EGCS.Portal Visual Studio Solution

The EGCS.Portal solution contains code, Master Page, web parts, web services, and other components which define how the portal looks, works and communicates with GCPM.

It does not contain code for connecting the GCCF environment (GCKey and SecureKey)—that code is found in the GC.GCCF solution.

Prerequisites 

To build the EGCS.Portal solution, the following is required:

  • Visual Studio 2017
  • .NET desktop development
  • ASP.NET and web development
  • Node.js development
  • Data storage and processing
  • Office / SharePoint development

  • Install the SharePoint Server 2016 Client Components SDK (sharepointclientcomponents_16-4351-1000_x64_en-us.msi). Download here: https://www.microsoft.com/en-us/download/details.aspx?id=51679

  • Connect projects in the EGCS.Portal solution to SharePoint. Generally speaking, when developing Visual Studio projects which have a SharePoint component, SharePoint must be installed on the same PC as Visual Studio. The EGCS.Portal project has been configured so that SharePoint is not required. Pick the approach which is best suited to your development environment.

  • If SharePoint is installed, connect the SharePoint projects to a SharePoint site:

    • Perform this procedure for each SharePoint project

    • EGCS.Portal.SharePoint.Library

    • EGCS.Portal.SharePoint.PortalMasterPage

    • EGCS.Portal.WebService.SharePoint

    • In Solution Explorer, select the a SharePoint project

    • Fill in the “Site URL” in the “Properties” pane. This must be a valid SharePoint site URL on the server. SharePoint will deploy solutions to this site when the “Deploy” option is selected.

  • If SharePoint is not installed, configure a dummy SharePoint connection:

    • Close Visual Studio

    • Copy a .user file into the folder of each SharePoint project. This file will configure a fake connection to a SharePoint server. You will not be able to deploy code to the server (as it doesn’t exist) but it will keep Visual Studio from complaining.

    • Copy files:

      EGCS.Portal.SharePoint.Library.csproj.user > EGCS.Portal\EGCS.Portal.SharePoint\EGCS.Portal.SharePoint.Library

      EGCS.Portal.SharePoint.PortalMasterPage.csproj.user > EGCS.Portal\EGCS.Portal.SharePoint\EGCS.Portal.SharePoint.MasterPage

      EGCS.Portal.WebService.SharePoint.csproj.user > EGCS.Portal\EGCS.Portal.WebService\EGCS.Portal.WebService.SharePoint

Build the EGCS.Portal solution

To build the EGCS.Portal solution:

  1. Retrieve the latest version from TFS.
  2. Clean previous build output:
    • Open a PowerShell windows as Administrator
    • CD to the directory for the EGCS.Portal solution
    • Run Clean-Solution.ps1
  3. Configure SharePoint deployment options.
    If SharePoint is not installed on the PC with Visual Studio, or if you do not wish to deploy recompiled packages into SharePoint, change the SharePoint deployment options. The EGCS.Portal project includes a deployment configuration which will build a SharePoint WSP package, but not actually deploy the WSP to SharePoint. A custom script will copy the WSP package to the _Build output folder when deployment is performed.

    Perform this procedure for each SharePoint project:

    EGCS.Portal.SharePoint.Library

    EGCS.Portal.SharePoint.PortalMasterPage

    EGCS.Portal.WebService.SharePoint

    • In Solution Explorer, select a SharePoint project.

    • Right-click the project and select “Properties”
    • Select the “SharePoint” property page
    • Set “Active Deployment Configuration” to “Build WSP, don’t deploy to SharePoint”.
  4. To build the EGCS.Portal solution:
    • In Visual Studio right-click the solution node (top of the Solutions Explorer tree)
    • Select “Deploy”
    • It will likely be necessary to also select “Deploy” on
      EGCS.Portal.SharePoint.PortalMasterPage
      EGCS.Portal.WebService.SharePoint

      A folder named _Build will hold all of the build output. This folder is created by steps in the “Build Events” section of each project.

  5. Build SPFx Apps

    • Repeat this procedure for each SPFx project:

      EGCS.Portal\EGCS.Portal.SharePoint.SPFX.Terms

      EGCS.Portal\EGCS.Portal.SharePoint.SPFX.FactorAuth

      EGCS.Portal\EGCS.Portal.SharePoint.SPFX.MessageCentre

      EGCS.Portal\EGCS.Portal.SharePoint.SPFX.PortalUser

      EGCS.Portal\EGCS.Portal.SharePoint.SPFX.Project

    • Open a Node.js command window as Administrator
    • CD to each SPFx folder

    • The first time any SPFx App is built, it is necessary to install npm packages for that App. This is not required when rebuilding an App on the same development PC.

      Run commands npm install
      npm install -g yo gulp

    • Run Build.bat
    • Output will be written to _Build\SPFx folder.
EGCS.Portal Project and Folders

Project

Project Type

Notes

!Readme!

Solution Folder

Documentation used by developers on the portal project for their personal development environments.

Certificates

Solution Folder

Scripts used to generate certificates and a CA for the Integration Portal.

Certificates generated for use with the Integration Portal.

DLLs

Solution Folder

Copies of GC.GCCF, SharePoint and SmartGuide DLLs needed to compile projects

EGCS.Portal.Service.DocumentManagement

DLL

A background service to synchronize documents between the portal and GCPM.

Also synchronizes Secure Messages between the portal and GCPM.

EGCS.Portal.Service.DocumentManagement.CommandLine

Command line program

Runs EGCS.Portal.Service.DocumentManagement as a command line executable.

Debugging information is written to the terminal.

Suitable to development testing, not for production use.

Requires a copy of gccf.config and egcs.config in the bin folder.

EGCS.Portal.Service.DocumentManagement.WindowsService

Windows Service

Runs EGCS.Portal.Service.DocumentManagement as a Windows Service.

Intended for production use.

EGCS.Portal.Service.SiteCreation

DLL

A background service to create Client, Project, and Personal Zones as required.

A maintenance job to periodically add new Project Zones to existing Client Zones.

A maintenance job to clean up failed Client and Project sites.

EGCS.Portal.Service.SiteCreation.CommandLine

Command line program

Runs EGCS.Portal.Service. SiteCreation as a command line executable.

Debugging information is written to the terminal.

Suitable to development testing, not for production use.

Requires a copy of gccf.config and egcs.config in the bin folder.

EGCS.Portal.Service.SiteCreation.WindowsService

Windows Service

Runs EGCS.Portal.Service. SiteCreation as a Windows Service.

Intended for production use.

EGCS.Portal.SharePoint.Library

SharePoint 2016 Farm Solution

Core logic for the EGCS.Portal SharePoint code.

Common library code to send email, send SMS messages, manage user roles in sites, process Invitations, save and submit forms.

Configuration for EGCS.Portal SharePoint components is defined in this project as .NET settings.

Common database access class and common logging facility for SharePoint components.

Contains auto-generated web service proxies for all RDA Web Service services.

Translation strings for SharePoint components are defined in .NET RESX files.

Reference implementation of a “Table of Contents” web part used in the Integration Portal Welcome Centre.

EGCS.Portal.SharePoint.PortalMasterPage

SharePoint 2016 Farm Solution

Portal master page.

Note that translation terms are found in EGCS.Portal.SharePoint.Library (with the exception of a few cases hard-coded into JavaScript).

EGCS.Portal.SharePoint.SPFX

SharePoint App

Not used

EGCS.Portal.SharePoint.SPFX.FactorAuth

SharePoint App

SPFx App for two-factor authentication.

EGCS.Portal.SharePoint.SPFX.MessageCentre

SharePoint App

SPFx App for secure messaging.

EGCS.Portal.SharePoint.SPFX.PortalUser

SharePoint App

SPFx App for User Profile, Invitations, and User Management.

EGCS.Portal.SharePoint.SPFX.Project

SharePoint App

SPFx App for Client and Project site web parts: Client Profile, Project Profile, Disbursement Summary, Budget Summary, Repayment Schedule, Drafts, and Application Status.

EGCS.Portal.SharePoint.SPFX.Terms

SharePoint App

SPFx App for Terms and Conditions.

EGCS.Portal.WebService.Data

DLL

Common class shared by various components for data exchange.

Defines various structured objects returned by web service calls.

Changing this class means that all classes which reference it should be recompiled. Web service proxies for the RDA Web Service may need to be regenerated.

EGCS.Portal.WebService.Portal

IIS Web Application

The Portal web service. Allows GCPM to query and update the portal.

WCF web service.

Certificate authenticated.

Accesses the SharePoint API and the GCCF database.

EGCS.Portal.WebService.RDA

IIS Web Application

The RDA web service. Allows the portal to query and update GCPM.

WCF web service.

Certificate authenticated.

Connects to CRM via the Microsoft CRM SDK.

Hosted at the RDA, or near GCPM.

EGCS.Portal.WebService.SharePoint

SharePoint 2016 Farm Solution

The SharePoint web service.

RESTful web service, hosted and secured by SharePoint.

Available in all zones under the URL _layouts/15/EgcsPortal.

Requires POST submission of parameters.

Returns JSON-formatted objects.

EGCS.Portal.WebService.WebAPI

IIS Web Application

Microsoft WebAPI application.

Called from GCPM sandbox plugins.

Connects to the Portal web service.

EGCS.Portal.SmartGuide

DLL

Common DLL for custom functions available in SmartGuide.

Exports of Smartlets.
