GC.GCCF Visual Studio Solution

The GC.GCCF solution deals with connectivity to the GCCF environment and user identity. In includes code to extend both SharePoint and ADFS to meet the requirements of the CATS2 specification.

Prerequisites:
  • Visual Studio 2017
    • .NET desktop development
    • ASP.NET and web development
    • Data storage and processing
    • Office/SharePoint development
  • A SharePoint installation is not required to build the solution

To build the GC.GCCF solution:

  1. Retrieve the latest version from TFS.
  2. Clean previous build output:
    • Open a PowerShell windows as Administrator.
    • CD to the directory for the GC.GCCF solution.
    • Run Clean-Solution.ps1.
  3. Build the solution:
    • In Visual Studio right-click the solution node (top of the Solutions Explorer tree)
    • Select Deploy.
    • A successful build will report:

      ========== Build: 10 succeeded, 0 failed, 0 up-to-date, 0 skipped ==========

      ========== Deploy: 2 succeeded, 0 failed, 0 skipped ==========

Note:

  • Occasionally Visual Studio may not build one of the SharePoint solutions, claiming that the “project was not built because it is a dependent project”. This appears to be a Visual Studio bug. Retry the build, it should succeed.
  • A folder named _Build will hold all of the build output. This folder is created by steps in the “Build Events” section of each project.
  • If code in GCCF Library or GCCF Connector projects has changed, copy the output DLLs to the EGCS.Portal project. This process can be automated by putting a file name _Refresh_EGCS_DLL.txt into the GC.GCCF solution folder.
  • Check “Build Events” of those two projects for an example of how to configure this.
GC.GCCF Project and Folders

Project

Project Type

Notes

GCCF ADFS Extensions

IIS Web Application
  • A web application which extends the services offered by ADFS to include SAML SOAP binding logout and back-channel logout.
  • Usually installed on the ADFS server.
  • The reverse proxy server makes this web application appear under in URL space of ADFS. Traffic for /adfs/ls/gccf/* is routed to this web application instead of being processed by ADFS.
  • Uses OpenAM community Edition to send and receive SAML SOAP messages.

GccfAttributeStore

DLL
  • An ADFS attribute store.
  • This is a custom DLL which is loaded by ADFS. It is invoked during user login to map the user’s GCCF Persistent Anonymous Identifier to their PortalUserId.
  • Stores data in the database table GCCF.dbo.GccfPai.

OpenAM Community Edition

DLL
  • An open source library for SAML, written in C#.
  • Has been extended and customized for the requirements of the Portal.
  • Based on the ForgeRock OpenAM Community Edition Fedlet

DLLs

Visual Studio Solution Folder

This contains SharePoint and ADFS DLLs required to build all components of the GC.GCCF solution.

GCCF Connector

SharePoint 2016 Farm Solution
  • Web pages to manage the chain of redirects during user authentication for GCCF to ensure that Common Domain Cookie processing occurs, that the ADFS session cookie is copied, and that ADFS is invoked with the correct arguments to log into GCKey or SecureKey.
  • Adds several pages to the server path /_layouts/15/gccf/. Most of these pages do not have a user interface. They perform server-side processing and then redirect the browser.
  • HTTP Module Language reads the portal’s language cookie and sets the SharePoint user interface language.
  • HTTP Module SLO monitors web requests for user sessions which need to be terminated because a back channel logout notification has been received.
  • HTTP Module UserRegistration monitors web requests to ensure that all users have accepted portal Terms and Conditions, have registered a profile, and have performed two-factor authentication. Any web request which does not meet these conditions is redirected to the appropriate web page to complete the task.
  • Library code to encrypt cookies.
  • Library code to identify the type of user (GCCF User vs. RDA User).
  • The DLL output from the project is referenced in the EGCS.Portal solution.

GCCF Web Pages

SharePoint 2016 Farm Solution
  • Reference implementations of simple pages to display log in, log out, and error messages.
  • Translation files (.NET RESX files) for the GC.GCCF components.
  • Reference implementation of the GCCF Chooser pages, which allows users to select between GCKey and SecureKey.
  • Reference implementations of the user re-enrolment page. This allows users to request recovery of their portal account when they have lost or forgotten their GCKey/SecureKey, or when they wish to transfer their account to a different GCKey/SecureKey.

Entrust 9.3

Solution Folder
  • Installed files for Entrust 9.3.
  • Entrust is used to perform CRL checking for ADFS on ICM certificates.
  • Includes Entrust installer configuration files for ICM CATE (pre-production) and production environments.

Microsoft ARR

Solution Folder
  • Installer files for Microsoft ARR.

Metadata Rewriter for ADFS

Windows Forms Application
  • Read ADFS metadata and produce a signed metadata package suitable for GCCF.
  • Another alternative is to use the PowerShell script Update-AdfsMetadata.ps1 to re-sign (and optionally change the DNS name of) an existing portal metadata package.

SAML Decrypt Utility

Windows Forms Application
  • Decrypt SAML responses from GCKey and SecureKey. These responses are typically captured from Fiddler, or a similar utility.
  • Requires the ICM Encryption certificate, with private key, to be on the local computer and accessible to the user running the utility.
  • Very old and ugly code with several hacks. Not a formal, supported part of the package. Use at your own risk.

Configuration EGCS-SESC

DLL project

The DLL from this project is not used. The Project is a container to hold configuration files.

  • Configuration files for installing the Portal.
  • Configuration settings for various GC.GCCF components.
  • Configuration settings for multiple Portals are kept under separate environment names.
  • Current metadata for GCKey and SecureKey, both pre-production and production.
  • The root 1CA-AC1 certificate for pre-production and production.
  • Microsoft ARR ruleset extracts.
  • The portal’s ICM signing certificate (public key only).
  • ADFS web theme to support the Portal.

GCCF Common Domain Cookie

IIS Web Application
  • Writes the GCCF Common Domain language cookie during login to communicate the user’s desired language to GCKey / SecureKey.
  • Reads the GCCF Common Domain language cookie after login to communicate the user’s desired language back to the portal (in case it has changed).
  • Published under the DNS name assigned to the portal in the fjgc-gccf.gc.ca domain.
  • Typically installed on the ADFS server.

GCCF Library

DLL
  • A library DLL which is used by most other components of GC.GCCF and EGCS.Portal.
  • Contains all .NET settings used in other GC.GCCF code.
  • Includes a third-party zip compression library and a third-party library for accessing Memcached servers.
  • Centralized logging methods
  • Centralized database access methods
  • The DLL output from the project is referenced in the EGCS.Portal solution.
