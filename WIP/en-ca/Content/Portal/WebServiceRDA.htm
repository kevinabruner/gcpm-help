RDA web service

The RDA web service is the communication channel from the portal to GCPM (or legacy data). The RDA web service is found in the Visual Studio project EGCS.Portal.WebService.RDA.

  • A certificate-authenticated, WCF web service.
  • Uses the CRM SDK to query and update GCPM.
  • Any communication from the portal to the RDA uses this web service.
  • The portal communicates with the RDA Web Service via HTTP (not HTTPS). Internally all messages are encrypted and signed, so communications are secure and authenticated.

The RDA Web Service is likely hosted “close” to the CRM instance for GCPM (e.g. in the same data centre, or virtual infrastructure, if possible). This web service may be extended to communicate with legacy data sources at the RDA.

Prerequisites

For the example below, assume the RDA web service is installed as http://web-service-rda.dept-dept.gc.ca

  • A build of the EGCS.Portal solution, which is assumed to be available in a directory _Build.
  • An EGCS.Portal configuration file. See EGCS.Portal configuration
  • Choose a server to host the RDA web service. This should be a server located “near” the GCPM CRM server. The server must be able to make network connections to the GCPM server. It must be an IIS web server. The server does not need to be dedicated to the RDA web service, overall load from the RDA web service should be light.
  • Choose a HOST name which will be used by the portal to access the RDA web service.

    HOST name for this example: web-service-rda.dept-dept.gc.ca

  • Choose a port for the RDA web service. Connections to the RDA web service use HTTP, however, all messages are encrypted so that connections are secure despite using HTTP. The default port is 80, but it may be desirable to install the RDA web service on a non-standard port. As the portal will be able to make connections to this port, using a non-standard portal prevents the Portal from accessing other web content which may be available on port 80.
  • Ensure that the portal can resolve the HOST name to the correct network address for the RDA web service. This can be accomplished by DNS, or by modifying the C:\Windows\System32\drivers\etc\hosts file on the portal SharePoint server(s).

    Example: Portal SharePoint servers must be able to resolve the name web-service-rda.dept-dept.gc.ca

  • Ensure that firewall rules are in place to permit the portal to make connections to the RDA web service. The portal must be able to make connections to http://web-service-rda.dept-dept.gc.ca:<port>/ through any intervening networks and firewalls.
  • Choose or create a service account for the RDA web service. For an on-premises installation of the web service, this should be a normal domain account. It is recommended that this account be a system administrator in GCPM CRM. However it would be possible to assign the account to a custom security role which selectively grants read / write permissions to all required entities.
  • The RDA web service requires a certificate to authenticate itself to callers. See .RDA web service and portal web service certificates.

    The RDA web service should have a certificate with a client and server profile, and a subject name which matches the HOST name chosen above.

    Example subject name: web-service-rda.dept-dept.gc.ca

    The certificate must include a private key.

    The certificate should be installed in the Local Computer, Personal certificate store on the server hosting the RDA web service.

    certutil.exe -importpfx web-service-rda.dept-dept.gc.ca.pfx

  • The EGCS.Portal configuration file should be updated with the thumbprint of the RDA Web service certificate. This setting will cause the portal expect the RDA web service certificate as the server certificate for connections to the RDA web service.
    <Token> -> RDAWebServiceClientThumbprint

  • The EGCS.Portal configuration file should be updated with the thumbprint of the RDA Web service certificate. This setting will cause the portal to expect the RDA web service certificate as the server certificate for connections to the RDA web service.

    <Token> -> RDAWebServiceClientThumbprint

  • The service account must have read access to the certificate private key.
    • Launch mmc.exe, add the “Certificates” snap-in for the local computer. Locate the certificate in the “Personal” store.
    • Right-click on the certificate and select All tasks-> Manage Private Keys…
    • Ensure the service account has “Read” access.
    • The CA certificate which issued the RDA web service certificate should be registered in Trusted Root Certification Authorities. This certificate should not have a private key.

Installation

  1. If it has not been created, provision an IIS web application to host the RDA web service.
  • Provision the web application at C:\inetpub\rda-portal
  • It should be use a HTTP binding (not HTTPS).
  • It should be bound to the port selected in the preresquisites.
  • It is suggested to configure the host name on the binding. The name must match the HOST name selected in prerequisites.

  • Assign an application pool with the desired service account to run the application.
  • Create a folder for the RDA web service.
    C:\inetpub\rda-portal\EGCS.Portal.WebService.RDA
  • Convert the RDA web service folder into an IIS web service.
    Assign an application pool with the desired service account to run the application.

  1. Install the portal web service
    .\_Build\Scripts\ Install-WebServiceRDA.ps1 –Config .\_Build\Config\<environment>.egcs.config
    This command will install web application files to path
    C:\inetpub\rda-portal\EGCS.Portal.WebService.RDA
  2. Edit web.config settings for the RDA web service.
    <serviceCertificate>     should be updated with the thumbprint of the certificate issued to the RDA web service in the prerequisites.
    Other settings: 
    SettingDescription
    DatabaseConnection

    A .NET connection string which for the logging database (installed in the next step)

    GCPMConnectionString

    Microsoft CRM SDK format connection string for the GCPM CRM server.

    RDAWebServiceClientThumbprint

    A list (one or more) of thumbprints of certificates authorized to call the RDA web service. By default, this should be a single certificate issued to the portal. In a development scenario, there may be additional thumbprints for developer certificates to call the RDA web service. Thumbprint of the root CA certificate which issued the RDA web service certificate, and the RDA web service client certificate(s).

    RDAWebServiceRootCAThumbprint

    Thumbprint of the root CA certificate which issued the RDA web service certificate, and the RDA web service client certificate(s).

  3. Create a logging database for the RDA web service.
    • Edit the file _Build\Database\Portal_RDA_WebService Database.sql
    • Edit the FILENAME for the mdf and ldf file to place the files where desired.
    • Adjust size and growth parameters for the database if required.
    • Execute _Build\Database\Portal_RDA_WebService Database.sql to create the database
    • Grant the service account for the RDA web service access to SQL Server, in the “public” role
    • Edit the file _Build\Database\Portal_RDA_WebService Database.security.sql
    • Update the service account name in all places (replace GSCS\portal.int)
    • Execute _Build\Database\Portal_RDA_WebService Database.security.sql
