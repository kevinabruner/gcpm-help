SharePoint web service

The SharePoint Web Service is the collective name used for a series web services used to support the portal. It is implemented by the project EGCS.Portal.WebService.SharePoint. These web services are intended to support client-side web parts and JavaScript, such as the SPFx web parts.

  • These web services are hosted directly by SharePoint.
  • SharePoint web services are RESTful.
  • Web service methods are called via HTTP POST.
  • Parameters to web service methods are JSON-encoded.
  • Return results from web service methods are JSON-encoded.
  • Web services are hosted in a relative path /<site>/_layouts/15/EgcsPortal/<web service page>.
  • The method name for the web service is appended to the web service URL.E.g.: /client/AC000922/_layouts/15/EgcsPortal/Account.svc/ApplicationStatusGet

This example calls the web service Account.svc method ApplicationStatusGet in Client zone AC000922.

  • Web services are called with the security context of the SharePoint user. The web service executes using that user’s identity.
