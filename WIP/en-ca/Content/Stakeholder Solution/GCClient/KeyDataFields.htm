Key data fields for GC Clients

GC Client forms may be re-configured at your organization. These are the fields / grids designed for the GC Client:

GC Client status

  • Client status: Indicates is the client is ready for funding. A GC Client status must be Ready for funding in order to recommend projects and disbursements. Status is modified using the Client Transition > Set client ready for funding transition.
  • Hold: Indicates the GC client hold status. Status is modified using the Client Transitions options.
    If status = yes, it is a warning to put certain operations on hold until a decision is made regarding recovery or default status.
  • Recovery state: Indicates the GC Client recovery status. Status is modified using the Client Transition options.
    If status = yes, all projects under the client are subject to recovery and no further disbursement are done until the status = No.
  • Default state: Indicates the GC Client default status. Status is modified using the Client Transition options.
    If status = yes, all projects under the client are set to default and no disbursements are made until the issue is resolved and the status = No.
  • GC Client portal: Indicates if the GC Client has a portal account.
    If status = yes, a link to the portal site is provided. When the status is first set to yes, the Client Zone is created on portal for the client, and a link is inserted into the GC Client record. An officer reviewing the GC Client record can view documents submitted by the client. Note, these documents can only be edited from the portal.

GC Client portfolio

This grid and these fields provide a snapshot of the financial interactions your organization has had with this client: 

  • GC Client primary contact: Identifies the key contact for the organization. Your organization may choose to have a business rule that this should identify a person with a particular role, for example, the person with financial signing authority, or the project manager.

Project rollup

These fields are updated dynamically and display current information, in read-only format.

  • Funding history: A grid that displays current and previous projects and applications.
  • Client exposure: A calculated field. Formula = Sum of the total principal outstanding owing by the client + the disbursed (net) amounts from conditionally repayable contributions.
  • Number of projects: A calculated field. The total number of projects submitted by the client.
  • Total project costs: A calculated field. This field is also displayed in the funding history grid.
  • Total assistance: A calculated field. The sum of all committed assistance by the organization for all projects.
  • Net disbursed: A calculated field. The sum of all disbursed amounts for all projects.
  • Principal repaid: A calculated field. The sum of repayments for all projects.
  • Principal outstanding: A calculated field. The sum of principal outstanding for all projects. The principal outstanding is usually the disbursed amount of a repayable project less the principal repaid. For a conditionally repayable project, the principal outstanding would be the expended amounts determined as being repayable less any amounts repaid.
  • Principal in arrears: The sum of amounts billed to the client for all project less the amount received by the client.
  • O/S advances: The value of outstanding advances to the client.
  • Interest due: The amount of interest the client currently owes.
  • Principal written off: The sum of amounts owed by the client for all projects that have been written off.
  • Interest written off: The interest that has been written off.

Financial system data

  • Vendor customer: Identifies the name of the financial system Vendor Customer associated to this GC Client.
