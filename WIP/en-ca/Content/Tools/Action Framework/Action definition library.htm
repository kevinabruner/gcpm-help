Action definition library

These are the action definitions included with GCPM. They can be used to validate and / or automate various aspects of your transitions.

Validations

Name (EN)

Description
Compare co-obligants with project client Checks the co-obligants, to ensure the GC Client is not part of the co-obligants.
Compare field with field

Compare a field's value with another field's value.
Compare fields with value

Compares a field value to another value or a null value (used with isNull and isNotNull functions). The function uses these parameters:

Entity: Used to filter the field list for the second parameter but not used in the compare function.

Fields: List of fields we want to compare to the value.

Condition: Logical operation we want to use (greater, lower, etc).

Value: The value we want to compare. This parameter is optional with the logical operation isNull and isNotNull.
Compare two option value Compares a field a field with a two option set value.
Is a GC Client Checks if the GC Client exists or not.
Validate amendment checklist Validates at leat one item in the amendment check list was set to Yes.
Validate disbursement fiscal year Checks that the fiscal year is valid for a disbursement.
Validate option set value Checks if the value of an option set value is the same than the value specified by the user
Validate project closure checklist Checks that all the items of the closure checklist has been set to Yes
VENDORS_IsValid Checks the vendor contains all it's main data
Verify vendor customer is complete Checks that the FI Vendor Customer is completed

 

 

Actions
Name (EN) Description

Call custom action on current record

Allows to call a custom action from CRM.

Clone record and related

Allows to call a clone record configuration.

Clone record only

Allows to call a clone record configuration.

Copy amendment to project

Copy all the amended elements from the current amendment to the project.

Create entity record from current record

Create a record based on another existing record.

Create entity record from text values

Creates a record based on values passed by the user.

Create secure message

Allows to create a secure message for the portal.

Default: spread decisions

Allows to spread the information across project and co-obligants when a project or client becomes in Default.

Download document

Generates a document that's within a configuration of the document generator. Once the document is generated, a note is created with the document as attachment in the project. The function is based on 3 parameters and returns the generated document:

Configuration: Document generator configuration.

Template: The template that we will use to generate the document.

Record: Record data used to generate the documentReturn parameter:

Document: Generated document.

Enable lock plugin override

Allows to override the plugin when it's activated

Send email

This function is used to send emails. The function takes 2 parameters:

SendTo: The field to whom send the email. The lookup is usually linked to a user that has an email.

SendFrom: The field from whom send the email. The lookup is usually linked to a user that has an email.

Set amendment transition

Set the transition for the amendment.

Set boolean options

Set the value of a two option field.

Set date field value

Set a date of a field.

Set field value

Set the value of a field.

Set inactive record

Set the status of the record to inactive.

Trigger action

Trigger a CRM Custom action with it's logical name.

Trigger workflow

Trigger a CRM Workflow based on the workflow ID.

Update option set value

Sets the value of an option set based on 2 parameters:-Field: name of the option set field-Value: The value we want to set on the option set.
