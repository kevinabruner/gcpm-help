GCPM sample business process flows

Several business process flows (BPFs) are included with GCPM. These can be copied and modified to suit the needs of your organization.

Each program can integrate these business process flows.

These sample programs illustrate features your organization may use. Two programs are included:

  • GCPM Core Program
  • Women Entrepreneurship Strategy
    • Designed with a more complex program hierarchy and intakes.
    • The program hierarchy permits reporting on projects at different levels of granularity, when a program terms and conditions has sub-programs, and even multiple streams. Each sub-program / stream can have multiple intakes that may accommodate unique applications, or timed intakes.

Disbursement Review BPF

The disbursement BPF collects all outstanding claims, advances and deferred items in a disbursement review. A review may result in approval and payment, deferrals or rejected claims.

Disbursement Review BPF
Feature Description
Transition template name Disbursement transition template
Chevrons / stages in the BPF Disbursement review / S. 34 approval / S. 33 approval / payment
Validations / actions in the BPF

Validates if:

  • the project is in a disbursable state,
  • the project is not on hold,
  • Client is not in recovery
  • Total disbursed would not exceed 100% of approved assistance

Once payment is issued, actions include: 

  • Set disbursement internal status to “Payment Approved”
  • Trigger plugin code that: Applies any advance settlement, applies any hold back released, creates deferred item copies
  • Set status to Completed or Cancelled
Possible decisions S. 34 approval, S. 33 approval

GCPM Core Program BPF

The program BPF configures the life cycle of the projects for a program.

Feature Description

Transition template name

GCPM Core Program transition template

Chevrons / stages in the BPF

Application / Eligibility and prioritization / Assessment / Contribution agreement / Administration / Control period / Default or recovery / Completion / Close
Validations / actions in the BPF
  • Proceed to initial review
  • Validate program and Profit / Non-profit
  • Application is eligible - proceed to advanced assessment
  • Create risk assessment
  • Officer assessment completed - prepare recommendation
  • Submit recommendation action set
  • Recommend application for approval
  • Authorize project action set: Project amendment required prior to agreement execution / Offer accepted / Change amendable to yes / Change requested to contribution agreement - prepare amendment to offer/ Set amendable CA / Send project to repayment / Validate end of disbursement
Possible decisions

Client requests to withdraw application
