Configure clone records

The values set in configuration records are based on the relationship structure of the entities to be cloned. The system includes cloning for the following relationship structures that can be used as reference for other clone configurations.

  1. Children with multiple parents
  2. Template entities

To configure a clone record: 

  1. In Advanced Find, search for CFG Clone and create a new record.
  2. Fill in the fields using onscreen help and the notes below: 

    • Name: The name of this CFG Clone record.
    • Default configuration: Indicates if this is the default configuration for the source entity.
      Note: A custom Save As can configured button uses the default configuration. (Requires custom Save As button to be configured, if desired)
    • Ignore: Indicate if this configuration ignores this record and copies only the related records.
    • Description: provide a description for the clone configuration record. The description is useful to differentiate between different clone configurations for the same source entity.
    • Target entity:
      • Same as source: Indicates if the source and target entity are the same (true) or different. This must be false when copying records.
      • Parent regarding field: (for copying only) The lookup field in the target entity that points to the source entity record. This field is needed if you are triggering the copy process from the target entity instead of the source entity.
  3. Field map Complete these fields if you are cloning or copying records.
    Tip: When copying data between two sets of records of the same entity type, use this method instead of the “Cloning Records” method if the source and target fields are different.
    • From
      • Source entity: Provide the logical name of the entity to clone.
      • Source hierarchy field: Choose the self-referential relationship in the source entity, if any. This is needed to maintain the record hierarchy.
      • Source fields: The list of fields you want to copy, delimited by commas.
        Note: If you are copying fields to a different entity type, the target field must be the same data type or it will fail. The source and target fields are mapped in order, so the two lists must have the same number of fields listed.
    • To
      • Target entity: The logical name of the entity you want to create.
      • Target hierarchy field: The lookup field of the self-referential relationship in the target entity, if any. This is needed to maintain the record hierarchy.
      • Target fields: The list of fields you want to copy, delimited by commas.
  4. Clone related configurations.
    See Configure clone related records
