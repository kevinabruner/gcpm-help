Cloning records versus copying records

Cloning records

Choose to clone records when you want to create a copy of a record as the same entity type. For this type of configuration, you only need to fill in the details for the source entity, because cloning implies that both source and target are the same. Filling in both the source and target will make no difference.

Field

Description

Source entity

The logical name of the entity you want to clone.

Source hierarchy field

The lookup field of the self-referential relationship in the source entity, if any. This is needed to maintain the record hierarchy.

Source fields

The list of fields you want to copy, delimited by new lines.

Copying Records

Choose to copy records when you want to create a copy of a record as a different entity type. For this type of configuration, you need to fill in both the source entity and target entity details, because the entity and field names are different between the two.

Tip: When copying data between two sets of records of the same entity type, you should use this method instead of “Cloning Records” if the source and target fields are different.

Field

Description

Parent regarding field

The lookup field in the target entity that points to the source entity record. This field is needed if you are triggering the copy process from the target entity instead of the source entity.

Target entity

The logical name of the entity you want to create.

Target hierarchy field

The lookup field of the self-referential relationship in the target entity, if any. This is needed to maintain the record hierarchy.

Target fields

The list of fields you want to copy, delimited by new lines.

Copying fields

To specify which fields to copy, list the fields in the Source Fields field. If you are copying fields to a different entity type, the target field must have the same data type or it will fail. The source and target fields are mapped in order, so the two lists must have the same number of fields listed.

Related fields

You can copy data not only from the source entity record but also from its parent records, however many levels away as you wish.

To reference a related field in the Source Fields field, use this format:

lookupToParent1.lookupToParent2...lookupToParentN.fieldName

Any records that are retrieved are cached to speed up the process, especially when cloning or copying many children records. Consider impacts on performance when referencing fields many levels away.

Special fields

Special fields copy data from fields in related records or to associate the target entity record to the source entity record. These fields can only be used in the “Source Fields” field. Related fields can be referenced through these special fields the same way.

Field

Description

this

Refers to the source entity record that is being cloned or copied.

This is useful when the target entity has a lookup field that points to the source entity.

thisparent

Refers to the parent record of the source entity record.

This is useful when you want to copy a parent field without having to retrieve it from the database.

that

Refers to the target entity record that is being created.

This is useful when you want to copy data from a parent record of the target entity.

thatparent

Refers to the parent record of the target entity record.

This is useful when you want to copy a parent field without having to retrieve it from the database.
