Cloning or copying related records

Related records

You can include related records as part of the cloning or copying process by simply creating a CFG Clone Related record for each type of related record you want to clone or copy.

When specifying the source related child records, you are not limited to just direct descendants but you can also clone or copy the child records of a parent, grandparent, great grandparent, etc.

Field

Description

Name

The name of the CFG Clone Related record.

Root configuration

The root CFG Clone configuration record. This flattens the hierarchy of CFG Clone Related records when viewed from the root CFG Clone configuration record.

This is also to prevent the entire record hierarchy of the related CFG Clone record from being cloned or copied, because you must explicitly specify the record hierarchy from the root configuration.

Relationship

The type of relationship between parent and related entity. If the related entity is a lookup record, then it is a N:1 relationship.

Parent-related configuration

The parent CFG Clone Related record. This is required when cloning or copying grandchildren and later records, i.e. two or more levels away.

Source regarding field

The lookup field(s) from the Target Configuration’s source entity to the source entity in either the Root Configuration or Source Related Configuration. You can go many levels away instead of just the direct parent or direct descendants.

Target configuration

The CFG Clone configuration record to use. The source and target entity in this record are the related entities of the source and target entity in either Root Configuration or Source Related Configuration.

Target regarding field

The lookup field from the Target Configuration’s target entity to the target entity in either the Root Configuration or Source Related Configuration. Only one lookup field is allowed.

Enable condition

Whether to filter the list of child records retrieved or not.

Condition

The FetchXML query that contains the filter to apply.

Parent record [N:1]

A parent record is a record referenced by a lookup field, e.g. an account record has a lookup field that points to the primary contact record. Normally, you would not clone the parent record when cloning or copying an account record; however, at times you may wish to create a clone of the parent record and its related records and associate the lookup field to the newly cloned record instead of the original parent record.

When cloning a parent record, the related records are configured in the target CFG Clone record and not in the root CFG Clone record, because the target CFG Clone record is treated as a root configuration.

Child records [1:N]

Child records is a list of records that are associated to a record, e.g. an account can have many contact records associated to it. When cloning or copying child records, the hierarchy of the related records is configured in the root CFG Clone configuration record.
