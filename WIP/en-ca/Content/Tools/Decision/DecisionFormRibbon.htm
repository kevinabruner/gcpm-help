Add transition buttons to form ribbons

To add a decision and status buttons to the form ribbon:

    1. Prepare to work with Ribbon Workbench. Create a solution with only [ENTITY] form.
    2. Open the Ribbon Workbench in XRM Toolbox.
    3. Add a flyout button.
      1. In the section set for Form, drag and drop a Flyout from the Toolbox on the left.
      2. Insert a Menu Section
      3. Inside the menu section, using drag and drop, insert another Flyout button and two Buttons. Fill in the names by clicking on each button and Flyout button.
    4. Add commands for load, loadTransitions, makeTransition, transitionHisotry and decisionHistory.
      1. Change the name of the entity (underlined in red in the screen shot below) for the name of the entity you are adding a transition to.
      2. To add a command use the + sign.
      3. Create a load command. Change the command Id.

      4. Add a Display Rule to hide the transition button when the form is in create mode.
        Click Add step and add a FormStateRule with the state Existing.

      5. Add an Enable Rule to the command as well. This will load the scripts on the loading of the page.
        Change the id as well as the name of the entity. Click Add Step and choose Custom Rule.
        FunctionName is EGCS.Main.Command.loadFormCommand, Library is $webresource:egcs_/shared/js/main.js.

      6. Add 2 Parameters, one of type CRM Parameter with the value PrimaryControl and one of type String with the value egcs_/entities/[ENTITY] /ribbons/form.

    5. Create a loadTransitions command.

      1. Change the command Id to reflect the entity.
      2. Click Add Action to create a custom Javascript action. Library is $webresource:egcs_/shared/js/main.js and Function Name is EGCS.Main.Command.[ENTITY].onLoadTransitionsMenu.
      3. Click Add Parameter to add a Crm Parameter with the value CommandProperties.
    6. Create a makeTransitioncommand.

      1. Change the command Id to reflect the entity.

      2. Click Add Action to create a custom Javascript action. Library is $webresource:egcs_/shared/js/main.js and Function Name is EGCS.Main.Command.[ENTITY].onClickMakeTransition.
      3. Click Add Parameter to add a Crm Parameter with the value CommandProperties.
    7. Create a transitionHistory command.

      1. Change the command Id to reflect the entity.

      2. Click Add Action to create a custom Javascript action. Library is $webresource:egcs_/shared/js/main.js and Function Name is EGCS.Main.Command.[ENTITY].onClickTransitionHistory.

    8. Create a decisionHistory command.

      1. Change the command Id to reflect the entity.

      2. Click Add Action to create a custom Javascript action. Library is $webresource:egcs_/shared/js/main.js and Function Name is EGCS.Main.Command.[ENTITY].onClickDecisionHistory.
