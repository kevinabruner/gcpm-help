Configure data sources for a snapshot

Data sources for a snapshot identify the data that will populate the document template. You can choose an entity and all of its fields, or only selected fields, to be displayed in a document template.

To configure a data source: 

  1. Using Advanced Find, search for CFG Data Source and create a new record.
    The image below uses the amendment clauses record as an example.
  2. Fill in the fields using onscreen help and the notes below: 
    • Parent entity alias: When chaining queries, this field is used to specify which entity in the parent data source’s Query you are chaining from.
    • Parent data source: To overcome CRM’s limitation of a maximum of 10 entities per query, this field allows you to chain queries one after another so that you can pull in more data.
    • Name: The name of the data source. This name will be used as the “folder” name for the data specified.
    • Description: Provide description so people know what this is for
    • All entity attributes: 
      • If set to true, all attributes in the first (root) are available for use in the document template.
      • If set to false, only attributes specified in the query will be available.
    • All related entity attributes: 
      • If set to true, all attributes in all entities related to the root entity are available for use in the document template.
      • If set to false, only attributes specified in the Query will be available.
    • Starting entity alias: Often times when creating multiple data sources, you will pull in similar data in other data sources, so this field allows you to specify which entity’s data you want using the entity’s alias in the FetchXML Query, including entities related to that entity.
    • Query: Enter a FetchXML query to define which data to retrieve.
      • Tip: Use Advanced Find to create a query and use the FetchXML it generates for the query.
      • Note: A maximum of 10 entities can be referenced within a single query.
      • Conditions can be built into the Fetch XML - e.g. choose a field, equals (choose a value or multiple values). Save the query (for ensure consistent re-use in the future)
      • In the image below, the query is constructed to retrieve a Project as well as GC Client, account, contact, salutation, and program information. A total of seven entities worth of data are being retrieved by this query. You can also add specific attributes, conditions and filters to narrow the data.

    • Name fields: Certain entities have unique ids for its records, so this field allows you to use those unique id’s as the node name in the XML data, allowing you to reference specify records in a list of records. This is useful for using data as triggers, or for conditional situations.
      For example, if X = given value, show the clause, otherwise hide the clause.
    • Hierarchical relationships: Enter the logical name of the relationship.
      Entities with self-referential hierarchical relationships (e.g. clauses have levels 1, 2, 3, 4 to provide sub-clauses).
  3. Save and close the record.
