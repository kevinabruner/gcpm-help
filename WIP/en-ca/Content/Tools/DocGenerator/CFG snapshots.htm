Configure a Document

A snapshot is the data from a set of fields, at a point in time, that can be inserted into a document or report. For example, a contribution agreement snapshot captures all of the clauses identified during project assessment.

To create a snapshot configuration record:

  1. In advanced find, search for CFG Document
  2. Fill in the fields using the onscreen help and notes below as a guide: 
    • Target entity: Choose the entity that provides the data fields for this snapshot.
    • Save as PDF: Select this to convert this document to a PDF file.

    Settings: Identify the formatting settings for the document (decimals, date, time, currency formats). If left blank, default CRM formats will be used in the contribution agreement. The standards apply to the entire document.
    Bilingual consideration: Configure English and French letters separately. Will permit use of different settings in different letters.

    • File name parameter: Specify parameters to use to create the file name. Uses same approach as the calculated field plugin.
      For example: egcs_name, egcs_sgclientid, createdon
    • File name: Configure the format of the file name, using the parameters specified in the field above. 
      For example: {0} - {1} - {2:yyyyy} generates a file with the name Client Name - Client ID - Year
  3. Data Sources: Add data source records to the grid. See Configure data sources for a snapshot
  4. Templates: Choose a template for the snapshot. These templates are configured with the CFG Snapshot Document Template entity.
  5. Save and close the record.

Note for developers: 

  • The settings use DotNet conventions*.
  • Anything you can do with Fetch XML can be done with a snapshot.
Data Sources

Data sources identify the source of the data for this snapshot.

  • Data sources can be re-used.
  • More than one data source can be added to a snapshot
  • Once a data source is added, you can download a template (using the TEMPLATE / Download) option in the menu, and configure as an XML file.
  • When data sources change, templates can be updated by using the Update Template button.
