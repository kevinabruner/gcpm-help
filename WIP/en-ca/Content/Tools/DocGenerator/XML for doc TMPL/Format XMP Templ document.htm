Formatting an XML template document

There are various XML tags available to format the document template.

  • To use an XML tag, first insert a “Plain Text Control” and type out the XML tag inside the control.
  • Make sure “Design Mode” is active in the in the “Developer” tab.

  • When using XML tags, double quotes must be the unformatted double quotes. The easiest way to insert an unformatted double quote is to type the double quote and hit Ctrl+Z to undo the formatting on the double quote.

References

References / Inserts a field in the document. This does the same thing as inserting a field from the XML Mapping pane, but the latter is the preferred method.

Syntax

<Content Select="XPath" Optional="true/false" />
Attributes

Select: 

    • This is the path to the data that is to be inserted / referenced.

    • If set to true the referenced field must exist and not be null, otherwise an error will be displayed; if set to false, nothing will be displayed if the referenced field does not exist or is a null value.

    •  

    Optional

    **If you see this error, the field doesn’t exist, the value is null, or the XPath is wrong.

    Example

    <Content Select="./Data/egcs_fc_profile/attributes/egcs_fcprofileid" Optional="true" />

    Conditional – EndConditional

    Surrounds a section of the document with a condition to show or hide the surrounded section based on the condition.

    Syntax

    <Conditional Select="XPath" Match="truth value" NotMatch="false value" />

    Text that should be shown or hidden based on the condition.

    <EndConditional/>
    Attributes

    Select: This is the path to the data that is to be inserted / referenced

    Match: what should the value be for the condition to be true (also known as: equal to)

    NotMatch: What should the value not be for the condition to be true (also known as not equal to)

    Examples

    <Conditional Select="./Data/egcs_fc_profile/attributes/egcs_ceaa/value" Match="701360002"/>

    Section that will be shown if the condition is met,

    <EndConditional/>

    Repeat – EndRepeat

    Repeats the content within the tags by looping through the available data. This works the same as inserting repeating content through the XML Mapping pane, however these tags work better for non-tabular data.
