Configure document locations

A CFG Document Location record represents a pre-defined location or a single folder in the external DMS. Document folder names can be dynamically defined based on data from the CRM entities.

This entity has a self-referential relationship, so that the folder structure can be created hierarchically. You can view the entire folder structure using CRM’s hierarchy view. You may have multiple folder hierarchies depending on if you have multiple root document locations.

To configure a CFG Document Location record: 

  1. Navigate to Configuration > Document Manager > CFG Document Location and create a new record.
    Alternatively, use the grid in a CFG Document Entity record to create document locations.

  2. Fill in the fields. Use the onscreen help and the notes below for guidance.
    • Repository: The document management system (DMS) that stores the documents.
    • Portal documents: If set to yes, portal documents will be stored at this location.
    • Portal submissions: If set to yes, portal submission documents will be stored at this location.
    • Document entity: Displays the document entity associated with this location
    • Parent location: The parent folder of this document location. (If filled in, this location will be a subfolder of the parent location.)
    • Regarding: Type the parent entity name.
    • Location parameters: If the folder requires a dynamically generated name, list the CRM fields (from the GCPM Document entity) used in the formula set in the Location field.
    • Max. length: Set the maximum folder name length allowed. Additional characters will be truncated.
    • Predefined location: Indicate whether this record is a predefined location in the external DMS or not.
      describe
      If yes, the plugin will create the corresponding Document Location record in GCPM.
    • Location: The name of the document location to create. This name can be dynamically generated based on a formula that uses CRM fields. Use placeholders in the format {0}, {1}, ...{n} to refer to the fields specified in the location parameters. Parameters can be used multiple times.
  3. Save the record. A Subfolders field is now visible.
  4. Create subfolders for this location, if desired.
  5. Continue creating these records to reflect the folder hierarchy required for your organization.

Tips: 

  • Create the root folder for an entity using the CFG Document Entity. Remaining folders can be created (as subfolders) using the CFG Document Location. The lookups will be auto-populated).
  • Use the Hierarchy view in a CFG Document Location record to view a graphic representation of the folder hierarchy.
