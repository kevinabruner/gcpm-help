Document manager plugin API

The document manager plugin provides a set of API functions used to perform certain actions programmatically.

Parameters with asterisks (*) are mandatory and cannot be null. For optional parameters of primitive data types, the default value is used if no value is provided, e.g. Booleans default to false and int to 0.

Public API

Retrieve folder

Retrieves a document location and its subfolders and documents recursively up to the specified depth. This does not actually retrieve the document itself, just the data about the record.

Authenticate

Used only by the SharePoint connector, the SharePoint authentication mechanism has been abstracted and encapsulated into its own custom action so that it can be easily overridden by a local implementation.

If you only override the authentication mechanism, the local implementation must minimally return the FedAuth cookie, Form Digest, and expiration date, as required by SharePoint’s REST API.

Internal API

The document manager plugin has a generic set of API messages and data type definitions that are used to make calls to the DMS through the DMS-specific connector. Because the connectors all receive the same requests and data, it is possible to override or add more connectors to support other external DMS, such as OneDrive, Dropbox, etc.
