Use Dynamics CRM duplicate detection rules

Review the duplicate detection rules that are included in GCPM. These include the three types of out-of-the-box settings described below, as well as the GCPM duplicate detection plugin.

  1. CRM counts with preventive duplicate detection rules. Navigate to Settings > System > Data Management > Duplicate Detection Rules to view these rules.

    • Duplicate Detection Rules are only applied when they are published
    • When enabled, users creating or editing records get a warning when a duplicate record is detected.
    • Duplicates are not automatically resolved and the warning is not enforced.
    • Note criteria in a duplicate detection rule is applied as an AND statement.
    • There is a limit of five rules per entity and a maximum of 450 characters that can be verified per rule.
    • When a warning of a duplicate appears, user can take steps to merge the records and remove the duplicates
  2. Entity-level restrictions can restrict users from creating duplicate records.

    • To create a new Key, expand the entity node in a solution and select Key.

    • Select the field that is going to be checked for duplicates and enter a Display Name.
    • When a duplication is detected, user gets an error message and record cannot be created or edited.
  3. Duplicate detection jobs. The Check for Duplicates wizard helps user to set up a bulk job that finds and cleans up duplicate records.

To create a new duplicate detection rule, see: Set up duplicate detection rules*. 

Tips for using duplicate detection rules
  • When testing GCPM configuration, you may choose to Unpublish duplicate detection rules.
    For example, the CRA business number has complicated validation rules, and users must enter real business numbers.
    • To get around this limitation during testing, the number 000 000 000 can be used as it meets the validation rule.
    • This work-around can only be used once if the duplicate detection rule for business number is active.
