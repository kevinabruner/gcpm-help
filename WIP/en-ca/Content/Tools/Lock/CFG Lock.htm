Configure lock records

To configure a lock record for an entity: 

  1. Using Advanced Find, search for CFG Lock. Create a new record.

  2. Fill in these fields, as required:

    • Parent lock configuration: Select the CFG lock record of the parent entity, if applicable.
      Note: Only use this field if this is a child record with a parent record (e.g. a claim item is the child of a claim).
    • Parent field name: Type the logical name of the lookup field on this entity that references the entity specified in the selected parent lock configuration.
    • Entity name: Select an entity name from the drop-down list. This is the entity that will be locked.
      The selected entity will be registered in the plugin.
    • Locking field: The logical name of the field that, when it contains a given value, will do the locking.
      The locking field must be of one of these field types: 
      • Two options (Yes / No): The plugin assumes:
        Yes (checked) = locked,
        No (unchecked) = unlocked.
      • Lookup or Option set: You must create a CFG Lock Field record for each value of the lookup or option set. Each value in the lookup or option set that is a locking value. Each value in the lookup or option set can be configured to lock different sets of fields.
      • Once configured, these appear in the Lock Field configurations tab.
        For example, a disbursement review that has a status payment issued can lock different fields than a review that has a status rejected.
        Note: On children and grandchildren entities, the locking field must be of the type Two Options, because the plugin will automatically cascade the lock to all children.
    • Lock field selector: Select all fields to be locked - except the locking field (that triggers the lock). Selected fields will be registered to the plugin.
      Note: if new fields are added to the entity associated with this config those fields will not be automatically selected. By default those fields will be editable. If you wish to lock them, simply open this config and check the checkbox for any new fields. The lock field selector displays when the locking field's type is Two Options.

  3. Save the record. These additional tabs may appear:

    • The  Lock Related Configurations grid displays related child CFG Lock records.
    • The Lock Fields Configurations tab displays when the locking field's type is lookup or option set.
    • The Metadata tab displays a quick view of the values set for this configuration. It provides an easier way to verify the data than using the field selector.
  4. Save and close the record.

Next steps: Create child and grandchild configuration records, if the lock impacts these records.

Best practice 1: 

When configuring the fields to be locked:

  • Do not include the actual Lock Field in the Locked Field List of the root entity. This ensures the field can be edited and the record unlocked.
  • Include the lock field in the locked field list of the related children and grandchildren’s CFG Lock records. This ensures the lock feature is controlled only by the root entity.
Best practice 2: 

GCPM includes many lock records that use a Two Option field. These records have a field egcs_locked (with display name = Locked, and field type = two options: yes / no). Using a consistent naming convention makes it easier to populate the locking field.
