Configure rollup records

CFG Rollup calculates specified calculations (or rollups) in fields when needed. A rollup record uses the value of a specified field in children records and calculates a result according to the aggregation type (sum, count, max, min, etc.) and places that result in a parent’s field (rollup field).

To configure a rollup record: 

  1. Navigate to Configuration > Calculation Tools > CFG Rollups and create a new rollup record.

  2. Use the onscreen help and the notes below to complete these fields.
    • Order: Indicates the order in which a rollup needs to execute when more than one is configured in an entity. Special consideration must be taken to set this field when calculated and rollup fields are included in the aggregation because the order calculations are done will affect the result.
    • Force synchronous: Plugin works in asynchronous mode by default, but synchronous mode can be enforced if field value is YES. Forces the plugin to be executed in a synchronous way so it can be executed immediately.
    • Rollup entity: Logical name of the parent entity that contains the rollup field.
    • Rollup field: Logical name of the rollup field in the parent entity.
    • Aggregation type: The type of rollup aggregation.
    • Aggregate entity: Logical name of the child entity to aggregate.
    • Aggregate field: Logical name of the field in the child entity that is aggregated. Use primary key field logical name (e.g.egcs_name) for aggregation type COUNT.
    • Regarding field: Enter the logical name of the lookup field to the parent entity.
    • Description: Provide a description for this configuration record.
    • Is Money: Check if rollup field data type is Currency.
  3. To use a condition, enter parameters and the condition query.
    See notes, below.
    • Enable condition: Check the box to enable a condition.
    • Condition parameters: Specify a comma-delimited list of attributes whose values are to be used in the condition query. See note for parameter syntax, below.
    • Condition: The condition that each record must satisfy to be included in the rollup. The value of this field is a FetchXML query string. You can use CRM's Advanced Find to build your FetchXML query string. For more complex queries, you can use a third-party tool called FetchXML Builder*.
  4. Metadata fields are auto-populated and used internally to pre-process expensive actions in the plugin to optimize performance..
    • Metadata: Calculation metadata.
    • Related entities: A list of comma delimited parent entities referenced in this calculation.
  5. Save and close the record.
Parameter syntax

The parameter syntax is as follows:

{lookup.attribute:format;default value }

Syntax considerations:

  • The lookup name is only needed when referencing parent attributes.
  • Curly braces are only needed when specifying a format and / or default value.
  • Use the modifiedon attribute for current date.
  • Format the attribute in the Condition or Formula field and not in the parameters field unless the value needs to be formatted twice.
Data Concatenation

The syntax for formatting and concatenating Condition/Formula and Condition/Formula Parameters fields is the same as the String Format (String format, params object[] args) method in the .NET Framework. In this syntax:

  • String parameter “format” is the value of the Condition/Formula field.
  • Object[] parameter “args” is the value of the Condition/Formula Parameters.

To reference the attributes in Condition/Formula Parameters field, the Condition/Formula field follows the String.Format placeholder convention: {0} to refer to the first attribute, {1} for the second, and so on. The maximum number must be less than or equal to the number of attributes listed in Condition/Formula Parameters field. Number placeholders can be used more than once in the formula.
