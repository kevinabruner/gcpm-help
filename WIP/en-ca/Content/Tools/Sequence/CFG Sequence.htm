Configure a sequence record

To create a new CFG Sequence record:

  1. Navigate to Configuration > Calculation Tools > CFG Sequence and create a new record.
  2. Use the onscreen help and notes below for guidance in filling in the fields: 
    • Target entity: Type the logical name of the entity where this sequence is applied.
    • Target field: Type the logical name of the field in the target entity that stores the generated sequence value.
    • Seed:Select an existing seed for instances where multiple entity records will generate seeds from the same pool. Or create a new seed
  3. Child sequence: 
    • Regarding: If this is a new sequence for child records, set this field to the logical name of the lookup field of the parent.
    • Resequence: If set to yes, the plugin will re-sequence the child records when a child record is deleted.
  4. Formula: 
    • Formula parameters: Specify a comma delimited list of logical field names whose values are to be used in the calculation or text formatting / concatenation. See note for parameter syntax, below.
    • Formula: See note on how formula does calculations and data formatting, below.
  5. Metadata fields are auto-populated.
    • Related entities: Parent entities that are referenced in the formula parameters.
    • Metadata: Formula metadata is stored here to optimize performance.
  6. Save and close the record.
Parameter syntax

The parameter syntax is as follows:

{lookup.attribute:format;default value }

Syntax considerations:

  • The lookup name is only needed when referencing parent attributes.
  • Curly braces are only needed when specifying a format and / or default value.
  • Use the modifiedon attribute for current date.
  • Format the attribute in the Condition or Formula field and not in the parameters field unless the value needs to be formatted twice.
Data Concatenation

The syntax for formatting and concatenating Condition/Formula and Condition/Formula Parameters fields is the same as the String Format (String format, params object[] args) method in the .NET Framework. In this syntax:

  • String parameter “format” is the value of the Condition/Formula field.
  • Object[] parameter “args” is the value of the Condition/Formula Parameters.

To reference the attributes in Condition/Formula Parameters field, the Condition/Formula field follows the String.Format placeholder convention: {0} to refer to the first attribute, {1} for the second, and so on. The maximum number must be less than or equal to the number of attributes listed in Condition/Formula Parameters field. Number placeholders can be used more than once in the formula.

Formula calculation and data formatting

To perform the calculation, the Formula field leverages the syntax used in .NET Framework DataTable class, specifically the DataColumn.Expression* property. The values in the calculation are pieced together just like formatting and concatenating fields, so each value can be formatted prior to the actual calculation.

Refer to Microsoft’s documentation on DataColumn.Expression property for valid calculation syntax. It uses the same syntax as SQL.
A cheat sheet is provided here: Calculation Syntax

When parameter in the Calculation/Formula Parameters field are accessed through placeholders, they can be formatted. Refer to Microsoft’s documentation on the Format method for valid data formatting syntax.
The general formula is {value,alignment:format}

A cheat sheet is provided in Data formatting syntax guide.

Example:

To apply yyyy-MM-dd format to the egcs_duedate field:

  • Parameters: egcs_duedate
  • Formula: {0:yyyy-MM-dd}

Since there’s no way to specify a time value, say you wanted to subtract or add months, days, or years to a field which data type is date/time, a special syntax is used to input time values.

Time values are to be specified in the Formula Parameters field and not in the Formula field, so that no parsing is required.

For example, if you want to add 10 days to a field named egcs_duedate that is an attribute of lookup egcs_application entity and its default value is January 1st, 2018.

Parameters are: {egcs_application.egcs_duedate; '2018-01-01'},10dd

Formula is: {0} + {1}
