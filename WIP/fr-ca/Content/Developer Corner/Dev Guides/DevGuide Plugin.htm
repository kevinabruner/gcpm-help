Guide de développement des plugiciels

Afin de normaliser le développement des plugiciels à l'échelle de la solution, le GPSC a une structure de base des plugiciels que tous les plugiciels doivent se conformer afin de s'assurer du respect des exigences du GPSC. Veuillez suivre ce guide lors du développement de nouveaux plugiciels pour le GPSC.

Remarques :

  • Plugiciels de mémoire cache de CRM
  • Le CRM n'initialise pas les plugiciesl toutes les fois.
  • Réaliser des plugiciels sur de multiples fils en utilisant l'agilité des fils.

Afin d'utiliser la structure de base des plugiciels, établir les éléments suivants. Ils sont simples et fonctionnent comme l'interface de plugiciel de CRM avec quelques additions.

  1. Bibliothèques de référence partagées

    2. La structure de base des plugins se situe dans les bibliothèques partagées suivantes dans le EGCS.CRM.Framework, vous devez dons ajouter les références ci-dessous à votre projet de plugin.

  2. Hériter de la classe de plugiciel

    3. Tout comme le développement standard de plugiciel de CRM, où vous héritez de l'interface IPlugin de CRM, vous devez tout d'abord hériter de la classe du plugiciel de base qui est simplement appelé Plugin (Plugiciel).

    Une fois que vous héritez de la classe du Plugin, vous devez mettre en œuvre les méthodes suivantes.

  3. Exécuter - Mise en œuvre de l'exécution du plugiciel par défaut

    4. Cette méthode remplace la méthode Exécuter (IServiceProvider serviceProvider) method provided by the IPlugin. Tout est installé pour vous, afin que vous n'ayez pas besoin d'écrire un code passe-partout pour commencer. La seule différence entre les deux méthodes de mention est le paramètre.

    La PluginContext class is similar to CRM's IPluginExecutionContext. You can use it as if it were the same. You will find that a lot of the things are set up for you, so you don't have to worry about it yourself, such as the pre- and post-image entities if there are any. The actual IPluginExecutionContext is also stored in PluginContext propriété en tant que CRM.

  4. Configurer les paramètres du plugiciel

    5. Actuellement, il y a quelques paramètres que vous pouvez configurer pour votre plugiciel. Vous devrez renvoyer un objet de type PluginConfiguration in the ConfigurePlugin method, which the following settings configured.

    1. PluginExecution - Required
      This is the only required parameter when creating a new PluginConfiguration instance. Although it is required, it is not required to configure it. The PluginExecution separates the plugin logic between different plugin events, plugin stages, and plugin modes. It acts as a switch that triggers the method assigned to a particular event/stage/mode.
      To configure PluginExecution, simply pass in an Action<PluginContext> à un événement de plugiciel spécifique, et éventuellement à une étape et à un mode de plugiciel spécifiques.

  • Profondeur maximale d'un appel - Facultatif

    • Établit la profondeur maximale d'un appel permise par le plugiciel. La profondeur d'appel se réfère à où dans la pile le plugiciel est appelé. Par exemple, le premier plugiciel qui se déclenche aura une profondeur de 1. Si le premier plugiciel appelle un autre plugiciel, le deuxième plugiciel aura une profondeur de 2.

    Un plugiciel peut appeler un autre plugiciel de multiple fois et chaque appel de plugiciel possède la même la profondeur d'appelé. Par exemple, si le premier plugiciel crée 5 nouveaux enregistrements et que chaque enregistrement déclenche un plugiciel, ces 5 appels de plugiciel auront tous une profondeur d'appel de 2.
    Par défaut, la profondeur maximale établie par le CRM est de 8, laquelle peut être changée.

Pratique exemplaire :

  • Vous pouvez souhaiter de déclencher votre plugiciel jusqu’à une certaine profondeur d'appel p. ex. afin d'arrêter les plugiciels récursifs jusqu'à un certain point.
  • Établit la profondeur maximale à 1 afin de s'assurer qu'un plugiciel est seulement déclenché directement par un enregistrement et non par un autre plugiciel.

Vous pouvez enregistrer des extensions de plugiciel afin qu'elles fonctionnent séparément ou avant et après l'exécution du plugiciel. Tout objet qui met en œuvre l'interface IPluginExtension est une extension de plugiciel valide

Contourner le mécanisme de mémoire cache du plugiciel de CRM

Puisque le CRM met en mémoire cache l'objet du plugiciel, au lieu de créer un nouveau objet de plugiciel chaque fois, simplement déplacer toute votre logique de plugiciel à une différente classe et créez un nouvel objet de cette classe pendant l'exécution. De cette façon, vous pouvez avoir toutes les variables de classe générale que vous voulez.

Extensions de plugiciel - Enregistreur d'étape de plugiciel

L'objectif de cette extension est d'épargner à l'utilisateur le tracas d'enregistrer les étapes de plugiciel manuellement. Lorsque configurée, cette extension créera, mettra a jour et supprimera les étapes de plugiciel alors que des enregistrements de configuration sont créés, mis à jour et supprimés.
