Principales étapes pour déployer GPSC 2.1

Le déploiement de GPSC a été mis à l'essai dan cet environnement (l'environnement de développement du  GPSC) :

  • Microsoft SQL Server 2016 (SP2-GDR) (KB4293802) - 13.0.5081.1 (X64) 20 juillet 2018 22:12:40 Copyright (c) Microsoft Corporation Developer Edition (64-bit)
  • Microsoft Dynamics 365 Version 1612 (9.0.2.3041) (DB 9.0.2.3041) sur le lieux
  • Module linguistique français de Microsoft Dynamics 365

Remarque sur les scripts d'exécution

Vous devez mettre à jour les scripts afin de se connecter dans différents environnements (sur les lieux ou en ligne). Par exemple :

  • Exemple d'une fonction pour commencer le déploiement - sur les lieux
    Import-CrmMultiplePackagesAsync -OrgUrl '<org name>' -ServerUrl ‘<server url>’ -IsCrmOnline $false -path '<path to release folder location>' -CsvImportList '\SCRIPTS\package_list_v2.csv'
  • Exemple d'une fonction pour commencer le déploiement - en ligne

    Import-CrmMultiplePackagesAsync -OrgUrl '<org name>' -ServerUrl '<server url>’ -IsCrmOnline $true -path '<path to release folder location>’ -CsvImportList '\SCRIPTS\package_list_v2.csv'

Structure des dossiers

/GCPM

/FINAL_PACKAGE

Ensemble de dossiers / liens de téléchargement direct qui sont lisible par l'outil PackageDeployer de SDK.

Comme exemple :

/SCRIPTS
  • Org_Settings_ONLINE_ONPREM.ps1 - fichier PowerShell qui établit les paramètres org .
  • Load_Packages_ONLINE_ONPREM.ps1 - fichier PowerShell qui exécute l’ensemble des fichiers de PackageDeployer à partir du dossier FINAL_PACKAGE
  • package_list_v2.csv - Ordre d'exécution de différents projets PackageDeployer
/UTILITY
  • Enable_BusinessRules_ONPREM_ONLINE.ps1 et businessrulelist.csv Script PowerShell et fichier d'entrée avec une liste des règles d'entreprise qui ont besoin d'être désactivées après l'installation initiale du système de GPSC
  • CleanUP-Views-ONPREM_ONLINE.ps1 et viewlist.csv Script PowerShell script et fichier d'entrée avec une listes des aperçus prêt à l'emploi qui ont besoin d'être désactivées après l'installation initiale du système de GPSC
  • Enable_Connections_ONPREM_ONLINE.ps1 et connectionrolelist.csv Script PowerShell et fichier d'entrée avec une liste des connections prêtes à l'emploi qui ont besoin d'être désactivées après l'installation initiale du système de GPSC
  • Import-CRMUsers_ONPREM_ONLINE.ps1 et users.csv Script PowerShell et exemple de fichier d'entrée pour ajouter des utilisateurs.
  • TeamToRole-ONPREM_ONLINE.ps1 et teamroles.csv Script PowerShell et exemple de fichier d'entrée avec des relations N:N entre les rôles et les équipes qui ont besoin d'être activées après l'installation initiale du système de GPSC
  • EntityMaps-ONPREM_ONLINE.ps1 et mapping.csv Script PowerShell et exemple de fichier d'entrée avec des renseigneemtns de mise en correspondance pour les relations entre les différentes entités qui ont besoin d'être importées après l'installation initiale du système de GPSC
/LOG

Dossier pour conserver les fichiers générés par l'outil PackageDeployer

Principales étapes pour déployer la Version 2.0 de GPSC

  1. Créer une organisation
    Remarque : utilisez le Dollar canadien comme devise par défaut.
  2. Configurer les paramètres de l'organisation en exécutant ce script:

    Org_Setting_ONLINE_ONPREM.ps1
    OU

  3. Mettre à jour manuellement les paramètres de l'organisation en utilisant l'interface de Dynamics 365. Naviguer vers
    Paramètres > Administration > Langues ou Paramètres > Administration > Paramètres système
    • Activer le module linguistique français
    • Activer le module linguistique
    • Activer l'aide générale et établir l'URL vers le fichier d'aide
    • Changer le format à Anglais canadien
    • Désactiver le suivi
    • Désactiver l'enregistrement automatique
    • Désactiver les privilèges
    • Désactiver l'intégration à la carte Bing
  4. Déployer les Solutions et les Données en exécutant le fichierLoad_Packages_ONLINE_ONPREM.ps1 .
    Remarque: le déploiement de la solution de GPSC peut prendre un moment.
  5. Configurer les Grilles Crowe. Naviguez vers Paramètres > Paramètres de la grille EditAble CRM et cliquez sur Demander une nouvelle licence.
    REMARQUE : une licence pour la grille Crowe n'est pas fournie. Vous devez vous procurer une licence pour votre organisation.
  6. Exporter des Grilles Crowe: naviguez vers Paramètres > Grilles EditAble CRM et cliquez sur EXPORTER GRILLES EDITABLE.

    REMARQUE : la configuration des Grilles Crowe est stockée dans les ressources Web suivantes.


    Remarque : après l'importation de toutes les grilles Crowe, vous devriez avoir 100 enregistrements.

    Remarque : certaines des Grilles Crowe ne peuvent pas être publiées (voir aperçu ci-dessous). Veuillez supprimer ces enregistrements :

    Voir l'aide sur les Grilles Crowe pour de plus amples renseignements.

  7. S'assurer que tous les processus sont activés. Utiliser la Recherche avancée pour générer une listes des processus inactifs :
    Remarque : les flux liés à la fonction d'aperçu a besoin d'être rouverte manuellement et réactivée.
  8. Exécuter les scripts PowerShell suivants :

    • Enable_BusinessRules_ONPREM_ONLINE.ps1
    • CleanUP-Views-ONPREM_ONLINE.ps1
    • Enable_Connections_ONPREM_ONLINE.ps1
    • TeamToRole-ONPREM_ONLINE.ps1
  9. Ajouter des utilisateurs. Le faire manuellement ou l'exécuterImport-CRMUsers_ONPREM_ONLINE.ps1
  10. Ajouter des utilisateurs aux équipes - cela doit être fait manuellement.
  11. Configurer le gestionnaire des documents.
    Voir Principales étapes pour la configuration du gestionnaire de documents
    • (si vous utilisez SharePoint) Reconfigurer la gestion des documents pour SharePoint : mettre à jour les liens dans CFG des Lieux de documents et dans les entités des lieux de documents afin qu'ils pointent vers votre site SharePoint et vos bibliothèques de documents.
    • Remarque : GPSC a été configuré avec un échantillon de la configuration de SharePoint, afin d'offrir un exemple à L'équipe de déploiement. Ces exemples d'enregistrement de configuration sont situés dans (CFG des Lieux de documents, CFG des Entités de documents et Lieux des documents). Pour désactivé cet exemple de configuration :
      • Désactiver tous les enregistrements dans l'entité CFG Document. Utilisez cette requête dans la Recherche avancée pour localiser tous les flux qui génèrent des aperçus :
      • Désactiver tous les flux qui génèrent des aperçus.
  12. Établir une intégration financière
    Si vous choisissez de configurer une intégration financière à un moment ultérieur, désactivez tous le flux dans la solution GCPM_FI_AUTOMATION.
