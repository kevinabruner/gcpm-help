API REST du système financier

L'API REST du système financier offre tous les points de terminaison qui communique aux étapes de flux du GPSC et à partir d'eux. L'API rédige actuellement une base de données à la mémoire et les clients de l'API sont requis de mettre en œuvre chaque point de terminaison pour communiquer avec leur système financier. Il est important de noter que lors de la mise en œuvre de chaque point de terminaison, la charge utile (ou le modèle) devrait être le même que ce qui est mis en œuvre dans l'API. L'API s'exécute avec un pont de terminaison Swagger lequel illustre chaque point de terminaison et quelle charge utile dont il a besoin. Par exemple, l'API REST du système financier du  GPSC est utilisé pour le développement et la mise à l'essai se retrouve ici  : https://gcpm-finapi-app.egcs-sesc.ca/index.html

Modèles

Les modèles suivants sont créés dans .NET Core en utilisant le cadre de l'entité. Le cadre de l'entité peut se connecter à la majorité des base de données SQL des fournisseurs majeurs comme MSSQL, Oracle, PostgresSQL, MySQL ou un magasin de donnée en mémoire comme la mise en œuvre utilisée dans le développement et la mise a l'essai du GPSC. Les mises en œuvre doivent adhérer à ce modèle pour s'assurer que la communication avec les entités CRM du GPSC et les flux soient prêt à l'emploi.

Points de terminaison

Les points de terminaison offert par l'API seront prêt à l'emploi si la mise ne œuvre utilise une base de données qui adhère au modèle. L'intégration avec le système financier se produit alors à une couche séparée (par exemple, met à profit un ETL pour déplacer les données vers et à partir du système financier à la base de données de l'API du système financier du GPSC).
