Intégration des Fournisseurs clients

Les Fournisseurs clients stockent les renseignements dans le GPSC qui sont liés au Fournisseur et aux Clients dans le système financier.

Le GPSC a plusieurs points qui font référence et qui interagissent avec un fournisseur client.

Conditions préalables:
  • Solutions d'intégration financière
  • Flux pour l'intégration financière
  • Financial integration custom workflow steps
  • Modèles d'entité du système financier

Associer un fournisseur ou un client avec un Client SC

Afin d'associer un fournisseur ou un client existant avec un Client SC :

  1. Dans un enregistrement de Client SC, naviguez vers les Données du système financier et cliquez sur + pour créer un nouveau enregistrement FournisseurClient.

  2. Changer le type de Founisseur Client à Cartographié.

  3. Remplissez les champs au moyen de la fonction d'aide à l'écran et les notes ci-dessous :
    • Nom : rempli automatiquement avec le nom officiel du compte. Votre organisation peut choisir si vous souhaitez verrouiller ce champ.
    • Est-ce un Fournisseur : Oui / Non. Oui affichera les champs relatifs au fournisseur.
    • Est-ce un Client : Oui / Non. Oui affichera les champs relatifs au client.
    • Type de carte Fournisseur Client : indique si c'est un FournisseurClient nouveau ou cartographié (existant).
    • Utilisateur de l'intégration financière : l'utilisateur qui soumet les données au système financier. Il sera automatiquement rempli si votre organisation décide de mettre en œuvre le remplissage automatique.
  4. Selon les signaux Est-ce un Fournisseur et Est-ce un Client, vous verrez le code de Fournisseur, le code de Fournisseur Client ou le code de Client.
  5. Entrez identifiant unique de votre système financier dans le champs approprié.

  6. Enregistrez l'enregistrements pour attirer les données.

Automatiser la création d'intégration financière des Fournisseur Client

Afin d'automatiser la création d'intégration financière des Fournisseur Client pendant le cycle de vie du projet :

  1. Naviguez vers Paramètres > Processus et ouvrez le flux Générer FournisseurClient du projet (ou un autre flux que vous voulez utiliser pour générer l'enregistrement d'intégration financière de l'enregistrement du FournissseurClient à partir de votre projet).
  2. Agrandir la fenêtre afin de voir son  ID dans la barre d'adresse.
  3. Copiez l'ID et conservez-le pour une utilisation ultérieure. Excluez %7b and %7d lors de la copie.
  4. Naviguez vers votre modèle de transition désiré : Programmes > et sélectionnez un programme.
  5. Agrandir les Modèles du cycle de vie des projets et cliquez sur recherche du Modèle de transition.
  6. Trouvez la transition adéquate et ouvrez son ensemble d'action.
  7. Si vous avez besoin de créer une action :
    • Ajoutez une nouvelle action, sélectionnez la définition de Déclencher flux et remplissez tout autre champ désiré.
    • Défilez vers le bas et entrez l'ID du flux que vous avez copiez dans l'étape ci-dessus et cliquez sur Enregistrer paramètres.
    • Enregistrez et fermez l'action.

Importer des Fournisseurs et des Clients

Si vous choisissez de faire une importation complètes des Fournisseurs et des Clients à partir de votre système financier vous devriez les éléments suivants à l'esprit :

  • Il y a une recherche pour l'intégration financière des Fournisseurs Clients aux niveaux du Compte, du Client SC et du Projet qui devrait être remplie une fois que l'enregistrement de l'intégration financière du Fournisseur Client sont créés.
  • Chaque compte et chaque Client SC ont seulement une intégration financière du Fournisseur Client. Créez un enregistrement avec les données du Fournisseur et du Client qui s'y trouvent. Ne créez pas deux enregistrements pour garder un fournisseur et un client séparé.

Si votre organisation ne fait pas une importation complète des objets FournisseurClient :

  • Le flux Generate VendorCustomer from Project n'est pas assez intelligent pour savoir s'il existe un FournisseurClient dans votre système financier qui devrait être cartographié à un projet de Client SC plutôt que d'en créer un nouveau.
  • Afin d'atténuer ce problème, une action de validation Verify Vendor does not already exist in Financial System (Vérifier que le Fournisseur n'existe pas dans le système financier) crée un avertissement dans le cas qu'un champ de l'intégration financière du Fournisseur Client sur le Client SC soit vide.
