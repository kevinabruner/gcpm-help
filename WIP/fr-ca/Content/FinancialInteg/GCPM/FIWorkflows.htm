Flux pour l'intégration financière

Une série de flux facilite les transactions du système financier dans le système et crée automatiquement des enregistrements GPSC et fait la transition entre les différents états pour les enregistrements d'intégration financière (voir la section de l'appareil d'état).

Voir tous les flux dans ce fichier Excel.

Flux d'engagement
Les flux pour gérer les engagements

Flux

Description

Il génère un engagement souple
  • Il génère un enregistrement d'intégration financière de l'engagement.
  • Il génère un enregistrement d'intégration financière du détail de l'engagement pour chaque enregistrement de flux de trésorerie sur le projet.
  • Il déclenche le flux enfant Sync and Send Commitment to Finance

En voyer des engagements fermes au projet
  • Si un projet n'a pas une intégration financière de l'engagement, ce flux fait appel au flux enfant Generate Soft Commitment (Générer un engagement souple)
  • Il met à jour le champ d'intégration financière de l'engagement à Type = Hard (Type = Ferme)
  • Il déclenche le flux enfant Sync and Send Commitment to Finance (Synchroniser et envoyer l'engagement à Finance)

Synchroniser et envoyer l'engagement à Finance

S'assure que ce projet a un engagement associé, sinon il échoue.

  • Il change l'intégration financière de l'engagement associé à ébauche.
    Consulter Intégration financière des états de l'entité (aussi appelé appareil d'état).
  • Si cette intégration financière de l'engagement à un ID du système financier, un appel à Update Commitment from Finance (Mettre à jour l'engagement à partir de Finance) est réalisé (voir ci-dessous).
  • Il met à jour le champ de l'intégration financière de l'utilisateur de l'intégration financière de l'engagement pour informer le système financier de la personne qui soumet la demande. Voir de plus amples détails sur la mise en place d'enregistrement d'intégration financière d'utilisateur.
  • Il soumet l'intégration financière de l'engagement à Finance :
    • Si la soumission est réussie, une transaction réussie est générée et le flux est terminé.
    • Si la soumission échoue, une erreur de transaction est générée, l'intégration financière de l'engagement est défini à un état d'erreur et le flux se termine.

Synchroniser le flux de trésorerie avec les détails de l'engagement - Créer

Il s'assure que l'intégration financière du détail de l'engagement n'a un flux de trésorerie en référence :

  • Il s'assure que cette intégration financière du détail de l'engagement a un projet associé, sinon il échoue.
  • Il prend un enregistrement d'exercice financier, selon l'exercice précisé sur la nouvelle intégration financière du détail de l'engagement.
  • Il stocke l'état actuel du signal Évaluation verrouillée du projet.
  • Il définit le signal de « Évaluation verrouillée » du projet à Non.
  • Il crée un enregistrement de flux de trésorerie, remplissant ces champs : Projet, Exercice financier, Montant, Montant dépensé et Référence au détail de l'engagement.
  • Il définit le signal de l'Évaluation verrouillée du projet à son état original.
  • Il définit le champ « Flux de trésorerie » sur l'intégration financière de l'engagement au flux de trésorerie nouvellement créé.
For these workflows: Sync Cash Flow with Commitment Detail – Create, Update, Delete
  • To improve performance, we recommend removing the steps that change the Assessment Locked flag. Instead, make a call to the action Enable Lock Plugin Override. This will allow the commitment detail to be created / updated / deleted without the expensive recursive action of unlocking the project.
  • Check for error status on the associated FI Commitment Detail so as not to update a cash flow record with invalid data.

Synchroniser le flux de trésorerie avec les détails de l'engagement - Supprimer

Il s'assure que l'intégration financière du détail de l'engagement a un flux de trésorerie en référence :

  • Il stocke l'état actuel du signal « Évaluation verrouillée » du projet.
  • Il définit le signal de Évaluation verrouillée du projet à Non.
  • Il désactive l'enregistrement de flux de trésorerie associé.
  • Il définit le signal de l'Évaluation verrouillée du projet à son état original.
For these workflows: Sync Cash Flow with Commitment Detail – Create, Update, Delete
  • To improve performance, we recommend removing the steps that change the Assessment Locked flag. Instead, make a call to the action Enable Lock Plugin Override. This will allow the commitment detail to be created / updated / deleted without the expensive recursive action of unlocking the project.
  • Check for error status on the associated FI Commitment Detail so as not to update a cash flow record with invalid data.

Synchroniser le flux de trésorerie avec les détails de l'engagement - Mettre à jour

Il s'assure que l'intégration financière du détail de l'engagement a un flux de trésorerie en référence :

  • Il stocke l'état actuel du signal Évaluation verrouillée du projet.
  • Il définit le signal de Évaluation verrouillée du projet à Non.
  • Il met à jour les champs suivants de l'enregistrement de flux de trésorerie associé : Montant, Montant dépensé.
  • Il remet le signal de l'Évaluation verrouillée du projet à son état original.
For these workflows: Sync Cash Flow with Commitment Detail – Create, Update, Delete
  • To improve performance, we recommend removing the steps that change the Assessment Locked flag. Instead, make a call to the action Enable Lock Plugin Override. This will allow the commitment detail to be created / updated / deleted without the expensive recursive action of unlocking the project.
  • Check for error status on the associated FI Commitment Detail so as not to update a cash flow record with invalid data.

Mettre à jour l'engagement - Modification

Il s'assure que cette modification de projet a un engagement associé, sinon il échoue.

  • Il met à jour l'intégration financière de l'engagement et ses détails selon les modifications au flux de trésorerie.
    • Si un flux de trésorerie à été ajouté, il déclenche Sync Cash Flow with Commitment Detail – Create (Synchroniser le flux de trésorerie avec les détails de l'engagement - Créer).
    • Si un flux de trésorerie à été mis à jour, il déclenche Sync Cash Flow with Commitment Detail – Update (Synchroniser le flux de trésorerie avec les détails de l'engagement - Mettre à jour).
    • Si un flux de trésorerie à été supprimé, il déclenche Sync Cash Flow with Commitment Detail – Delete (Synchroniser le flux de trésorerie avec les détails de l'engagement - Supprimer).
  • Il déclenche le flux enfant Sync and Send Commitment to Finance.

Mettre à jour l'engagement à partir de Finance
  • Si l'état de l'intégration financière de l'engagement n'est pas une Ébauche, le flux échoue.
  • Il fait un appel pour récupérer le dernier engagement et ses détails à partir du système financier, lequel met à jour l'intégration financière de l'engagement et son intégration financière des détails de l'engagement.
    • Si la soumission est réussie, une transaction réussie est générée et le flux est terminé.
    • Si la soumission échoue, une erreur de transaction est générée, l'intégration financière de l'engagement est défini à un état d'erreur et le flux se termine.
  • L'intégration financière de l'engagement doit avoir le Statut - Ébauche a modifier. Si vous mettez en œuvre un mise à jour périodique de tous les objets de Finance, le changement du statut à ébauche pour tous ces objets peut prendre du temps. Si c'est le cas, vous pouvez considérer mettre à jour seulement les objets de l'intégration financière qui sont liés aux projets « actif » (« actif » sera défini par votre organisation).

Flux de Fournisseur Client

Flux pour gérer les Fournisseurs Clients

Flux

Description

Il génère un FournisseurClient à partir d'un projet
  • Il crée un enregistrement de
  • Il remplit les données de l'enregistrement à partir du compte du projet actuel.
  • Il envoie l'enregistrement au système financier.
  • Il associe l'enregistrement à ce compte de projet et aux enregistrements de Client SC.
  • Il associe l'enregistrement à ce projet.

Il met à jour le compte à partir du  FournisseurClient
  • Il est appelé sur demande par l'agent à partir du compte.
  • Il met à jour l'enregistrement de FournisseurClient de ce compte à partir du système financier.
  • Il met à jour les renseignements du compte bancaire sur ce compte à partir de l'enregistrement d'intégration financière de FournisseurClient nouvellement mis à jour.

  • Il peut vouloir vérifier pour un état d'erreur ici, puisqu'une erreur pourrait faire en sorte que certains renseignements de l'intégration financière du FournisseurClient soient manquants (selon la façon dont vous mettez en œuvre) ce qui pourrait entraîner la suppression involontaire des valeurs dans votre compte.

Mettre à jour le FournisseurClient à partir d'un compte
  • Il est appelé sur demande par l'agent à partir d'un enregistrement de compte.
  • Il met à jour l'enregistrement du FournisseurClient du compte avec les données du compte.
  • Il soumet les changements au système financier.

  • Il définit l'intégration financière du FournisseurClient à un état d'ébauche. Consulter Intégration financière des états de l'entité (aussi appelé appareil d'état).

  • Il définit le champ de l'intégration financière de l'utilisateur pour informer le système financier de la personne qui soumet la demande. Consulter .
Flux d'intégration des créances
Flux pour gérer l'intégration des Créances

Flux

Description

Synchroniser et envoyer la Créance à Finance -  Projet
  • Si ce projet a déjà une créance, définir son statut à ébauche (voir Intégration financière des états de l'entité (aussi appelé appareil d'état) pour une explication de l'intégration financière des états d'entité)
  • Si l'intégration financière de la créance de ce projet se situe déjà dans le système financier, il déclenche un flux enfant « Update Receivable From Finance » (Mettre à jour la Créance à partir de Finance) (décrit ci-dessous)
  • Il crée ou met à jour l'intégration financière de la créance avec les versements du calendrier de remboursement les plus récents
  • Il déclenche le flux enfant « Send Receivables to Finance » (décrit ci-dessous)

Synchroniser et envoyer la Créance à Finance -  Modification
  • Si le projet de cette modification a une intégration financière des créances, définir son statut à ébauche.
    • Si l'intégration financière de la créance de projet de cette modification se situe déjà dans le système financier, il déclenche un flux enfant « Update Receivable From Finance » (décrit ci-dessous)
  • Il crée ou met à jour l'intégration financière de la créance avec les versements du calendrier de remboursement modifiés.
  • Il déclenche le flux enfant « Send Receivables to Finance » (Envoyer les Créances à Finance) (décrit ci-dessous)

Envoyer des Créances à Finance
  • Il s'assure que l'intégration financière de la créance a un référence au projet, autrement il échoue
  • Il change le statut de l'intégration financière de la créance a ébauche
  • Il met à jour le champ de l'intégration financière de l'utilisateur de l'intégration financière de la créance pour informer le système financier de la personne qui soumet la demande.
  • Il envoie la créance à Finance
  • S'il réussit le statut de l'intégration financière de la créance est défini à Terminé.
  • Autrement, il définit le statut de l'intégration financière de la créance à Erreur.

Mettre à jour la créance à partir de Finance
  • Si cette intégration financière de la créance n'est pas à l'état d'ébauche, il échoue
  • Il déverrouille le projet associé
  • Il récupère les derniers détails de l'intégration financière de la créance à partir du système financier
  • Il verrouille de nouveau le projet
  • Si la récupération est réussie, il ne reste rien a faire
  • Autrement, définissez le statut de l'intégration financière de la créance à Erreur.

  • Remarque : c'est un autre endroit où vous pouvez souhaiter passer outre le verrous plutôt que de déverrouiller l'évaluation de projet
    Pour obtenir de plus amples renseignements, consultez Surclasser un verrou.
Flux d'intégration de paiement
Les flux pour gérer l'intégration de paiements

Flux

Description

Il génère un paiement
  • Il crée une intégration financière du paiement
  • Il crée une intégration financière des détails du paiement selon les montants en dollar dans l'examen de débours :
    • Si le montant du paiement modifié > 0
      • Il crée une intégration financière du détail du paiement avec le codage du projet associé pour les « Coûts encourus »
    • Si le nouveau paiement anticipé > 0
      • Il crée une intégration financière du détail du paiement avec le codage du projet associé pour le « Paiement anticipé »
    • Si le règlement de paiement anticipé > 0
      • Il crée une intégration financière du détail du paiement avec le codage du projet associé pour les « Règlement de paiement anticipé - Coûts encourus » (montant positif)
      • Il crée une intégration financière du détail du paiement avec le codage du projet associé pour les « Règlement de paiement anticipé - Paiement anticipé » (montant négatif)
    • Si la réduction du CAFE > 0
      • Il crée une intégration financière du détail du paiement avec le codage du projet associé pour le « CAFE »

      Si la retenue > 0

      • Il crée une intégration financière du détail du paiement avec le codage du projet associé pour la « Retenue » (montant négatif)

      Si la libération de la retenue > 0

      • Il crée une intégration financière du détail du paiement avec le codage du projet associé pour la « Retenue » (montant positif)

Envoyer paiement à Finance
  • Il s'assure que cette intégration financière du paiement a une référence à l'examen du projet et des débours, autrement il échoue
  • Il change le statut de l'intégration financière du paiement à ébauche
  • Il met à jour le champ de l'intégration financière de l'utilisateur pour informer le système financier de la personne qui soumet la demande.
  • Envoyer le paiement à Finance
  • S'il réussit le statut de l'intégration financière du paiement est défini à Terminé.
  • Autrement, il définit le statut de l'intégration financière du paiement à Erreur.
