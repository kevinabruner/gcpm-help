Pratiques exemplaires : Questions d'évaluations

C'est une pratique exemplaire que d'établir les grandes lignes de votre processus d'évaluation avant de mettre la dernière main aux questions à poser dans le cadre de l'évaluation et de tirer parti de l'automatisation et des gains de productivité que les questions d'évaluation peuvent procurer.

Voici quelques idées de questions à poser pour évaluer adéquatement votre organisme :

Affichez les questions dans des endroits convenables.

Souhaitez-vous afficher les questions dans un seul endroit ou les répartir en tableaux, en fonction du titre de la section?

Par exemple, les questions au sujet des éléments du coût de revient peuvent être affichées sous le tableau à cet effet.

  • Servez-vous du champ Titre de la section pour répartir les questions dans différents tableaux.
  • Assurez-vous que les tableaux sont configurés dans chacune des sections du formulaire où paraîtront les questions de l'évaluation.
  • Les réponses texte et les valeurs à consulter ne doivent pas paraître dans les mêmes tableaux de Crowe. Il faut absolument qu'elles soient dans des tableaux distincts.
Décidez des questions sur les projets qui paraîtront.

Dans le cas de projets qui satisfont à certains critères, affichez des questions supplémentaires. Les sections du formulaire peuvent être configurées de manière à montrer (ou à occulter) de manière dynamique selon les réponses.

Par exemple, "Votre organisme est-il à but lucratif ou à but non lucratif?"

  • = à but lucratif, elles affichent des questions supplémentaires concernant les entités à but lucratif dans un tableau de Crowe distinct.

  • = à but non lucratif, elles occultent les questions non pertinentes à un organisme sans but lucratif.
  • Par ailleurs, les tableaux peuvent être configurés de différents manières (p. ex., un formulaire pour les entités à but lucratif et un autre pour les entités sans but lucratif).
Affichez les données dans la GPSC en fonction des résultats de l'évaluation.
  • Servez-vous du champ Code de question pour afficher ou occulter de manière dynamique les sections du formulaire en fonction de la réponse à une question de l'évaluation.
  • Par exemple, si la question "Les coûts du projet s'élèvent à plus de 5 M$" = "Non", alors la section du formulaire qui comporte des questions sur les projets de plus de 5 M$ est occultée.
Faites référence aux résultats de l'évaluation dans les documents.

Servez-vous des questions de l'évaluation pour vous assurer que l'évaluation des cadres est prise en compte dans les communications avec le client, l'accord de contribution et dans la phase de suivi du projet.

  • Nota : Le champ Code de question est alphanumérique (composé de lettres et de chiffres), mais ne comporte pas d'espaces.
  • Faites référence au champ Code de question dans les modèles de document en format XML.
  • Par exemple, la question "Le projet nécessite-t-il un changement de contrôle?" se voit attribuer le code de question = "CC".
    Dans le modèle d'accord de contribution en format XML, la clause associée avec un changement de contrôle entrevoie au code "CC". Inscrivez le bon code pour afficher la clause associée à la réponse "CC". Le document généré par le modèle affichera la clause appropriée en fonction de la réponse à cette question.
Quand doit-on créer plusieurs modèles d'évaluation

Un même modèle d'évaluation peut servir à rendre compte de plusieurs facteurs d'évaluation, évitant ainsi de devoir multiplier les modèles d’évaluation (p. ex., dans le cas d'organismes à but lucratif et de ceux à but non lucratif).

Cependant, s'il y a plus d'un ou deux facteurs d'évaluation conditionnelle, cela peut compliquer la configuration d'un seule et même modèle. S'ils sont davantage, il est conseillé de faire des modèles distincts. Par exemple :

  • Les coûts du projet s'élèvent à plus / moins de 5 M$

  • Entités à but lucratif et à but non lucratif
