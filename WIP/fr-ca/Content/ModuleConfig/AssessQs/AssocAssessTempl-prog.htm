Choisir un modèle d'évaluation pour un programme

Lorsqu'un enregistrement du Modèle d'évaluation | A est configuré comme le modèle d'évaluation actuel sur un programme, les questions dans le modèle sont copiés sur les projets dans le programme et un enregistrement est créer dans l'entité d'intersection Modèle de l'évaluation | PG. Un modèle d'évaluation peut être associé avec un ou plusieurs programmes.

Pour choisir un modèle d'évaluation pour un programme:

  1. Ouvrez un programme et naviguez vers la section Questions d'évaluation.

  2. Cliquez sur le champs des Question d'évaluation actuelles et choisir un modèle d'évaluation.
    Si vous remplacez le modèle actuel avec un nouveau modèle, le modèle précédent s'affiche dans la grille.
  3. Donner une date de début et de fin.
  4. Enregistrez et fermez le programme.
Les fonctionnalités des modèles de GPSC ont ces caractéristiques en commun:
  • Les modèles facilitent la création du contenu standard et réutilisable dans GPSC (par exemple, les questions, les clauses, les critères de risque, etc.).
  • Un modèle qui n'est pas associé avec un programme a | A nom (par exemple, Modèle de clause | A).
  • Lorsqu'un modèle est associé avec un programme, un enregistrement est créé dans une entité d'intersection. (par exemple, Modèle d'évaluation | PG) Alors que les utilisateurs ne consultent généralement pas les enregistrements dans une entité d'intersection, ils sont utiles pour l'établissement de rapport.
  • Les modèles de programme identifient les données (éléments du modèle) qui seront copiés aux nouveaux projets dans le programme.
    • Les éléments de modèle sont les questions ou les catégories et les éléments ou les clauses, etc. selon la caractéristique.
    • Les éléments de modèle sont copiée à un projet à un moment dans le temps. Une fois qu'ils sont copiés, les changements au modèle ne sont pas mis a jour sur les projets.
    • Le plugiciel clone est le mécanisme pour copier les éléments du modèle aux projets.
    • Un modèle peut être associé à un ou plusieurs programmes.
    • Les éléments de modèle sont un type d'entité d'intersection.
  • Un modèle peut être associé à un ou plusieurs programmes.
  • La date de début et de fin sur les modèles est à titre d’information seulement, aucun logique opérationnelles n'est associé à ces champs.

Remarque :

  • Les fonctionnalités du modèle sont différent des modèles de document , des modèles de transition et des modèles de courriel.
  • Cet aperçu affiche la façon dont les fonctionnalités des entités du modèle sont conçues.
Pratique exemplaire du modèle:
  • Si vous avez besoin de modifier un modèle, mettez l'état du modèle à inactif et créer un nouveau modèle (utilisez le bouton ENREGISTRER SOUS) avec les changements désirés. Cela assurer qu'aucune donnée n'est perdue pour les évaluations de risque déjà terminées. Ne supprimez pas les modèles qui ne sont plus utilisés.
