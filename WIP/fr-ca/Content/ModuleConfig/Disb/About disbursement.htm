À propos de débours

Le GPSC offre des outils axés sur les client pour la gestion des réclamations: les clients peuvent:

  • Présenter les réclamations et les paiements anticipés par un portail.
  • Présenter les réclamations et les paiements anticipés manuellement (par courriel, par la poste or par d'autres moyens). Ils peuvent être entrer manuellement dans le GPSC.
  • Voir les enregistrements qu'ils ont soumis. Tous les changements, ajustements ou examens sont réfléchis dans un examen de débours et non soumis dans les réclamations.

GPSC saisi la facture particulière et les renseignements liés aux coûts avec chaque réclamation et paiement anticipé. Votre organisation peut ajouter des champs comme requis pour un examen de réclamation ponctuel et des processus de paiement.

Les réclamations et les paiements anticipés sont examinés pour déterminer les exigences de paiement.

Examen des débours

Le projet est l'enregistrement parent des examens de débours. Le projet détermine quels éléments sont valides dans le budget du projet, quels sont les flux de trésorerie sont pour un exercice financier donné, quelles sont les conditions qui ont besoin d'être respectées pour qu'un paiement soit effectué et quels modèles à appliquer pour les examens de débours (selon le programme associé).

Principales fonctionnalités des examens de débours:

  • Les agents créent des examens de débours pour valider et traiter les réclamations et les paiements anticipés.
  • Les réclamations peuvent être examiner individuellement ou rassembler pour un examen consolidé.
  • Un flux de processus opérationnel intègre les décisions et automatise les mesures pour les débours qui sont conformes aux processus d'examen et de prise de décision de votre organisation.
  • Les éléments de coût dans une seule réclamation peuvent être approuvée, supprimée, reportée, ajustée ou séparée afin de traiter une réclamation avec plusieurs mesures.
    Par exemple, une réclamation est soumise pour un débours, seulement une partie de la laquelle à été encourue à ce jour. Un agent peut séparer la facture et recommander les éléments encourus qui sont payés et reporter le solde à une date ultérieure.
  • Les éléments reportés peuvent être compris dans les futurs examens de remboursement sans que le client ait besoin de refaire une présentation.

Dans l'examen du débours, les agents peuvent:

  • Gérer les retenues
  • Mettre en œuvre les paiements ou les recalculs à la suite d'une modification à la formule de financement sur le projet.
  • Déclencher les remboursements ou les créances requis pendant la phase active de la contribution.
    Remarque: à configurer localement en fonction des options d'intégration financière de votre organisation.
  • Coder les éléments de réclamations pour des exercices financiers particuliers.
  • Indiquez si le montant du paiement doit être appliqué comme un CAFE précédemment configuré.
  • Réconcilier tous les montants ou les montants partiels des avances non réglées.
  • Émettre de nouveaux paiements anticipés.
  • Créer des paiements négatifs lorsque les paiements en trop ou les avances non réconciliées ont besoin d'être récupéré.
    Remarque: à configurer localement en fonction des options d'intégration financière de votre organisation.
  • Terminer une liste de vérification numérique pour s'assurer de la diligence raisonnable sur l'examen des réclamations. La liste de vérification des débours peut être conçue pour chaque programme et saisi les données de renseignements de piste de vérification requises pour le traitement des réclamations.

Client SC

Le client SC associé offre certains contrôles pour gérer les situations dans l'entièreté du portefeuille du client. Si un client avec de multiples projets a un projet en défaut, ou qu'un recouvrement soit nécessaire, un agent peut suspendre l'enregistrement d'un client SC. Cette suspension est affichées dans tous les projets et les examens de débours du client SC et aucun paiement est possible jusqu'à ce que la suspension soit enlevée.

Automatisation
  • Créer une réclamation ou un paiement anticipé rempli avec les données du client sur le portail peut être automatisé.
  • Créer un document instantané pour la saisie des réclamations du client.
  • Les examens de débours ne sont pas créer automatiquement.
Décisions de conception
  • Les réclamations et les paiements anticipés sont des enregistrements séparés, afin de permettre de sélectionner les enregistrements à traiter dans un examen des débours. Ces décisions peuvent être basées sur des considérations externes (par exemple, exigence assujettie au temps pour que l'avance soit émise).
  • Les calculs des coûts set des paiements ont été conçus dans l’examen de débours selon les tâches de traitement des réclamations communes à chaque ADR.
  • Votre organisation peut modifier les processus opérationnels, les mesures, les transitions et les règles opérationnelles par défaut pour le débours afin de refléter les exigences particulières à l'organisation comme le risque et l'audit.
  • Certaines fonctionnalités de CAFE sont comprises dans le GPSC (par exemple, créer un CAFE d'un examen de débours, indiquer qu'un paiement demandé d'être imputé à un CAFE), mais la gestion et la maximisation d'un CAFE n'est pas inclus.
