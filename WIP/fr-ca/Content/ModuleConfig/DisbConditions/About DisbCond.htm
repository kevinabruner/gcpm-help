À propos des conditions de débours

Un ensemble de conditions de débours peut être copier à chaque projet dans un programme donné, ne associant un modèle de condition de débours au programme. Les conditions de débours peuvent aussi être ajouter au projet manuellement pendant l'évaluation du projet. Les conditions de débours sont identifiées pendant l'évaluation du projet et utilisé pendant la phase d'administration du projet de ces façons:

  • Déterminer si un projet est prêt pour le débours
  • Établir des éléments d'examen périodique
  • Définir les éléments de clôture du débours

Les agents examinent les conditions de débours afin de déterminer si le projet est dans un état prêt pour un débours.

Par exemple; une condition de débours permanente existe pour surveiller les autres sources de financement pour un projet. Au quatrième examen de débours, un agent détermine qu'il n'y a pas suffisamment de preuves pour les fonds de contrepartie. L'agent change le statut à « not disbursable » (N'est pas prêt pour un débours). Dans cet état, le projet ne peut être transféré (un message indique que le projet n'est pas prêt pour un débours).

Remarque: l'utilisation du champs prêt pour un débours dans les transitions peut être modifiée pour satisfaire à vos exigences organisationnelles.

Automatisation
  • Si les conditions de débours sont associés à un programme, les conditions sont copiées aux projets dans ce programme avec la configuration clone.
Outils associés

  • Le statut de débours est associé à la grille de conditions de débours dans le chevron administration du projet.

  • Les conditions préalables au débours ont besoin d'être respecter afin que le statut de débours soit changé automatiquement à prêt pour un débours.
