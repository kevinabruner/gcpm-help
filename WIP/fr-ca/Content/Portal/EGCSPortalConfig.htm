Configuration du portail de la Solution d'entreprise pour les subventions et les contributions (SESC)

Un environnement qui charge EGCS.Portal project are stored as .NET settings in the EGCS.PortalSharePoint.Library project. Any environment which loads EGCS.Portal.SharePoint.Library.dll doit aussi charger les paramètres de configuration du DLL, sans quoi le code de portail rétablira les paramètres compilés par défaut, qui seront presque certainement incorrects.

  • La configuration est typiquement chargée du web.config, or a .config file for the executable which references the EGCS.Portal.SharePoint.Library.dll.
  • Dans tous les cas, le fichier de configuration comprendra un renvoi à EGCS.Portal.SharePoint.Library.Properties.Settings :

<configSections>

    <sectionGroup name="applicationSettings" type="System.Configuration.ApplicationSettingsGroup, System, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" >

<section name="EGCS.Portal.SharePoint.Library.Properties.Settings" type="System.Configuration.ClientSettingsSection, System, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" requirePermission="false" />

    </sectionGroup>

</configSections>

Les paramètres sont stockés dans le format XML. Ils sont considérés comme statiques. Ils sont chargés lorsqu'un environnement d'exécution utilisant EGCS.Portal.SharePoint.Library.dll est initialisé, et ils ne changent pas pendant la durée de vie de l'exécution. L'environnement d'exécution doit être rechargé pour lire les nouveaux paramètres. Il s'agit donc d'exécuter la fonction IISRESET, de recharger le pool d'applications ou d'arrêter ou redémarrer un service ou un exécutable.

Installation

EGCS.Portal les paramètres de configuration sont déployés par un script PowerShell. Le script fera en sorte de copier les paramètres dans tous les emplacements sur le serveur actuel qui en a besoin, ainsi que de réinitialiser ou recharger toute application dépendante.

L'installation de la configuration EGCS.Portal configuration requires a build of the EGCS.Portal solution, which is assumed to be available in a directory “_Build ».

  1. Installer EGCS.Portal configuration, specifying the configuration file to be loaded

    .\_Build\Scripts\Install-Egcs_Config.ps1 .\_Build\Config\<portal name>.egcs.config

  2. Se déploie dans :
  • Application Web SharePoint pour le portail. Les modifications sont écrites dans web.config for the web application, all zones. The portal web application is recycled in IIS.
    Files: C:\inetpub\wwwroot\wss\VirtualDirectories\<folder>\web.config
  • Le service Windows de gestion de documents. Redémarre le service.
    Fichier: C:\Program
    Files\EGCS\EGCS.Portal.Service.DocumentManagement\egcs.config
  • Le service Windows de création de documents. Redémarre le service.
    Fichier: C:\Program Files\EGCS\EGCS.Portal.Service.SiteCreation\egcs.config
  • Le service Web de portail. Recycle le pool d'applications.
    Fichier: C:\inetpub\wwwroot-webservice-http\EGCS.Portal.WebService.Portal\egcs.config

Configuration

Le fichier de configuration se trouve dans le projet EGCS.Portal.SharePoint.Library.

EGCS.Portal.SharePoint.Library\Config\<portal name>.egcs.config

Paramètres

Le fichier de configuration est divisé en plusieurs sections :

Jeton

La section des jetons (« Token ») comprend des paires de noms et de valeurs qui sont substituées dans le reste du fichier. Le modèle de substitution est %Token%. L'utilisation de jetons n'est pas requise; il s'agit simplement d'un raccourci permettant d'éviter de répéter le même texte.

De nouveaux jetons peuvent être ajoutés à la section des jetons si on le souhaite.

Description

DnsNameRdaWebService

Nom DNS du service Web d'agence de développement régional (ADR).

Production : La valeur « 1 » est recommandée (établir une valeur plus élevée si des périodes intensives d'inscription d'utilisateurs sont prévues)

SMSAccountKey

Une clé de compte servant à authentifier le portail par l'intermédiaire du service Swift SMS.

SMSSendRealSms

Lorsque la valeur est « vrai », les messages SMS sont envoyés du portail sans restriction.

Lorsque la valeur est « faux », le portail n'enverra de messages SMS qu'aux numéros de téléphone énumérés dans le tableau GCCF.dbo.WhitelistMobileSMS.

TwoFactorAuthenticationObfuscateAddresses

Le portail devrait-il masquer les adresses de courriel et les numéros de téléphone SMS au moment de les afficher?
Par exemple : jane.user@example.com devient j***********m

UrlClientSiteRoot

Chemin d'accès parent dans SharePoint pour toutes les zones client.
Par exemple : /client

UrlFormEdit

URL pour lancer un formulaire SmartGuide. On ne devrait pas avoir à le changer.

UrlRegister

URL où un utilisateur peut accepter une invitation. Ce devrait être le centre de bienvenue du portail.

UrlSearch

URL pour envoyer les demandes de recherche.

UrlTermsAndConditionsAnonymous

URL pour afficher les conditions pour les utilisateurs anonymes. On ne devrait pas avoir à le changer.

UrlTermsAndConditionsAuthenticated

URL pour afficher les conditions pour les utilisateurs authentifiés.

UrlUserSiteRoot

Chemin d'accès parent dans SharePoint pour toutes les zones personnelles.
Par exemple : /my-moi

UrlWelcome

URL du centre de bienvenue du portail.
