Configuration de GC.GCCF

Un environnement qui charge GC.GCCF solution are stored as .NET settings in the GCFF Library project. Any environment which loads GC.GCCF.Library.dll doit aussi charger les paramètres de configuration du DLL, sans quoi le code de la  FJGC rétablira les paramètres compilés par défaut, qui seront presque certainement incorrects.

  • La configuration est typiquement chargée de web.config, or a .config file for the executable which references the GC.GCCF.Library.dll.
  • Dans la plupart des cas, le fichier .config file will contain a reference to GC.GCCF.Properties.Settings :

<configSections>

<sectionGroup name="applicationSettings" type="System.Configuration.ApplicationSettingsGroup, System, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" >

<section name="GC.GCCF.Properties.Settings" type="System.Configuration.ClientSettingsSection, System, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" requirePermission="false" />

</sectionGroup>

</configSections>

An exception to the above is the case of the GCCF Attribute Store, a DLL loaded by ADFS and which loads GC.GCCF.Library.dll. In this case there is no .config file. When GCCF Attribute Store is registered with ADFS, a limited number of settings are also registered with ADFS and used in place of a .config file. These settings relate to the database connection for the GCCF database and logging. The GCCF attribute store uses a carefully-chosen subset of functions in GC.GCCF.Library.dll which does not require other settings.

Les paramètres sont stockés dans le format XML. Ils sont considérés comme statiques. Ils sont chargés lorsqu'un environnement d'exécution utilisant GC.GCCF.Library.dll est initialisé, et ils ne changent pas pendant la durée de vie de l'exécution. L'environnement d'exécution doit être rechargé pour lire les nouveaux paramètres. Il s'agit donc d'exécuter la fonction IISRESET, de recharger le pool d'applications ou d'arrêter ou redémarrer un service ou un exécutable.

Installation

GC.GCCF les paramètres de configuration sont déployés par un script PowerShell. Le script copie les paramètres dans tous les emplacements du serveur actuel qui en a besoin, et il réinitialise ou recharge toutes les applications dépendantes.

L'installation de la configuration GC.GCCF configuration requires a build of the GC.GCCF nécessite une version de la solution , dont on suppose qu'elle est disponible dans un répertoire « _Build ».

  1. Installer GC.GCCF configuration, and specify the configuration file to be loaded:

    .\_Build\Scripts\Install-GCCF_Config.ps1

  2. Le fichier de configuration déploie ce qui suit :
  • Application Web SharePoint pour le portail. Modifications are written to web.config for the web application, all zones. The portal web application is recycled in IIS.

    Files:
    C:\inetpub\wwwroot\wss\VirtualDirectories\<folder>\web.config

  • L'application Web des domaines communs de la FJGC

    Recycle le pool d'applications.

    Fichier : C:\inetpub\wwwroot_gccf_commondomain\cdc

  • L'application Web des extensions ADFS :

    Recycle le pool d'applications.

    Fichier : C:\inetpub\wwwroot_gccf_adfs\AdfsExtensions

  • Le service Windows de gestion de documents.

    Redémarrage le service.

    Fichier : C:\Program Files\EGCS\EGCS.Portal.Service.DocumentManagement\gccf.config

  • Le service Windows de création de sites.

    Redémarrage le service.

    Fichier : C:\Program Files\EGCS\EGCS.Portal.Service.SiteCreation\ gccf.config

  • Le service Web de portail.

    Recycle le pool d'applications.

    Fichier : C:\inetpub\wwwroot-webservice-http\EGCS.Portal.WebService.Portal\gccf.config

Configuration – Fichier de configuration maître

Le fichier maître de configuration associe les serveurs à des noms d'environnement. Il permet aux scripts d'installation de PowerShell de sélectionner le bon environnement au moment d'installer les composantes GC.GCCF. Le fichier se trouve :

GC.GCF\Configuration EGCS-SESC\Config\config.xml

Ajoutez des lignes à ce fichier pour associer des serveurs ou des domaines à un nom d'environnement. Les lignes sont testées par ordre; le premier nom de serveur ou de domaine correspondant sélectionne l'environnement

Par exemple :

<Map Server="PWHITE-SPS4" Environment="phillip" />

<Map Domain="GsCs.local" Environment="integration" />

<Map Domain="ACOA-APECA.GC.CA" Environment="integration" />

Un script d'installation GC.GCCF exécuté sur le serveur « PWHITE-SPS4 » sélectionnera l'environnement « phillip ».

Un script d'installation exécuté sur un serveur dans le domaine « GSCS.LOCAL” or “ACOA-APECA.GC.CA » sélectionnera l'environnement « intégration ».

Configuration – Fichier de configuration de l'environnement

The environment configuration file is loaded when a PowerShell installer reads config.xml and selects an environment. The Environment Configuration File defines most of the settings needed to install GC.GCCF components for a specific environment.

Le fichier de configuration de l'environnement se situe dans l'emplacement suivant :

GC.GCF\Configuration EGCS-SESC\Config\<environment>.config.xml

La section des jetons (« Token ») définit la plupart des valeurs de configuration qui doivent être changées pour un portail. La section des jetons comprend des paires de noms et de valeurs qui sont substituées dans le reste du fichier, et dans d'autres fichiers de configuration. Le modèle de substitution est %Token%, e.g.: %PortalName%..

De nouveaux jetons peuvent être ajoutés à la section des jetons si on le souhaite.

Configuration – Paramètres d'application gccf.config

Application settings are the .NET settings for the GC.GCCF.Library application. Effectively this is the contents of app.config. This configuration data is copied into SharePoint’s web.config, or becomes the gccf.config file for other executables or web applications.

Ces paramètres définissent les valeurs d'exécution utilisées par le code GC.GCCF

The gccf.config file is located here: GC.GCF\Configuration EGCS-SESC\Config\GccfLibrary\<environment>.gccf.config
