Base de données de la fédération des justificatifs du gouvernement du Canada (FJGC)

La base de données de la FJGC est une base de données de SQL Server qui sert de référentiel au portail.

  • Il ne s'agit pas d'une base de donnée SharePoint; cette base de données n'est utilisée que par les composantes du portail.
  • Elle devrait être sauvegardée en même temps que les bases de données SharePoint.
  • Elle ne stocke pas le contenu des zones client, des zones de projet ou des zones personnelles elles-mêmes. Ces zones sont des collections de sites SharePoint (SPSite) and sites (SPWeb) que l'on stocke dans des bases de données de contenu SharePoint.
  • Une liste complète des tables est documentée ci-dessous. At a high level, the GCCF database stores:
    • Location of Client, Project, and Personal Zones
    • Mappings of GCCF Users between their GCCF identity (PAI) and their portal identity (PortalUserId)
    • Profils d'utilisateur de la FJGC
    • Jetons d'authentification à deux facteurs
    • Conditions
    • Invitations
    • Configuration de la page maître
    • Registre des documents  (les documents eux-mêmes sont stockés dans  SharePoint)
    • Registre des messages sécurisés (les messages sécurisés eux-mêmes sont stockés dans SharePoint)
    • Fichiers de journal

Sécurité et rôle

Access to read and write GCCF database tables is performed exclusively through stored procedures. Users should belong to the GCCFAdfs or GCCFUser roles in the GCCF database to execute portal stored procedures. These roles are created by the installation script GCCF Database.security.sql in the GC.GCCF solution.

Comptes de service utilisés par le portail qui devraient appartenir au rôle GCCFAdfs :

  • Compte de service Active Directory Federation Services (ADFS)
  • Comptes de service pour le domaine commun de la  FJGC et les applications Web ADFS (il s'agit habituellement d'un compte de service ADFS, mais ce n'est pas requis)

Comptes de service utilisés par le portail qui devraient appartenir au rôle GCCFUser :

  • Compte de service SharePoint
  • Compte de service pour le service Web de portail  (habituellement le compte de service SharePoint)
  • Compte de service pour les services Windows de création de sites et de gestion de documents (habituellement le compte de service SharePoint)

Conditions préalables:

  • Une version de la solution GC.GCCF solution, which is assumed to be available in a directory _Build.
  • Serveur  SQL. Le serveur SQL utilisé par SharePoint est acceptable.
  • Une liste de comptes de service qui ont besoin d'un accès à la base de données de la FJGC. Voir la section « Sécurité et rôles » ci-dessus.

Installation

  1. Ouvrez une session dans SQL Server avec un compte DBA.
  2. Créez la base de données.
    • Edit file _Build\Database\GCCF Database.sql
    • Edit the FILENAME for the mdf and ldf file to place the files where desired, on the server.
    • Adjust size and growth parameters for the database if required.
    • Execute _Build\Database\GCCF Database.sql to create the database.
  3. Remplissez les tables de code.
    • Exécuter _Build\Database\GCCF Database.codetables.sql
    • Si les définitions utilisées par le ministère ne correspondent pas aux définitions de site utilisées par le portail d'intégration, les rangées dans MasterPageTab (onglet de la page maître) devront être mises a jour.
  4. Configurez la sécurité.
    • Non scripté : accordez l'accès aux comptes de service à SQL Server s'il ne l'a pas  (les comptes  SharePoint et les comptes de service ADFS y auront accès à l'aide de leur propre programme d'installation). Aucun rôle précis n'est requis pour le serveur, outre le rôle public (ne changez pas de rôles pour  ADFS ou SharePoint
    • Modifier le fichier_Build\Database\GCCF Database.security.sql
    • Create users in the GCCF database for all service accounts.
    • Assign service accounts to the roles GCCFAdfs or GCCFUser.
    • Execute _Build\Database\GCCF Database.security.sql
  5. (Facultatif) Chargez les conditions.
    • Cette étape est requise si le portail utilisera des conditions.
    • English and French HTML text must be loaded into table TermsAndConditions.
    • See example in GCCF Database.demodata.sql in the GCCF Library project.
  6. The contents of the FormDefinition table need to be updated to match forms offered by the department. FormName must match the SmartGuide name of the form.
