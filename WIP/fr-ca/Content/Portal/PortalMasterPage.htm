Portail - aperçu de la page maître

Le EGCSPortalMasterPage Projet est composé de deux Pages maître; une série de Contrôles Web; un Dossier de mises en page pour garder les biens du site, y compris les images, JavaScript et les fichiers CSS; et une classe d'aide générique C#.

Les traductions sont gérées en utilisant le gestionnaire de ressource du projet EGCS.Portal.SharePoint.Library.

Fichiers maîtres

The two Master Files included are WET4Canada.master and WET4Search.master. Both of these master pages have the same code behind class, WET4PublishingMaster.cs. The differentiator is that the WET4Search master page has a left navigator added to it in order to support search result refinement in the Portal Search Center.

Contrôles Web

Pour concrétiser le contenu qui dépend grandement de la logique, les Pages maître dépendent des Contrôles Web. Les Contrôles Web de la Page maître du portail comprennent les contrôles suivants :

Contrôles Web pour les pages maître

Contrôle Web

Détails

*BottomBar.cs

Il concrétise le bas, la « Barre d'onglet » en bleu clair de l'entête de la Page maître. Il contient une série d'onglets tirés de la Base de données qui reflète les pages disponibles actuellement dans l'étendue du portail. Cliquer sur un onglet redirige l'utilisateur à cette page

JsScript
Paramètres
     fil Nom

Il télécharge le script fourni (Nom == emplacement relatif du script) dans la page maître

Langue

Il concrétise le Lien pour basculer la langue, ajoute une fonction JavaScript à la page maître laquelle définit le cookie LCID à la valeur de la langue vers laquelle la page est basculée (la valeur LCID est une configuration qui peut être changée).

*Menu déroulant Message
  • Il génère un icône de Lettre à partir du menu déroulant Message dans la partie du haut de la rangée blanche dans l'entête de la page maître.
  • Les contenus dans le menu déroulant comprennent, Message personnel, Message au client si vous vous situez dans une zone client ou de projet, et les comptes de message non lu du projet et redirige les URL à chaque centre de messagerie respectif.
  • L'icône de message possède aussi un cumul qui fait le total de messages non lu provenant de chaque zone.

*Barre centrale
  • Il concrétise le bas, la barre en bleu foncé de l'entête de la Page maître.
  • La principale fonction de la Barre centrale est la navigation par les Zones du portail.
  • La Barre centrale contient toujours un lien au centre d'accueil ainsi qu'un Créer menu déroulant afin de créer des messages et des formulaires (si l'utilisateur est connecté).
  • Si l'utilisateur est connecté comme utilisateur du FJGC, ils voient aussi une liste de leurs clients disponibles.
  • Si un utilisateur est dans un zone de client, ils voient aussi une liste des projets disponibles pour ce client.

Gestionnaire des scripts

Il concrétise les contrôles du côtés serveur pour la boîte de recherche avec une requête de leur entrée.

SearchCrawlExclusion

 

*Menu déroulant utilisateur

WETSiteMap

 

*Contrôles Web qui concrétisent les  HTML qui sont créé en utilisant un StringBuilder et des valeurs de fils constantes avec des éléments fictifs. Cela permet à un montant dynamique d'élément HTML d'être concrétisé à la page selon les variables, y compris la portée actuelle, le type d'utilisateur connecté (le cas échéant)m le nombre de clients ou de projets disponibles, le nombre de pages disponibles pour la portée actuelle, etc.

Dossier des mises en page.

Le Dossier des mises en page contient des biens statiques qui sont utilisés par la page maître. Cela comprend les images, CSS et JavaScript. Ils sont catégoriser dans des sous-dossiers à des fins d'organisation.

À des fins de marquage, la plupart des étiquettes et des changements textuels peuvent être effectués en trouvant l'étiquette appropriée dans le fichier EgcsTranslation de la bibliothèque du projet . Certaines étiquettes (comme les messages d'erreur dans les dialogues) et les images se retrouvent dans le dossier des MasterPage Layouts (Mises en page de la Page maître). Pour de plus amples renseignements, consultez Personnalisation du portail
