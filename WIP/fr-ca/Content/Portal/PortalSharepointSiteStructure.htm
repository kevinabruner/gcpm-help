SharePoint - structure du site du portail

Le portail est une application Web de SharePoint, et par conséquent tout le contenu dans le portail est composé des collections de site (SPSite) ou sites (SPWeb) de SharePoint. Certains de ces sites sont créé automatiquement par les services de portail et quelques-uns de ces sites sont créés par un administrateur SharePoint au moment que le portail est installé.

Remarque:

Deux items dans le bagage des priorités pour un site qui mène la fonction de la  Page maître :

  • PortalScope depends on the property bag for the current web it is being accessed in. Only one of these property values will exist for a given SPWeb.
    • <SPWeb>.AllProperties[“egcs_PortalUserId”] (Personal zone)
    • <SPWeb>.AllProperties[“egcs_AccountNumber”] (Client zone)
    • <SPWeb>.AllProperties[“egcs_FCProfileID”] (Project zone)

  • Pour déterminer l'information de l'entête :

    • SPWeb>.AllProperties["MasterPageHeaderScope"]= ‘Welcome’, ‘Client’ ou ‘Project’

  • Une page pour l'authentification à deux facteur laquelle héberge la partie Web

Notes
  • Un ministère peut souhaiter créer des variations de site de  SharePoint sous la racine anonyme afin d'aider l'hôte ou d'autre contenu particulier à la langue. par exemple,
    /en, /fr

  • Les pages statiques qui rapporte « Erreur », « Connexion réussie », « Déconnexion échouée », etc. peuvent aussi être hébergé sur les  Variations de sites. Ces URL de page sont configuré dans gccf.config.

  • Si le ministère souhaite mettre en œuvre leur propre page Sélecteur, elle peut être hébergée sous les Variations de sites.

Zones client
Site du portail de la Zone client

Item

Description

But / contenus
  • Il représente un Client SC publié sur le formulaire du GPSC.
  • Il affiche les renseignements de profil d'un Client SC.
  • Il présente des demandes pour le client.
  • Il envoie et reçoit des messages sécurisés pour le Client SC.
  • Il partage des documents avec l'ADR.
  • Il invite et gère l'accès des autres utilisateurs au client.
  • Il contient toutes les Zones projet pour le client.

URL typique

 /client/<Account Number>

Permissions
  • Il est restreint aux utilisateurs de la FJGC qui ont été particulièrement invité au client.
  • Le ministère peut choisir de donner automatiquement l'accès aux groupes des membres du personnel (utilisateurs de l'ADR).
  • La sécurité est mise en œuvre par les groupes de sécurité et les rôles SharePoint.

Objet de SharePoint

Collection de sites (SPSite)

De quelle façon est-il créé?
  • Automatiquement, par le service Windows pour la création de site.
  • Lorsqu'un membre du personnel de l'ADR choisi de publier un client sur le portail, le portail reçoit une notification (par le service du portail Web).
  • Une rangée est ajoutée à GCCF.dbo.SiteQueueAccount.
  • Le service de création de site crée périodiquement tous les nouveaux sites de Client SC dans la file d'attente.

Applications
  • egcs-portal-share-point-spfx-project-client
  • spfx-portal-user-solution
  • spfx-message-centre-solution

Pages et parties Web
  • default.aspx hosts DraftSub and AppStatus web parts.
    Il permet aux utilisateurs d'examiner les ébauches de soumissions et de demandes lesquelles ont été présentées.
  • message.aspx hosts SecureMessageCentre web part.
    contrôle sécurisés de la messagerie.
  • members.aspx hosts UserManagement web part.
    Il examine les utilisateurs qui ont accès au Client. Les utilisateurs avec le privilège administrateur peuvent le rôle des autres utilisateurs ou les enlever du site.
  • invitation.aspx hosts siteInvitation web part.
    Il invite les nouveaux utilisateurs de la FJGC au client.
  • profile-profil.aspx hosts ClientProfile web part.
    Il affiche les renseignements de profile à propos du client.

Notes
  • The root location for client sites is configured in EGCS.Portal Configuration – UrlClientSiteRoot
  • Les noms de page doivent correspondre à la définition de la portée du client dansGCCF.dbo.MasterPageTab
Zones projet
Site du portail des Zones projet

Item

Description

But / contenus
  • Il représente un projet (FCProfile) publié sur le portail.
  • Il affiche les renseignements de profile à propos d'un projet.
  • Il présente des réclamations, des paiements anticipés et des rapports de rendement pour un projet.
  • Il envoie et reçoit des messages sécurisés pour un projet.
  • Il partage des documents avec l'ADR.
  • Il affiche le résumé du débours et les renseignements du budget à propos d'un projet.

URL typique

/client/<Account Number>/<FCProfileID>

Permissions
  • Hérité de la Zone client.
  • Toutes les zones projet pour un client partager les permissions de zone (tous les utilisateurs invités au client ont le même niveau d'accès à tous les Projets pour le client).

Objet de SharePoint

Site (SPWeb)

De quelle façon est-il créé?
  • Automatiquement, par le service Windows pour la création de site.
  • Une nouvelle zone projet est ajoutée à un client lorsqu'un nouveau projet (FCProfile) est créé dans le GPSC

Applications
  • egcs-portal-share-point-spfx-project-client
  • spfx-message-centre-solution

Pages et parties Web
  • default.aspx hosts DraftSub web part.
    Il permet aux utilisateurs d’examiner les ébauches de soumissions
  • message.aspx hosts SecureMessageCentre web part.
    Il sécurise le contrôle de la messagerie.
  • summary-sommaire.aspx hosts web parts BudgetSummaryand DisbursSummary. Il affiche le résumé des postes budgétaires du budget et le résumé des postes budgétaires du débours, respectivement.
  • repaymentsched-calendrierderemb.aspx hosts Repayment Schedule.
    Le calendrier de remboursement pour le projet.

Notes

Les noms de page doivent correspondre à la définition de la portée du projet dansGCCF.dbo.MasterPageTab
Zones personnelles
Site du portail des Zones personnelles

Item

Description

But / contenus
  • Une Zone personnelle d'utilisateur sur le portail.
  • Il présente des demandes a l'ADR.
  • Il envoie et reçoit des messages sécurisés pour le projet.
  • Il partage des documents avec l'ADR.

URL typique

 /my-moi/<Guid>

Permissions
  • Il est restreint à un seul utilisateur de la FJGC.
  • Le ministère peut choisir de donner automatiquement accès aux groupes de membre du personnel (utilisateur de l'ADR).
  • La sécurité est mise en œuvre par les groupes de sécurité et les rôles SharePoint.
  • Il n'y a pas de capacité d'inviter d'autres utilisateurs de la FJGC dans cette zone.

Objet de SharePoint

 Site (SPWeb)

De quelle façon est-il créé?
  • Automatiquement, par le service Windows pour la création de site.
  • Le service maintiendra une file des Zones personnelles non-attribuées dans le tableau de la base de données GCCF.dbo.SiteQueueUser.

  • Les nouvelles Zones personnelle seront automatiquement créées comme zones sont enlevées de la file d'attente.

  • La taille de la file est établie dans EGCS.Portal Configuration – SiteCreationUserSiteQueueSize.

Applications
  • egcs-portal-share-point-spfx-project-client
  • spfx-message-centre-solution

Pages et parties Web
  • default.aspx hosts DraftSub and AppStatus web parts.
    Il permet aux utilisateurs d'examiner les ébauches de soumissions et de demandes lesquelles ont été présentées.
  • message.aspx hosts SecureMessageCentre web part.
    Il sécurise le contrôle de la messagerie.

Notes
  • The root location for personal sites is configured in EGCS.Portal Configuration – UrlUserSiteRoot
  • Les noms de page doivent correspondre à la définition de la portée Accueil dans GCCF.dbo.MasterPageTab (where PageUrlScope = ‘Personal’)
  • <Guid> for a Personal Zone is randomly chosen.
