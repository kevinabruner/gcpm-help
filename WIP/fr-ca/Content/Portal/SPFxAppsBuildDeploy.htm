Bâtir et déployer des applications SharePoint Framework (SPFx)

Préparation du serveur SharePoint :

  1. Installe le CU de mars 2018 pour SharePoint 2016 ou un plus récent (ce CU contient les correctifs nécessaire pour SPFx). Il est grandement recommandé d'installer le CU actuel de SharePoint 2016 pour les derniers correctifs à la sécurité.

  2. S'assurez que les Paramètres de l'inscription et que les services de Gestion des applications ont été approvisionnés à la ferme du portail de SharePoint.

  3. Suive les étapes dans l'article suivant pour Gérer le catalogue des applications dans le Serveur SharePoint. Effectuez les étapes dans « Configure the App Catalog site for a web application » (Configurer le site du catalogue des application pour une application Web) (les autres sections n'ont pas besoin d'être réalisés).
  4. La documentation du portail suppose que le Catalogue des application sera créé à l'URL suivant :


    https://<portal>/sites/AppCatalog

    See Manage the app catalog in SharePoint Server*.

    Remove JSON as a restricted file type for SharePoint. This is required in order for SPFx Apps to deploy JSON files for web parts.
    $wa = Get-SPWebApplication https://<portal>
    $wa.BlockedFileExtensions.Remove("json")
    $wa.Update()
    See Set up your development environment* for further information.

Créer des applications SPFx

Les étapes pour créer sur effectués sur une machine de développement.

  1. Installer la version 6.11.5 du Node.js (n'installez pas la dernière version - elle ne fonctionne par avec SPFX) node-v6.11.5-x64.msi

    See: https://nodejs.org/en/blog/release/v6.11.5/

  2. Exécutez “Node.js command prompt” en tant qu'administrateur
  3. CD dans le dossier de projet SPFx

    cd ~\GCPM-Portal\EGCS.Portal\EGCS.Portal.SharePoint\<SPFX Project Folder>

  4. (On first build, not required for subsequent builds) Run the following commands:
    npm install
    npm install -g yo gulp
    It may take some time for these installs to complete. These packages are not checked in to TFS and it should never be.
  5. Run Build.bat to build. The SPFx App package and JavaScript files will be copied to the _Build folder for the solution (~\GCPM-Portal\EGCS.Portal\_Build\SPFx)

Déployez les Applications SPFx et les fichiers Javascript

Les étapes du déploiement sont effectuées sur le serveur du Portail SharePoint et par le navigateur Web.

  1. Dans un navigateur Web, vous connectez dans le Portail comme Administrateur SharePoint.
  2. Naviguez vers le Catalogue des applications pour le serveur :

    https://<portal>/sites/AppCatalog/AppCatalog/Forms/AllItems.aspx

  3. Téléchargez ou glissez et déposez chaque ensemble application de _Build\SPFx a u Catalogue d'application dans le navigateur Web (ce sont les fichiers avec l'extension sppkg). Cela enregistrera l'application dans le catalogue.

    Appuyez sur « Trust » (De confiance) dans la case d'alerte

  4. If it does not exist, create a document library “/cdn” on the Portal server.

    The root site collection is assumed to be Anonymous, which means that all users will automatically have access to any SPFx JavaScript files placed in /cdn.

    $web = Get-SPWeb https://<portal>

    $web.Lists.Add("cdn", "cdn",

    [Microsoft.Sharepoint.SPListTemplateType]::DocumentLibrary)

  5. Deploy SPFx App JavaScript and JSON files to https://<portal>/cdn

    To deploy JavaScript for all Apps:

    .\_Build\Scripts\Deploy-SPFxJavaScript.ps1

    To deploy JavaScript for one specific App:

    .\_Build\Scripts\Deploy-SPFxJavaScript.ps1 -TargetFolder Auth

    (Here ‘Auth’ is the name of a specific JavaScript folder for the App being redeployed, which is different than the name of the App.)

  6. Ajouter une Application a une collection de sites :
    • Naviguez vers le site désiré en utilisant un administrateur SharePoint ou un compte qui des droits d'administrateur pour la collection de sites. Il faut noter que le Compte système de SharePoint ne peut pas installer des applications (le compte de service pour le portail).
    • Sélectionnez l'icône d'engrenage sur la barre de navigation supérieure à la droite et sélectionnez « Ajouter une Application »
    • Cliquez sur l'Application désirée afin de l'installer.
    • Attendez que l'Application soit installer.
  7. Pour ajouter une partie web SPFx à une page ASPX :
    • Naviguez à la page dans laquelle vous voulez afficher la partie Web.
    • Modifiez la page.
    • Sélectionnez Ajouter partie Web.
    • Sélectionnez la partie Web désirée du groupe « GCPM Portal - portail GPSC ».
    • Cachez le chrome pour les parties Web :

      .\_Build\Scripts\Disable-WebPartChrome.ps1 https://<portal>/<pageUrl>.aspx

Mettre a jour les Applications SPFx

Le SharePoint Framework annexe un identifiant unique à chaque fichier JS pour changer la cache du navigateur lorsque l'application change (identifiant change avec chaque version peut importe la version de l'Application).

Il y a deux façon de mettre à jour les Applications du SharePoint Framework, un développeur devrait choisir la façon selon le besoin du scénario à réaliser.

Scénario 1 (Mise a jour simple de JS)

La mise à jour de JavaScript pour le changement de logique, le changement de CSS, les fils de langue ou tout autre changement simple. Dans ce cas la création de l'application et le remplacement des fichiers JS et de l'ensemble de le de fichier sera assez pour refléter les changements dans l'ensemble de la ferme.

  • Créer l'application.
  • Redéployer le fichier sppkg au Catalogue d'application.
  • Redéployer application de JavaScript.

Scénario 2 (Mise à jour du manifeste de l'Application)

Toute mise a jour qui nécessitera un changement dans le fichier de manifeste de l'Application aura besoin d'augmenter la version de l'Application. Lorsqu'une Application augmente sa version, une tâche programmée de SharePoint enverra le nouveau manifeste a tous les emplacements où l'application est installée. La version de l'Application peut être changée dans l’ensemble de fichier dans le dossier de configuration de l'application dans la solution.

  • Créer l'application.
  • Redéployer le fichier sppkg au Catalogue d'application.
  • Redéployer application de JavaScript.
