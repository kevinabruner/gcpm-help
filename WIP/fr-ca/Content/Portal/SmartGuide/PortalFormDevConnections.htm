Élaborer des formulaires de portail - connexions

Il y a deux types de connexions configurées pour le GPSC : les fonctions d'extension d'API e les appels de services Web de SharePoint.

Fonctions d'extension

Les fonctions d'extension sont utilisées dans l'ensemble de plusieurs formulaires et sont comprises dans le tableau ci-dessous.

Les fonctions d'extension d'API pour les connexions de portail

Fonction d'extension

Description
SmartletSerializer

 

Cette fonction se produit lors de l'enregistrement ou la soumission du formulaire.

Il évolue par le formulaire et sérialise les champs d'entrée et les métadonnées.

Le nœud racine est nommé d'après le nom interne du Smartlet avec ses nœuds enfants qui sont des métadonnées et toutes les entités qui existent sur le formulaire.

Les Métadonnées enfants sont les champs qui sont indiqué comme étant possible de sérialiser, mais qui ne suivent pas la convention d’appellation entity.field.

Les nœuds enfants ont leurs champs comme enfants directs. Le fil  XML sérialisé est conservé dans le champ de configuration nommé « xml »

GeneratePDF

Cette fonction se produit à la Soumission du formulaire. Il génère le fil codé based64 de la représentation PDF du formulaire et de l'entrée de l'utilisateur. Les contenus du PDF sont déterminés par ce qui se trouve dans le champ de récapitulation des données sur la page Examen.

Vous pouvez utiliser la convention d'appellation suivante pour changer quelles sont les pages qui concrétisent le PDF :

byte[] bytes = smartlet.generateFile(Constants.FileType.PDF, $"P{page1Id}}:PrintOut |P{page2Id}:PrintOut");

La convention d'appellation est ‘P’ + IdOfPageToAdd + ‘:’ + ‘NameOfTemplate’. Toutes les pages additionnelles suivent la même convention d'appellation avec un caractère de pipeline « | » entre le nom du modèle et le caractère « P ».
InitializeRequestParameter

Il tire les données des paramètres de requête et définit leurs champs de configuration correspondants.

Il se produit à l'initialisation du Smartlet.
InitSmartlet

Il se produit au chargement du formulaire, il appelle le service WebLoadDraft webservice and SmartletDeserializer .

Les autres appels à la fonction d'extension et au service Web peuvent être fait de façon à être plus dépendant du formulaire.
ImportCsv

Il permet à l'utilisateur de télécharger un modèle offert, le remplier avec les données et le télécharger sur le formulaire afin de créer des rangées dans un groupe répétitif.

Deux paramètres, le nom du champ de téléchargement du SmartGuide et le groupe répétitif ont besoin d'être fourni.

Remarque :

  • L'utilisateur doit téléverser un CSV avec les entêtes qui correspondent aux étiquettes des champs du groupe répétitif.
  • Le formulaire ne se soucie pas si la langue du modèle correspond à la langue actuelle du SmartGuide (peut téléverser le modèle Anglais dans le contexte du formulaire Français et vice versa) par contre, il suppose que l'entrée correspondra à la langue CSV - ce qui signifie que les devises suivront les formats français dans le modèle français s'ils utilisent un formatage de la devise.
  • Les groupes répétitifs qui ont des champs déroulant ou à sélection ou les champs calculés seront aussi ignoré et ne seront pas nécessaire dans le modèle CSV.
Appels au service Web de SharePoint
  • SharePointConnections: il contient tous les appels au service Web que le formulaire réalise.
  • Tous les appels de service Web à CRM sont effectués par le service Web SharePoint. Pour de plus amples renseignements, consultez Les formulaires dynamiques du portail

Lors de l'ajout de nouveaux services Web, il est important de comprendre les entêtes suivantes ainsi que leurs valeurs par défaut.

Entête Valeur

Type de contenu

application/json;odata=verbose

Cookie

${requestHeader("Cookie")}$

Référant

${"https://" + requestHeader("Host") + "/"}$

Lors de l'ajout d'une URL au service Web, inclure le paramètre {Client-Site} set to the configuration variable site.
