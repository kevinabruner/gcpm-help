Solution GC.GCCF de Visual Studio

The GC.GCCF solution deals with connectivity to the GCCF environment and user identity. Cela comprend un code d'extension pour SharePoint et ADFS pour se conformer aux dispositions de la norme CATS2.

Conditions préalables:
  • Visual Studio 2017
    • Développement .NET Desktop
    • Développement web et ASP.NET
    • Stockage et traitement des données
    • Développement Office/SharePoint
  • Pour construire la solution, ce n'est pas nécessaire d'installer SharePoint.

Pour construire la solution GC.GCCF :

  1. Récupérer la dernière version de TFS.
  2. Nettoyer la sortie de build antérieure :
    • Ouvrez une fenêtre PowerShell comme administrateur.
    • CD au répertoire pour la solution GC.GCCF.
    • Exécutez Clean-Solution.ps1.
  3. Créer la solution :
    • Dans Visual Studio, cliquez à droite sur le noeud solution (au haut de l'arbre Solutions Explorer)
    • Sélectionnez Déployer.
    • Un build réussi signalera :

      ========== Build : 10 ont réussi, 0 ont échoué, 0 mise à jour, 0 ignoré ==========

      ========== Déployer : 2 ont réussi, 0 ont échoué, 0 ignoré ==========

Remarque:

  • Il arrive parfois que Visual Studio ne construise pas l'une des solutions SharePoint, affirmant que le « projet n'a pas été construit parce qu'il s'agit d'un projet interdépendant ». Cela semble être un bogue dans Visual Studio. Relancez la construction, le processus devrait fonctionner.
  • Un dossier nommé _Build comportera toutes les sorties liées à la construction. Ce dossier est créé en suivant les étapes présentées dans la section "Build Events” (Construction d'événements) de chacun des projets.
  • Si le code de la bibliothèque du GPSC ou des projets de connecteur du GPSC a été modifié, copiez les DLL de sortie dans le projet lEGCS.Portal project. This process can be automated by putting a file name _Refresh_EGCS_DLL.txt into the GC.GCCF.
  • Vérifiez le “Build Events” (Construction d’événements) de ces deux projets pour voir un exemple de la façon de configurer cela.

GCCF ADFS Extensions

Application Web IIS
  • Une application Web qui prolonge les services ADFS afin d'inclure la fermeture de session avec lien SAML SOAP et la fermeture de session utilisant le canal arrière.
  • Habituellement installé sur le serveur ADFS.
  • Ce serveur du proxy inverse permet l'affichage de cette application Web dans l'espace pour l'URL de ADFS. Le trafic lié au /adfs/ls/gccf/* est acheminé vers l'application Web au lieu d'être traité par l'ADFS.
  • Utilise le OpenAM community Edition pour envoyer et recevoir des messages SAML SOAP.

GccfAttributeStore

DLL
  • Magasin d'attributs ADFS.
  • Il s'agit d'un DLL personnalisé chargé par ADFS. Il est invoqué au moyen de l'ouverture de session par un utilisateur pour schématiser le Persistent Anonymous Identifier (identificateur anonyme persistent) du GPSC en fonction de son PortalUserId.
  • Entreprose les données dans le tableau de base de données GCCF.dbo.GccfPai.

OpenAM Community Edition

DLL
  • Une bibliothèque de source ouverte pour SAML écrite en C#.
  • prolongée et personnalisée en fonction des exigences du portail.
  • Fondée sur ForgeRock OpenAM Community Edition Fedlet.

DLLs

Dossier de solutions Visual Studio

Ce dossier comporte les DLL de SharePoint et de ADFS nécessaires pour bâtir toutes les composantes de la solution GC.GCCF.

GCCF Connector

Solution de batterie SharePoint 2016
  • Pages Web pour gérer la chaîne de redirections dans le cadre des authentifications d'utilisateurs pour le GCCF afin d'assurer la réalisation du traitement du cookie de domaine commun, d'assurer que le cookie de la session ADFS est copié et que l'ADFS est invoqué avec les arguments adéquats pour la connexion avec une CléGC ou SecureKey.
  • Ajoute plusieurs pages au cheminement de serveur /_layouts/15/gccf/. La plupart de ces pages n'ont pas d'interface utilisateur. Elles effectuent un traitement côté serveur avant de rediriger le navigateur.
  • La langue du module HTTP lit le cookie de langue du portail et détermine la langue d'interface d'utilisateur de SharePoint.
  • HTTP Module SLO monitors web requests for user sessions which need to be terminated because a back channel logout notification has been received.
  • surveille les demandes Web afin d'assurer que tous les utilisateurs ont accepté les modalités du portail, ont enregistré leur profil et ont effectué une authentification a deux facteurs.HTTP Module UserRegistration Toute demande Web ne respectant pas ces conditions sera redirigée sur la page Web approprié afin d'achever la tâche.
  • Code de bibliothèque pour encoder les cookies.
  • Code de bibliothèque pour identifier le type d'utilisateur (utilisateur du FJGC ou utilisateur d'une ADR).
  • La sortie de la DLL du projet est indiquée a titre de référence dans le EGCS.Portal solution.

GCCF Web Pages

Solution de batterie SharePoint 2016
  • Mise en oeuvre des références aux pages simples pour l'affichage de l'ouverture de session, la fermeture de session et les messages d'erreur.
  • Fichier de traduction (fichiers RESX .NET) pour les composantes GC.GCCF.
  • Mise en oeuvre des références aux pages de sélecteur GCCF, qui permettent aux utilisateurs de faire un choix entre une CléGC ou SecureKey.
  • Mise en oeuvre des référence à la page de réinscription des utilisateurs. Cela permet aux utilisateurs de demander le rétablissement de leur compte de portail s'ils ont perdu ou oublié leur CléGC ou SecureKey ou s'ils souhaitent transférer leur compte dans une autre CléGC ou SecureKey.

Entrust 9.3

Dossier de solution
  • Fichiers installés en vue d'Entrust 9.3.
  • Entrust est utilisé pour effectuer des vérifications CRL des ADFS des certificats ICM.
  • Comprend les fichiers de configuration installateur Entrust pour CATE de GJI (pré-production) et les environnements de production.

Microsoft ARR

Dossier de solutions
  • Fichiers installeur pour Microsoft ARR.

Metadata Rewriter for ADFS

Application de formulaires Windows
  • Lecteur des métadonnées ADFS et production d'une trousse de métadonnées approuvées convenable pour la FJGC.
  • Une autre façon de procéder est d'utiliser le script Update-AdfsMetadata.ps1 PowerShell pour resigner une trousse de métadonnées de portail (et peut-être en modifier le nom DNS).

SAML Decrypt Utility

Application de formulaires Windows
  • Décrypter les réponses SAML de la CléGC ou SecureKey. Ces réponses sont habituellement saisies à partir de Fiddler ou d'un service similaire.
  • Exige qu'un certificat de chiffrement ICM, comportant une clé privé, soit sur l'ordinateur local et sont accessible à l'utilisateur chargé d'exécuter le service.
  • Code très vieux et inadapté présentant plusieurs failles. Cette partie de la trousse n'est pas officielle ou soutenue. Utilisez à vos propres risques.

Configuration EGCS-SESC

Projet de l DLL

La DLL de ce projet n'est pas utilisée. Le projet est un conteneur comportant les fichiers de configuration.

  • Fichiers de configuration pour l'installation du portail.
  • Paramètres de configuration pour différente composantes GC.GCCF .
  • Les paramètres de configuration pour les portails multiples sont maintenus dans un environnement séparé.
  • Métadonnées actuelles pour CléGC et SecureKey, en pré-production et en production.
  • Le certificat racine 1CA-AC1 pour la préproduction et la production.
  • Extraits des ensembles de règles de Microsoft ARR.
  • Le certificat de signature ICM du portail (clé publique uniquement).
  • Thème Web de l'ADFS pour soutenir le portail.

GCCF Common Domain Cookie

Application Web IIS
  • Rédige le cookie de langue du domaine commun de la FJGC pendant l'ouverture de session afin de communiquer la langue souhaitée de l'utilisateur à CléGC et SecureKey.
  • Lit le cookie de langue du domaine commun GCCF après l'ouverture de session afin de communiquer la langue souhaitée de l'utilisateur au portail (en cas de changement).
  • Publie sous le nom DNS assigné au portail dans le domaine fjgc-gccf.gc.ca
  • Habituellement installé sur le serveur ADFS.

GCCF Library

DLL
  • Une bibliothèque DLL qui est utilisée par la plupart des autres composantes de GC.GCCF and EGCS.Portal. et
  • comporte tous les paramètres .NET utilisés dans l'autre code GC.GCCF.
  • En incluant une bibliothèque de compression sous format .zip de tiers et une bibliothèque de tiers afin d'accéder aux serveurs Memcached.
  • Méthodes de connexion centralisée.
  • Méthodes d'accès à la base de données centralisées.
  • La sortie DLL du projet est citée à titre de référence dans la solution EGCS.Portal.
