Aperçu du service Web FormManager

La classe des services Web FormManager renferme plusieurs services Web qui soit communiquent avec le portail, soit à CRM par les appels de service Web des ADR à partir de formulaires SmartGuide.

Des instructions sur l'ajout à SmartGuide de références aux services Web se trouvent à Conception de formulaires pour le portail avec SmartGuide.

Services Web FormManager utilisés dans les formulaires SmartGuide
Services Web utilisés dans les formulaires SmartGuide

Service Web

Paramètres

Retours

Description

LoadDraft

chaîne ReferenceNumber,

chaîne XML

Sur initialisation Smartlet, interroge le document SubmissionSoumission de l'instance actuelle de l'écran établi pour XML pour désérialiser et remplir dans des champs Smartlet.

SaveDraft

chaîne ReferenceNumber,

chaîne FormXml,

chaîne FormPdfBase64,

bool IsSubmit

résultat bool

On Save button click or master page language toggle sends the serialized FormXml and referenceNumber to the current form instance’s SubmissionSoumission document set. FormPdfBase64 is blank and IsSubmit is set to false.

Résultats retournés déclenchent le champ SaveResult pour informer l'utilisateur de la réussite ou de l'échec.

En cliquant sur le bouton Envoyer, la chaîne codée FormPdfBase64 encoded string is included and IsSubmit passe de faux à vrai.

Si le résultat retourné est vrai, l'utilisateur passe à la page Envoi réussi qui affiche son ReferenceNumber.

UserCanEditForm

 

résultat bool

Au lancement du Smartlet, indiquez si l'utilisateur actuel a, ou non, la permission appropriée pour sauvegarder et soumettre le formulaire.

GetProjectOutcomes

string FC_Profile_ID

Data.FCProfile.egcs_fc_result[]

Sur Initialisation de Smartlet pour les rapports sur la mesure des performances, le Smartlet appelle le service Web GetProjectOutcomes web service which queries the RDA web service GetProjectOutcomes du portail, qui interroge le service Web des ADR  et retourne un tableau d'objets de données qui correspondent aux résultats de projet qui sont disponibles au profil d'analyse du financement dont l'identité a été communiquée.

GetProgramsList

 

Data.Program.egcs_TP_Profile[]

N'est pas utilisé actuellement dans aucun écran, mais procure une liste des programmes actuellement disponibles advenant que l'on veuille que Smartlet extrait les données de CRM plutôt que de les coder en dur dans une liste déroulants Smartlet.

GetFCProfileID

 

FCProfileID

Au niveau de projets (écrans Réclamations, Avance et Rapport sur la mesure des performances), l'initialisation de Smartlet récupère l'identité du profil d'analyse du financement.

Remarque: est retourné comme FCProfileID et pas une ficelle en tant que SmartGuide a ses connexions SharePoint configurées en tant que valeurs de retour JSON.

GetBudgetCategories

chaîne FC_Profile_ID,

Data.Claim.egcs_FC_BudgetItem[]

Sur Initialisation de Smartlet sur les écrans Réclamation et Avance, interroge le service Web des ADR GetBudgetCategories, qui retourne un tableau d'objets de données qui dont employés pour remplir les options de liste déroulante des catégories de budget dans les tableaux des éléments de réclamations et des éléments de paiements anticipés aux écrans des réclamations et des paiements anticipés, respectivement.

GetProgramQuestionsFromCRM

GUID applicationFormIdGUID

Data.FormManager.egcs_ApplicationForm

À l'écran de l'application dynamique, lorsque l'on passe de la page 1 à la page 2, l'écran appelle GetProgramQuestionsFromCRM which passes the supplied GUID applicationFormIdGUID (which is set by the Program Drop down on page 1 to the RDA web service GetProgramSpecificQuestions si le programme actuel des écrans a été modifié, nécessitant ainsi une mise à jour aux questions dynamiques provenant de CRM qui paraissent à la page 2.

L'objet retourné renferme un tableau de questions et leurs métadonnées appropriées pour construire et joindre en annexe SmartletFields avec le GUID représentant le modèle retourné à l'écran.

Services Web utilisés dans les formulaires SmartGuide

Service Web

Paramètres

Retours

Description
CreateDraft int FormDefinitionId résultat de la chaîne

Appelé à partir de la liste déroulante Créer une page maître, crée une instance d'écran dont le type est déterminé par le FormDefinitionId fourni dans le champ actuel d'application du portail.

Le résultat renvoyé est l'URL vers laquelle la page est redirigée, qui rend le brouillon nouvellement créé dans SmartletRender.aspx.
RemoveDraft chaîne ReferenceNumber, SubmissionRemoveResult

Appelé de la partie Web des ébauches à partir de la page d'activités récentes, élimine l'ébauche correspondant au ayant été fourni.
ListDraft  

SubmissionListDraftResult

Appelé depuis les ébauches de composants WebPart à la page des activités récentes, dresse une liste des projets d'écrans disponibles au Web actuel. Les renseignements fournis comprennent les suivants : numéro de référence, type, données crées, création par, données modifiées, modification par.  En plus des liens pour modifier ou éliminer l'ébauche.
