Service de Windows : création de site

Le service Web Gestion des documents chapeaute la création et la maintenance des zones Client, Projet et Personnel pour le portail.

  • Il s'agit d'un service Windows nommé EGCS.Portal.Service.SiteCreation..
  • Il est installé dans un serveur SharePoint.
  • Le service fonctionne constamment comme une tâche en arrière-plan sur le serveur.
  • À l'interne, le service programme et exécute des tâches pour créer et tenir à jour des zones du portail (collections de sites et sites SharePoint).
  • Le service se branche à une batterie de serveurs SharePoint au moyen d'un modèle d'objet SharePoint côté serveur.
  • Le service se branche à la base de données de la Fédération des justificatifs du gouvernement du Canada (FJGC).
  • Le service se branche au service Web des ADR.
  • Le service devrait être configuré de sorte à fonctionner comme un compte de service SharePoint.

Tâches de service

The Site Creation service runs a set of jobs, which are configured via settings in EGCS.Portal. The class SiteCreationDispatcher oversees the scheduling and execution of all jobs. La plupart des tâches s'exécutent à intervalles répétés pendant que le service fonctionne; d'autres s'exécutent selon un horaire fixe.

Les files d'attente de clients à créer et l'établissement de zones de portail actualisées et stockées dans la base de données de la FJGC.

Classe

SiteWorkerClientQueue

EGCS.Portal Settings

SiteCreationClientSiteEnabled

SiteCreationClientSitePollIntervalSeconds

SiteCreationClientSiteWorkerCount

Notes
  • Une zone Client représente un client du GC dans GPSC.
  • Une zone Projet représente un projet (FCProfile) dans GPSC.
  • Fait un suivi du tableau GCCF.dbo.SiteQueueAccount pour de nouvelles zones Client qui doivent être créées. Des rangées sont ajoutées à ce tableau au moyen du service Web du portail (comme appelé depuis le service WebAPI).
  • La tâche peut créer en parallèle plusieurs zones Client. Les agents de travail de la classe SiteWorkerClient exécutent la création de sites comme telle.
  • Lorsqu'une zone Client est créée, des zones Projets pour tous les projets se trouvant dans GPSC sont également créées.
  • Utilise des modèles de composantes WebPart et des modèles de sites Plug-and-Play de SharePoint, qui sont stockés dans le répertoire des services.
Création d'une zone Personnel
Création d'une zone Personnel
   

Classe

SiteWorkerUserQueue

EGCS.Portal Settings

SiteCreationUserSiteEnabled

SiteCreationUserSitePollIntervalSeconds

SiteCreationUserSiteWorkerCount

SiteCreationUserSiteMaximumFailedCount

SiteCreationUserSiteQueueSize

Notes
  • Une zone Personnel est un site privé d'un utilisateur sur le portail.
  • Un utilisateur est lié à un PortalUser dans GPSC.
  • Monitors the table GCCF.dbo.SiteQueueUser to keep count of the number of unassigned Personal Zones. If the number of unassigned zones falls below SiteCreationUserSiteQueueSize, the job will begin creating new Zones. The job can create multiple Personal Zones in parallel. Worker agents of the class SiteWorkerUser perform the actual site creation.
  • Utilise des modèles de composantes WebPart et des modèles de sites Plug-and-Play de SharePoint, qui sont stockés dans le répertoire des services.

Maintenance d'une zone Client
Maintenance d'une zone Client
   

Classe

SiteWorkerClientMaintenance

EGCS.Portal Paramètres

SiteCreationClientSiteMaintenanceEnabled

SiteCreationClientSiteMaintenanceStartTime

SiteCreationClientSiteMaintenanceIntervalMinutes

Notes
  • Contrairement à d'autres tâches, celle-ci a une heure de démarrage établie.
  • Il s'agit d'une tâche relativement onéreuse. Elle doit demander des informations sur chacune des zones Client et Projet sur le portail à partir de GPSC (ce qui peut signifier de passer dans des dizaines de milliers de sites). A suggéré que cette tâche ne devait s'exécuter qu'une fois par jour, durant le quart de nuit.
  • Vérifie chacune des zones Client sur le portail pour déterminer si de nouveaux projets se sont ajoutés à GPSC. Ajoute de nouvelles zones Projet aux zones Client.
  • S'assure que les noms français et anglais des zones Client et des zones Projet sont à jour dans les tableaux de base de données GCCF.dbo.account and GCCF.dbo.egcs_fc_Profile.

Conditions préalables

  • A build of the GC.GCCF and EGCS.Portal solutions, which is assumed to be available in a directory _Build. The actual service comes from the EGCS.Portal solution, but gccf.config configuration is also required.
  • La base de données GCCF doit également être fournie.
  • Le compte de service doit avoir accès à la base de données GCCF database via the database role GCCFUser.
  • Les composantes PowerShell Plug-and-Play de SharePoint doivent être installées dans le serveur SharePoint :
    Install-Module SharePointPnPPowerShell2016
  • Ajoutez des chemins d'accès administrés SharePoint pour prendre en charge les zones Client et les zones Personnel. Les chemins d'accès devraient correspondre aux paramètres provenant de EGCS.Portal – UrlClientSiteRoot, UrlUserSiteRoot.
    New-SPManagedPath -RelativeURL /client -WebApplication https://<portal web application>
    New-SPManagedPath -RelativeURL /my-moi -WebApplication https://<portal web application>.
  • La collection de sites racine (/) du portail doit exister.
  • Activez la création de collections de sites par Plug-and-Play de SharePoint. Ce script déclenchera un IISRESET.
    Créez un nouveau site SharePoint pour héberger la fonction Administration des clients pour Plug-and-Play de SharePoint. Ce site servira à créer de nouvelles collections de sites par le service. Les utilisateurs de portail ne s'en serviront pas. The URL is assumed to be /sites/TenantAdmin but can be changed in EGCS.Portal configuration.
    New-SPSite -Url https://<portal>/sites/TenantAdmin -OwnerAlias "i:0#.w|<owner account>"
  • Activez Administration des clients et configurez le nouveau site.
    _Build\Scripts\allowtenantapiv16.ps1 -WebApplicationUrl https://<portal web application>

    $site = Get-SPSite https://<portal web application>/sites/TenantAdmin

    $site.AdministrationSiteType = [Microsoft.SharePoint.SPAdministrationSiteType]::TenantAdministration

    $site.Update()

Installation

Il faudrait disposer d'une nouvelle version du service et l'installer dès que l'on procède à une recompilation du projet EGCS.Portal project is recompiled, or the GC.GCCF. Elle a des dépendances sur les bibliothèques à lien dynamique (DLL) provenant de ces deux projets.

  1. Installez le service Gestion de documents de Windows
    .\_Build\Scripts\Install- SiteCreationService.ps1 –Config .\_Build\Config\<environment>.egcs.config
    This command will:
    • Stop the service (if it is running)
    • Install service files to path
      C:\Program Files\EGCS\EGCS.Portal.Service.SiteCreation
    • Register the service with Windows (if the service is not registered)
      Service Name: EGCS.Portal.Service.SiteCreation
    • redémarrera le service» (s'il était en mode exécution auparavant)
  2. Installez la configuration EGCS.Portal configuration (egcs.config):

    .\_Build\Scripts\Install-GccfCommonDomain.ps1

    Le fichier de configuration est copié au répertoire du service.

    Le service s'arrêtera et redémarrera.

  3. Installez la gccf.config configuration

    .\_Build\Scripts\Install-GCCF_Config.ps1

    Le fichier de configuration e configuration est copié au répertoire du service.

    Le service s'arrêtera et redémarrera.

  4. Lorsque c'est la première fois que le service est installé, il faudra lui attribuer un compte de service et le démarrer manuellement. Cela ne devrait se faire qu'après l'installation des deux fichiers de configuration.
    1. À l'aide du composant logiciel enfichable « Services », configurez un nom d'utilisateur et un mot de passe pour le compte de service pour EGCS.Portal.Service.SiteCreation. This should be the SharePoint service account.
    2. Set the service start-up mode to “Automatic” or “Automatic (Delayed Start)”
    3. Start the service via the “Services” snap-in, or via PowerShell.
    • Start-Service EGCS.Portal.Service.SiteCreation.
