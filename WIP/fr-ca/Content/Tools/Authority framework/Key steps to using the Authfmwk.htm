Principales étapes pour l'utilisation du cadre d'autorité

Les principales étapes pour configurer les approbations de le GPSC:

  1. Dans le cadre de la conception du processus d'affaires, déterminer les approbations utilisées à votre organisation. Pour chaque approbation, notez:

    • Le nom des types d'approbation (par exemple, approbation de la modification, approbation du projet, émission du paiement, etc.).
      Ces types sont configurés avec le CFG Type d'autorité.

    • Identifier les rôles des personnes concernées (par exemple, gestionnaire / directeur / DG) ainsi que l'ordre des approbations (qui examine en premier, en deuxième et qui approuve).
      Ces rôles sont configurés avec un CFG Ensemble d'autorité
    • Identifier les personnes qui remplissent ces rôles.
      Créer les Équipes.
    • Déterminer les équipes qui sont associées avec un Ensemble d'autorité particulier. (par exemple, Examinateur 1=Gestionnaire, Examinateur 2=Directeur,  Approbateur=Directeur général)
      Les associations entre l'Ensemble d'autorité et les  Équipes sont configurées avec des enregistrements CFG Voie d'accès d'autorité.
  2. Configurer les flux de processus d'affaires dans lesquels les approbations seront utilisés.
  3. Configurer les modèles de transition pour les flux de processus d'affaires. Utilisez le CFG Type d'autorité approprié dans la transition afin de s'assurer que le cadre d'autorité est configuré pour le flux de processus d'affaires qui est utilisé.
  4. (facultatif) configurer les actions / les ensembles d'actions qui peuvent être déclenchés par une approbation ou des actions qui peuvent déclenchées une approbation.

  5. Mis a part les entités de configuration dans le Cadre d'autorité, il y a deux entités pour faire le suivi des approbations en attente et enregistrer les décisions sur les approbations.

    • Les enregistrements Activité d'approbation sont générés afin de fonctionner en tant que tâche pour les personnes concernées dans le processus d'approbation.

    • Les enregistrements Activités d'approbation sont générés afin d'enregistrer une décisions et les renseignements liés dans le processus d'approbation. Ces enregistrements remplissent la grille du Processus d'approbation dans une entité de décision <Entité>. Lorsqu'une décision est prise dans un enregistrement Action d'approbation, le statut de l'Activité d'approbation reliée est fixé à  Compléter.
      Consultez Composants du cadre de décision-transition pour en savoir plus à propos des entités qui font parties du Cadre de décision.
