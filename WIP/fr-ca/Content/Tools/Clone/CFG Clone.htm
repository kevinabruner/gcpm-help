Configurer des enregistrements de clone

Les valeurs établies dans les enregistrements de configuration sont basées sur la structure relationnelle des entités à être clonées. Le système comprend le clonage pour les structures relationnelles suivantes qui peuvent être utilisées comme référence pour les autres configurations de clone.

  1. Entité enfant avec de multiples entités parents
  2. Entités modèles.

Pour configurer un enregistrement clone:

  1. Dans recherche avancée, cherchez CFG Clone et créez un nouvel enregistrement.
  2. Remplissez les champs au moyen de la fonction d'aide à l'écran et les notes ci-dessous :

    • Nom: le nom de cet enregistrement CFG Clone.
    • Configuration par défaut: indique su c'est la configuration par défaut pour l'entité source.
      Remarque: un bouton Enregistrer sous personnalisé qui peut être configuré utilise la configuration par défaut. (Requiert qu'un bouton Enregistrer sous soit configuré, au besoin)
    • Ignorer: indique si cette configuration ignore cet enregistrement et copie seulement les enregistrements liés.
    • Description : présente une description pour la configuration de lenregistrement clone. La description est utile pour différencier les différentes configurations de clone à partir de la même entité source.
    • Entité cible:
      • Même que la source: indique si l'entité source et l'entité cible sont les mêmes (vrai) ou si elles sont différentes. Cela doit être faux lors de la copie des enregistrements.
      • Entité parent en ce qui concerne le champ: (pour la copie seulement) le champ de recherche dans l'entité cible qui pointe à l'enregistrement de l'entité source. Ce champ est nécessaire si vous déclenchez le processus de copie à partir de l'entité cible au lieu de l'entité source.
  3. Mise en correspondance des champs : remplir ces champs si vous clonez ou copiez ces enregistrements.
    Conseil : lors de la copie des données entre deux ensembles d’enregistrement du même type d'entité, utilisez cette méthode au lieu de la méthode de « Clonage des enregistrements » si les champs sources et cibles sont différents.
    • À partir
      • Entité source: donne le nom logique de l'entité au clone.
      • Champ hiérarchique source: choisissez la relation qui se fait référence à elle-même dans l'entité source, s'il y en a une. C'est nécessaire pour maintenir la hiérarchie de l'enregistrement.
      • Champs source: la liste des champs que vous souhaitez copie, délimiter par des virgules.
        Remarque: si vous copiez des champs de différents types d'entité, le champ cible doit être du même type de données ou cela échouera. Les champs source et cible doivent être mis en correspondance en ordre, les deux listes doivent avoir le même nombre de champs.
    • À
      • Entité cible : le nom logique du nom de l'entité que vous souhaitez créer.
      • Champ hiérarchique cible: le champ de recherche de la relation qui se fait référence à elle-même dans l'entité cible, s'il y en a une. C'est nécessaire pour maintenir la hiérarchie de l'enregistrement.
      • Champs cible : la liste des champs que vous souhaitez copier, délimiter par des virgules.
  4. Configurations reliées au clone.
