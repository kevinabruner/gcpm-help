Configurer les types de décision

L'entité Type de décision stocke l'ensemble des décisions qui peuvent être prises au sein de votre organisation. Ces types de décision sont utilisés dans le cadre des flux de processus d'affaires.

Associez le type de décision à un cheminement de transition afin d'obtenir les renseignements liés aux documents qui seront générés pour chacune des décisions. Si cela est précisé dans un cheminement de transition, un enregistrement de décision est créé dans l'entité de décision <EntityName> correspondant au moment de la transition. Si un processus d'approbation est nécessaire en vue de la transition, le type de décision doit être précisé.

Remarque : Le processus d’approbation nécessite le type de décision. Pour que soit lancé un processus d'approbation, le type de décision et le type d'approbation doivent être établis dans le cadre d'un enregistrement de cheminement de transition. Consulter Configurer un cheminement de transition.

De nombreux types de décisions sont pré-configurés dans le GPSC. Quelques-uns des types de décision liés à ce projet comprennent :

  • Approbation de projet

  • Évaluation de projet approuvée.
  • Évaluation de projet prête en vue des recommandations.
  • Approuver l'analyse de projet
  • ...et beaucoup d'autres.

Configurer un nouveau type de décision :

  1. Parcourir le Administration > Décision > Types de décisions. Créer un nouvel enregistrement.

  2. Remplissez les champs au moyen de la fonction d'aide à l'écran et les notes ci-dessous :
    • Nom (AN / FR) : Saisissez un nom qui permet de décrire la décision.
    • Document configuration: Choose an existing CFG Document record. Laissez vide si aucun document ne doit être créé pour cette décision.
    • Document Template: Choose an existing CFG Document Template record. If you entered a CFG Document but leave this field empty, only the XML snapshot will be generated. If the Document Template is selected, a human readable document will be generated such as Word or PDF depending on the configuration.
    • System generated documents: To generate multiple document records for a decision, add more Decision Type Document records to the grid.
  3. Enregistrez et fermez l'enregistrement.
  4. Un enregistrement de document sera créé et annexé à la décision et à l'entité parent de la décision.

  5. S'il existe un processus d'approbation pour la décision, le document sera créé lors de l'approbation finale et annexé également à ce compte rendu de décision d'approbation finale.

  6. Pour annexer un document à une décision, l'enregistrement doit d'abord être enregistré.
    Remarque: le bouton Remarque: le bouton Enregistrer doit être utilisé. N'utilisez pas Enregistrer & Fermer. Une fois fermé, le dossier ne peut être ouvert que via un historique des décisions).

  7. Une fois que l'enregistrement est enregistré, les documents peuvent être ajoutés à la sous-grille au bas de la page.
