Créer des ressources Web

Trois ressources doivent être crées. Les ressources Web sont pour trois entités différentes :

  • [Entité] Formulaire de décision
  • [Entité] Formulaire d'historique
  • [Entité] Ruban.

Pour des renseignements sur la création d'un script de format TypeScript et de son équivalent JavaScript, consultez le Cadre (pour TypeScript).

Créer une ressource Web qui sera utilisée dans le ruban [Entité] :

  1. Créer une ressource Web et lui attribuer un nom egcs_/entities/[ENTITY LogicalName]/ribbons/form.js.
    For example: egcs_/entities/egcs_fc_disbreview/ribbons/form.js.
    Source code location:$/GCPM/EGCS/EGCS.CRM/EGCS.CRM.Framework/EGCS.CRM.WebResources/egcs_/entities/egcs_fc_disbreview/ribbons/form.js.
  2. Copiez le code à partir du form.ts d'une autre entité prévoyant une transition. (p. ex. utilisez l'entité d'examen des débours).
  3. Modifiez le code de la fonction d'exportation afin d'utiliser les valeurs de votre nouvelle entité.

Créez une ressource Web qui sera utilisée dans le formulaire de décision de l'[Entité].

  1. Créez une ressource Web pour la décision de l'[Entité] et attribuez-lui un nom.egcs_/entities/egcs_[ENTITY]_decision/forms/default.js

    Par exemple : egcs_/entities/egcs_disb_decision/forms/default.js.
    Source code location: $/GCPM/EGCS/EGCS.CRM/EGCS.CRM.Framework/EGCS.CRM.WebResources/egcs_/entities/egcs_disb_decision/forms/default.ts.

  2. Copiez le code du default.ts d'une autre entité de décision, par exemple une décision de débours.
  3. Modifiez les lignes suivantes afin d'utiliser les valeurs de votre nouvelle entité.
    Les modifications obligatoires sont en rouge. Vous pouvez également modifier le nom du débours variable, mais cela n'est pas nécessaire à son fonctionnement.
  4. Ajoutez le script de egcs_/shared/js/main.js aux bibliothèques de formulaires des décisions de l'[Entité].

  5. Ajoutez le script de egcs_/shared/js/main.js script to Onload et traitez le nom du script créé dans le cadre de cette étape comme un paramètre.
    Remarque : ce paramètre ne devrait pas comporter le suffixe .js. Vérifiez l'option permettant de traiter le contexte d'exécution comme un paramètre initial.

  6. Cet aperçu d'écran présente l'événement de chargement sur le formulaire de décision de débours :

Créez une ressource Web qui sera utilisée dans le formulaire historique de l'[Entité].

  1. Créez une ressource Web qui sera utilisée dans le formulaire historique de l'[Entité] appeléeegcs_/entities/egcs_[ENTITY]_history/forms/default.js

    Par exemple : egcs_/entities/egcs_disb_transitionhistory/forms/default.js.
    Source control location: $/GCPM/EGCS/EGCS.CRM/EGCS.CRM.Framework/EGCS.CRM.WebResources/egcs_/entities/egcs_disb_transitionhistory/forms/default.ts.

  2. Copiez le code d'un autre historique d'[Entité] : default.ts, par exemple, l'historique de débours.
  3. Modifiez ces lignes afin d'utiliser les valeurs de votre nouvelle entité.
    Les modifications obligatoires sont en rouge. Vous pouvez également changer le nom de la variable, mais cela n'est pas nécessaire à son fonctionnement.

  4. Ajoutez le script egcs_/shared/js/main.js aux bibliothèques de formulaires des décisions de l'[Entité].

  5. Ajoutez le script egcs_/shared/js/main.js à l’événement de formulaire de chargement et traitez le nom du script créé dans le cadre de cette étape comme un paramètre.
    Remarque : ce paramètre ne devrait pas comporter le suffixe .js.

  6. Vérifiez l'option qui permet de traiter le contexte d'exécution comme un paramètre initial.
