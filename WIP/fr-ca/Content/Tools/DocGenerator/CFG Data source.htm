Configurer les sources de données pour un instantané

Les sources de données pour un instantané identifient si les données peupleront le modèle de document. Vous pouvez choisir une entité et tous ses champs, ou seulement des champs choisis, qui seront affichés dans le modèle de document.

Pour configurer une sources de données :

  1. Au moyen de la recherche avancée, cherchez CFG source de données et créez un nouvel enregistrement.
    L'image ci-dessous utilise les clauses de modification à titre d'exemple.
  2. Remplissez les champs au moyen de la fonction d'aide à l'écran et les notes ci-dessous :
    • Alias d'entité parent : au moment d’enchaîner des demandes d'information, ce champ est utilisé pour préciser l'entité dans la demande d'information de source de données parent à partir de laquelle vous enchaînez.
    • Source de données parent : afin d'outrepasser les limitations du CRM d'un maximum de 10 entités par demande d'information, ce champ vous permet d’enchaîner les demandes d'information les unes après les autres de façon à extraire plus de données.
    • Nom : le nom de la source de données. Ce nom sera utilisé comme nom de « dossier » pour les données précisées.
    • Description : fournissez une description pour que les gens sachent de quoi il s'agit
    • Tous les attributs d'entités : 
      • Si ce champ est réglé a Vrai, tous les attributs dans le premier (racine) peuvent être utilisés dans le modèle de document.
      • Si ce champ est réglé à Faux, seuls les attributs précisés dans la demande d'information seront disponibles.
    • Tous les attributs d'entités reliés : 
      • Si ce champ est réglé à Vrai, tous les attributs dans toutes les entités liées à l'entité racine peuvent être utilisés dans le modèle de document.
      • Si ce champ est réglé à Faux, seuls les attributs précisés dans la demande d'information seront disponibles.
    • Alias d'entité de départ : souvent, lorsque vous créez des sources de données multiples, vous extrairez des données semblables d'autres sources de données; ce champ vous permet de préciser les données de l'entité que vous voulez utiliser au moyen de l'alias de l'entité dans la demande d'information FetchXML, y compris les entités liées à cette entité.
    • Demande d'information : entrez une demande d'information FetchXML afin de préciser les données à extraire.
      • Conseil : utilisez la Recherche avancée afin de créer une demande d'information et d'utiliser le FetchXML qu'elle génère pour la demande d'information.
      • Remarque : un maximum de 10 entités peut être référencé dans une seule demande d'information.
      • Les conditions peuvent être établies dans Fetch XML – p. ex., choisir un champ, égal (choisir une valeur ou de nombreuses valeurs). Enregistrer la demande d'information (pour assurer une uniformité dans la réutilisation a l'avenir)
      • Dans l'image ci-dessous, la demande d'information est construite afin d'extraire un Projet en plus d'un Client SC, un comte, un contact, une salutation, et de l'information sur le programme. Au total, des données d'une valeur de sept entités sont extraites par cette demande d'information. Vous pouvez ajouter des attributs particuliers, des conditions et des filtres afin de préciser les données.

    • Nom des champs : certaines entités possèdent des identificateurs uniques pour leurs enregistrements, ce champ vous permet donc d'utiliser ces identificateurs uniques en tant que nom de nœud dans les données XML, ce qui vous permet de référencer des enregistrements précis dans une liste d'enregistrements. Cela est utile pour utilisation de données en tant que déclencheurs, ou pour les situations conditionnelles.
      Par exemple, si X = valeur donnée, afficher la clause, autrement cacher la clause.
    • Relation hiérarchique : inscrivez le nom logique de la relation.
      Les entités ayant des relations hiérarchiques qui se font référence à elles-mêmes (p. ex., les clauses ont des niveaux 1, 2, 3, 4 pour fournir des sous-clauses).
  3. Enregistrez et fermez l'enregistrement.
