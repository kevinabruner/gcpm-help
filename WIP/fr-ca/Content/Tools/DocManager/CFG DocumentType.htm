Configurer les types de documents

Les documents qui sont auto-générés dans le GPSC nécessitent un modèle de document et un enregistrement CFG Type de documents.

Un modèle de document (p. ex., pour un document de décision) définit le contenu et le format des données. L'enregistrement CFG Type de documents identifie où stocker un document généré par le système (p. ex., dossier du projet ou de la décision du projet).

Pour configurer un enregistrement CFG Type de documents :

  1. Naviguez à Configuration > Gestion de document > CFG Type de documents et créez un nouvel enregistrement.
  2. Remplir les champs. Servez-vous de la fonction d'aide à l'écran et des notes ci-dessous pour vous guider.
    • Nom : fournissez le nom du type de document. (Par exemple, Entente de contribution)
    • Entité du document : indiquez le nom de l'entité associée au type de document.
    • Dépôt de documents : sélectionnez le dépôt de documents pour le document.
    • Type de contenu : (facultatif) l'ID de type de contenu externe du SGD auquel ce type de document est cartographié.
    • Lieux de document : choisissez un lieu de document à partir de la liste affichée. (p. ex., dossier de documents juridiques) 
      Si le lieu approprié ne figure pas dans la liste, configurez un nouvel enregistrement CFG du lieu du document.
    • Document du portail : indiquez Oui si le document peut être communiqué au client dans le portail.
    • Métadonnées : (cette fonction n'était pas mise en œuvre au moment de la version 2). Mapper les champs CRM aux champs des métadonnées dans SharePoint ou GCDocs.
  3. Enregistrez et fermez l'enregistrement.
