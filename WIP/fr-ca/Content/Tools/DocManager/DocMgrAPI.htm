API du plugiciel de gestion de document

Le plugiciel de gestion de document offre un ensemble de fonctions API utilisées afin d'effectuer certaines actions de façon programmatique.

Les paramètres avec astérisques (*) sont obligatoires et ne peuvent être nuls. Pour les paramètres facultatifs de types de données primitives, la valeur par défaut est utilisée si aucune valeur n’est fournie, p. ex., booléen devient par défaut fausse et int à 0.

Authentiquer

Utilisé seulement par le connecteur SharePoint, le mécanisme d'authentification a été résumé et encapsulé dans sa propre action personnalisée de façon à ce qu'il puisse être facilement surclassé par une mise en œuvre locale.

Si vous ne faites que surclasser le mécanisme d'authentification, la mise en oeuvre locale doit retourner au minimum le FedAuth témoin, le bulletin de formulaire, et la date d'expiration, tel qu'exigé par REST API de SharePoint.

API interne

Le plugiciel de gestion de document comporte un ensemble générique de messages API et de définitions de types de données qui sont utilisés pour faire des appels au SGD au moyen d'un connecteur propre au SGD. Parce que les connecteurs reçoivent tous les mêmes demandes et données, il est possible de surclasser ou d'ajouter des connecteurs afin de soutenir d'autres SGD externes, comme OneDrive, Dropbox, etc.
