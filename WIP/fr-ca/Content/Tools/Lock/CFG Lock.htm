Configurer des enregistrements de verrouillage

Pour configurer un enregistrement de verrouillage pour une entité:

  1. En utilisant la Recherche avancée, cherchez pour CFG Verrouillage. Créer un nouveau enregistrement.

  2. Remplissez ces champs, au besoin:

    • Configuration de verrouillage parent: sélectionnez l'enregistrement CFG verrouillage de l'entité parent, le cas échéant.
      Remarque: n'utilisez ce champ que si c'est un enregistrement enfant avec un enregistrement parent (par exemple, un élément de réclamation est l'enfant de la réclamation).
    • Nom du champ parent: entrez le nom logique du champ de recherche sur cette entité qui renvoie à l'entité précisé dans la configuration de verrouillage parent.
    • Nom de l'entité: sélectionné un nom d'entité à partir de la liste déroulante. C'est l'entité qui sera verrouillée.
      L'entité sélectionnée sera enregistrée dans le pugiciel.
    • Champ de verrouillage: le nom logique du champ qui, lorsqu'il contient une valeur donnée, réalisera le verrouillage
      Le champ de verrouillage doit être l'un de ces types de champ:
      • Deux options (Oui / Non): le plugiciel suppose:
        Oui (avec un crochet) = verrouillé,
        Non (sans crochet) = déverrouillé.
      • Recherche ou Ensemble d'option: vous devez créer un enregistrement CFG Champ verrouillage pour chacune des valeurs de la recherche ou de l'ensemble d'option. Chaque valeur dans la recherche ou l'ensemble d'option qui est une valeur de verrouillage. Chaque valeur dans la recherche ou l'ensemble d'option peut être configurer pour verrouiller différents ensembles de champs.
      • Une fois configurer, ces valeurs s'affichent dans l'onglet des configurations de Champ de verrouillage.
        Par exemple, un examen de débours qui a un statut paiement émis peut verrouiller des champs différents d'un examen qui a un statut rejeté.
        Remarque: sur les entités enfant ou petit-enfants, le champ verrouillage doit être du type Deux options, puisque le plugiciel va passer le verrouillage automatiquement à toutes les entités enfants.
    • Sélecteur du Champ de verrouillage: sélectionnez tous les champs a être verrouiller - à l'exception du champ verrouillage (qui déclenche le verrouillage). Les champs sélectionnés seront enregistrés dans le plugiciel.
      Remarque: si les nouveaux champs sont ajoutés à l'entité associée avec cette configuration, ces champs ne seront pas automatiquement sélectionnés. Par défaut ces champs seront modifiable. Si vous souhaitez les verrouiller, ouvrez simplement cette configuration et vérifiez si la cas a cocher pour tous les autres champs. Le sélecteur de champ de verrouillage s'affiche lorsque le type du champ est Deux options.

  3. Sauvegardez l'enregistrement. Ces onglets additionnels peuvent s'afficher:

    • La grille des configurations reliées au verrouillage affiche les enregistrements enfant CFG Verrouillage reliés.
    • L'onglet des Configuration des champs verrouillage s'affiche lorsque le type de champ verrouillage est une recherche ou un ensemble d'option.
    • L'onglet des Métadonnées affiche un aperçu rapide de l'ensemble des valeurs pour cette configuration. Il offre une façon plus facile de vérifier les données que par l'utilisation du sélecteur de champ.
  4. Enregistrez et fermez l'enregistrement.

Prochaines étapes: créer des enregistrements de configuration enfant et petit-enfant, si le verrouillage a une incidence sur ces enregistrements.

Pratique exemplaire 1 :

Lors de la configuration des champs à être verrouillés:

  • Ne pas inclure l'actuel champ verrouillage dans la liste des champs verrouillés de l'entité racine. Cela permet la modification du champ et le déverrouillage de l'enregistrement.
  • Inclure le champ verrouillage dans la liste de champ verrouillé des enregistrements enfants et petit-enfants CFG Verrouillage. Cela permet le contrôle de la caractéristique de verrouillage seulement par l'entité racine.
Pratique exemplaire 2 :

GPSC comprend de nombreux enregistrements de verrouillage qui utilise un champ Deux options. Ces enregistrements ont un champ egcs_locked(avec nom d'affichage = verrouillé et type de champ = deux options: oui / non). Utilisez une convention d'appellation conforme rend plus facile de remplier le champ verrouillage.
