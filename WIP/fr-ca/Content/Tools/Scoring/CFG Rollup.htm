Configurer les enregistrements de cumul

CFG Cumuls effectue les calculs indiqués (ou cumuls) dans les champs, au besoin. L'enregistrement d'un cumul utilise la valeur d'un champ précis dans des enregistrements enfant et calcule un résultat selon le type d'agrégation (somme, dénombrement, maximum, minimum, etc.) et place ce résultat dans le champ d'un parent (champ cumulatif).

Pour configurer un enregistrement de cumul:

  1. Naviguez à Configuration > Outils de calcul > CFG cumuls et créez un nouvel enregistrement de cumul.

  2. Servez-vous de l'aide à l'écran et des notes ci-dessous pour remplir les champs:
    • Ordre : Indique l'ordre dans lequel un cumul doit être exécuté lorsque plus d'un est configuré dans une entité. Il faut porter une attention particulière au réglage de ce champ lorsque des champs calculés et des champs de cumul sont inclus dans l'agrégation car les calculs d'ordre auront une incidence sur le résultat.
    • Synchrone forcée : Le plugiciel fonctionne par défaut en mode in asynchrone, mais il est possible de forcer le mode synchrone si la valeur du champ est OUI. Force le plugiciel à s'exécuter en mode synchrone pour qu'il puisse s'exécuter immédiatement.
    • Entité de cumul : Nom logique de l'entité parent qui contient le champ de cumul.
    • Champ de cumul : Nom logique du champ de cumul dans l'entité parent.
    • Type d'agrégation : Le type d'agrégation de cumul.
    • Entité d'agrégat : Nom logique de l'entité enfant à agréger.
    • Champ d'agrégat : Nom logique du champ dans l'entité enfant qui est agrégée. Utilisez le nom logique du champ des clés primaires (p. ex., egcs_name) pour le type d'agrégation DÉNOMBREMENT.
    • Champ regardant: Inscrire le nom logique du champ du cumul a l’entité parent.
    • Description : Donne une description de cet enregistrement de configuration.
    • Est monnaie : Vérifiez si le type de données du champ de cumul est Devise.
  3. Pour utiliser une condition, saisir les paramètres et la requête de condition.
    Voir les notes ci-dessous.
    • Activer la condition : Cocher la case pour activer une condition.
    • Paramètres de condition : Précisez une liste délimitée par des virgules des attributs dont les valeurs doivent être utilisées dans la requête de condition. Voir la note ci-dessous pour la syntaxe des paramètres.
    • Condition : La condition à laquelle chaque enregistrement doit satisfaire doit être incluse dans le cumul. La valeur de ce champ est une chaîne de requête FetchXML. Il est possible d'utiliser les recherches avancées de CRM pour établir la chaîne de requête FetchXML. Dans le cas des requêtes plus complexes, il est possible d'utiliser un outil tiers appelé FetchXML Builder*.
  4. Les champs Métadonnées se remplissent automatiquement et utilisés à l'interne pour prétraiter des mesures onéreuses dans le plugiciel et optimiser le rendement.
    • Métadonnées : Métadonnées de calcul.
    • Entités liées : Liste délimitée par des virgules des entités parentes mentionnées dans ce calcul.
  5. Enregistrez et fermez l'enregistrement.
Syntaxe de paramètre

La syntaxe du paramètre est la suivante:

{lookup.attribute:format;default value }

Considérations de syntaxe:

  • Le nom de la recherche n'est nécessaire que pour faire référence aux attributs parents.
  • Les accolades ne sont nécessaires que lorsque l'on précise le format et / ou la valeur par défaut.
  • Utilisez l'attribut modifiedon pour la date actuelle.
  • Format the attribute in the Condition or Formula field and not in the parameters field unless the value needs to be formatted twice.
Concaténation de données

La syntaxe de mise en forme et de concaténation des champs Condition / Formula et Condition / Formula Parameters est la même que String Format (String format, params object[] args)celle utilisée pour la méthode dans le .NET Framework. Dans cette syntaxe:

  • String parameter “format” est le valeur du champ de condition / formule.
  • Object[] parameter “args” est le valeur des paramètres de condition / formule.

Pour référencer les attributs dans le champ Paramètres de condition / formule, le champ Condition / Formule suit String.Format placeholder convention: {0} to refer to the first attribute, {1} pour le second, etc. Le nombre maximal doit être inférieur ou égal au nombre d'attributs énumérés dans le champ de paramètres condition / formule Les espaces réservés pour les nombres peut être utilisé plus d'une fois dans la formule.
