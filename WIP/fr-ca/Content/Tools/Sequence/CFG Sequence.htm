Configurer un enregistrement de séquence

Afin de créer un nouvel enregistrement CFG séquence :

  1. Naviguez à Configuration > Outils de calcul > CFG séquence et créez un nouvel enregistrement.
  2. Utilisez l'aide à l'écran et les notes ci-dessous pour une orientation sur la façon de remplir les champs :
    • Entité cible : entrez le nom logique de l'entité où la séquence est appliquée.
    • Champ cible : entrez le nom logique du champ dans l'entité cible qui conserve la valeur de la séquence générée.
    • Graine : sélectionnez une graine existante pour les instances où de multiples enregistrements d'entité généreront des graines du même bassin. Ou créez une nouvelle graine
  3. Séquence enfant :
    • Concernant : si c'est une nouvelle séquence pour les enregistrements enfants, définissez ce champ par le nom logique du champ de recherche de l'enregistrement parent.
    • Reséquençage: si définit à oui, le plugiciel reséquencera les enregistrements enfants lorsqu'un enregistrement enfant est supprimé.
  4. Formule :
      Paramètres de la formule : Établir une liste d'attributs délimités par des virgules dont la valeur sera utilisée pour le formatage ou la concaténation.
    • Consulter la note ci-dessous pour la syntaxe du paramètre.
    • Formule : consultez la note ci-dessous sur la façon dont la formule réalise les calculs et le formatage des données.
  5. Champs de métadonnées sont autopopulés
    • Entités liées : les entités parents auxquelles les paramètres de formule font référence.
    • Métadonnées : les métadonnées de la formule sont conservées ici pour optimiser le rendement.
  6. Enregistrez et fermez l'enregistrement.
Syntaxe de paramètre

La syntaxe du paramètre est la suivante:

{lookup.attribute:format;default value }

Considérations de syntaxe:

  • Le nom de la recherche n'est nécessaire que pour faire référence aux attributs parents.
  • Les accolades ne sont nécessaires que lorsque l'on précise le format et / ou la valeur par défaut.
  • Utilisez l'attribut modifiedon pour la date actuelle.
  • Format the attribute in the Condition or Formula field and not in the parameters field unless the value needs to be formatted twice.
Concaténation de données

La syntaxe de mise en forme et de concaténation des champs Condition / Formula et Condition / Formula Parameters est la même que String Format (String format, params object[] args)celle utilisée pour la méthode dans le .NET Framework. Dans cette syntaxe:

  • String parameter “format” est le valeur du champ de condition / formule.
  • Object[] parameter “args” est le valeur des paramètres de condition / formule.

Pour référencer les attributs dans le champ Paramètres de condition / formule, le champ Condition / Formule suit String.Format placeholder convention: {0} to refer to the first attribute, {1} pour le second, etc. Le nombre maximal doit être inférieur ou égal au nombre d'attributs énumérés dans le champ de paramètres condition / formule Les espaces réservés pour les nombres peut être utilisé plus d'une fois dans la formule.

Calcul de formule et formatage des données

To perform the calculation, the Formula field leverages the syntax used in .NET Framework DataTable class, specifically the DataColumn.Expression* property. Les valeurs de calcul sont rassemblées, tout comme les champs du formatage ou de la concaténation, de façon à ce que toutes les valeurs puissent être formatées avant les calculés réels.

Se référer la documentation de Microsoft sur la propriété DataColumn.Expression pour une syntaxe de calcul valide. On utilise la même syntaxe que SQL.
Un aide-mémoire est fourni en Syntaxe de calcul.

When parameter in the Calculation/Formula Parameters field are accessed through placeholders, they can be formatted. Se référer la documentation de Microsoft sur le méthode du Format pour une syntaxe de formatage de données valide.
La formule générale est : {value,alignment:format}

Un aide-mémoire est fourni en Guide de syntaxe pour le formatage de données.

Exemple :

To apply yyyy-MM-dd format to the egcs_duedate field:

  • Paramètresegcs_duedate
  • Formule: {0:yyyy-MM-dd}

Since there’s no way to specify a time value, say you wanted to subtract or add months, days, or years to a field which data type is date/time, a special syntax is used to input time values.

Time values are to be specified in the Formula Parameters field and not in the Formula field, so that no parsing is required.

For example, if you want to add 10 days to a field named egcs_duedate that is an attribute of lookup egcs_application entity and its default value is January 1st, 2018.

Les paramètres sont : {egcs_application.egcs_duedate; '2018-01-01'},10dd

La formule est: {0} + {1}
