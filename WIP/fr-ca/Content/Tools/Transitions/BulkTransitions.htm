Bulk transitions

Use these instructions as a guide to move a group of projects from one stage to another stage. For example, if your organization receives 1000 applications and wishes to move 300 (in one action) to the project stage, then assessment stage, etc. You will need to tailor these instructions to meet the particular processes in place at your organization.

Notes about bulk transitions:

  • Warnings do not display when bulk approving. To mitigate, create a report that identifies potential problems prior to a bulk transition.
  • Errors will halt the transition only for those projects that fail. Any successful project will still transition. CRM will not tell you which projects failed; you will go look at which statuses did not transition from your set of applicable projects. If you would rather have the entire operation either succeed or fail, you may require custom code.
  • The field On Behalf Of will not be populated unless the workflow is configured to do so for all cases.

To automate a process, first do it manually, and automate each step. If these examples are not sufficient, you can document your process manually and then proceed to automate.

Proceed to assessment – non-decision

  1. Create a workflow that will run on the project entity.
  2. Set it as on-demand with no other triggers.
  3. Have it generate a Project History record, which automatically triggers a transition.

    • From and To fields must line up with an existing transition or the transition framework is going to deny the request
    • Comments should reflect something about the fact that this was automated in bulk

Prepare recommendation – decision without approval

  1. Create a workflow

  2. Use the execution time of the workflow to populate a decision date of today.

Approve project – decision with approval

  1. This workflow starts the approval process.

  2. Once run, the second step will be to approve each of the Approval Actions that will be created.

  3. Use Advanced Find to search for the available approval actions.

  4. Select applicable rows and bulk edit

  5. Any successful project will be transitioned to Evaluation Authorized.
