Action definition library

Some of the action definitions included in GCPM are described below.

Action Description
Change Option Set Value

Sets the value of an option set based on 2 parameters:

Field: name of the option set field

Value: The value we want to set on the option set
Compare Field With Field Compare a field's value with another field's value
Compare Fields

Is used to compare a field value to another value or a null value (used with isNull and isNotNull functions). The function uses 5 parameters:

  • Entity: Used to filter the field list for the second parameter but not used in the compare function
  • Fields: List of fields we want to compare to the value
  • Condition: Logical operation we want to use (greater, lower, etc)
  • Value: The value we want to compare. This parameter is optional with the logical operation isNull and isNotNull
Complete approval activity Complete approval activity (more description needed)
Create task Allows to create a task
Create task lead officer

Creates a task to the project officer regarding the project.

No parameters needed but could add parameters for the subject, description of task, the person to assign it, etc.
Download document

Generates a document within a configuration of the document generator. Once the document is generated, a note is created with the document as attachment in the project.

The function is based on 3 parameters and returns the generated document.

Configuration: Document generator configuration

Template: The template that we will use to generate the document.

Record: Record data used to generate the document Return parameter:

Document: Generated document
Send Email

Sends emails. The function takes 2 parameters:

SendTo: The field to whom send the email. The lookup is usually link to a user that has an email.

SendFrom: The field from whom send the email. The lookup is usually linked to a user that has an email.
Set amendment transition Set the transition for the amendment
Set date field value Sets the date field value
Set inactive record Sets the current record to the Inactive state
Test exception Used for testing purposes. It causes this action to fail all the time to block the validation process.
