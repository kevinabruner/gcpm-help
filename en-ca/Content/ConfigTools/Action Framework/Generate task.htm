Actions to generate tasks

Actions can generate tasks for users that are pre-populated with a subject, a regarding link, identify who the task is assigned to and who has assigned the task.

When a user receives a task that was generated in a business process flow, the user can navigate to the appropriate location to complete the task.

Example - "Approval Activity" task: 

  • Task created to notify an approver that there is something to approve.
  • Subject = "You have a project to approve"
  • Regarding = name of project (with link to project)
  • Status = task status is marked "Complete" when the approval is completed.

Notes: 

  • Notification tasks are configured in a transition template, in the "Notification action set" field.
  • The text of the notification is modified in the form for the "Create approval activity to team" (a CFG action record).
  • To view pending approvals, look at the Manager Dashboard. or Officer Dashboard, as appropriate. Approval activities are listed here.
  • A custom action creates an Approval Activity task.

Assigning tasks to a team: 

  • see TRZ description in webex Sept 2018
  • Uses teams defined in approval team (eg. manager team, VP team, Region 1 team, etc.)
