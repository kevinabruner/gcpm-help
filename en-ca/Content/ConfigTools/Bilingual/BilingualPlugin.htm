Bilingual plugin

Modification

Only modified/reconstruct “PostMultilingualRetrieveHelper” and “PostMultiLingualRetrieveMultipleHelper” plugin of the original source in “MultiLingual.cs” file.

Setup

Separator

Separator is the character between English and French text.

The double number sign (##) is the separator used in the plugin. It is reference in theGetNameByLanguageId method in MultiLingual.cs.

Data

Use a workflow or the calculated field plugin to configure a multilingual record.

For example: EnglishText##FrenchText. On Create or Update of the record, the workflow or plugin will fire to setup/configuration text.

 

Multilingual entity (fp_multilingualentity)

Records created in the CFG Bilingual entity contain the parent record and its associated lookup that needs to be processed.

See Configure bilingual records.

Plugin

Both “Retrieve” and “RetrieveMulitple” message should be registered against the entity (bilingual record).

Note:

  • If the bilingual record is associated (lookup) on another entity, that entity should also be registered for both “Retrieve” and “RetrieveMulitple”.

The screenshots below show “Salutation” being the entity, and “Contact” being the related (Parent record of Salutation) entity.

Action/Functionality

On retrieve or retrieve multiple of a record, in this case “Salutation”, only English or French text will be shown to the user based on their language.

Limitations to the multilingual plugin

There are certain locations (charts, dashboards, grids) where the bilingual field does not appear in the user's language of choice, but instead displays the concatenated name (EN | FR). This results in a long name that is often truncated on the screen display.
