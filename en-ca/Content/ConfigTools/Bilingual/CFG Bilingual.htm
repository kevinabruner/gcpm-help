Configure bilingual records

GCPM fields can be configured to display in either English or French, depending on the user's language of choice (defined in user settings*). See example below.

To configure fields to display in the user's language of choice:

  1. In Advanced Find, search for CFG Bilingual and create a new record for the field that should display either in English or French.
  2. Complete the fields, using onscreen help and the information below:
    • Name has no format, but must be unique.
      Tip: use a standard, such as name of entity to be translated, followed by the word translation. In the above example, Name = Business type translation
    • Target entity: the logical name of the entity you wish to set up translation for.
      In our example, Target entity = egcs_sh_businessstructure
      Note: this field is case sensitive.
    • Fields to translate: a comma delimited list of fields to translate (use logical name).
  3. Save and close the record.

CFG Bilingual works in conjunction with the CFG Calculation plugin to display the users's language of choice for bilingual fields when needed. See About the bilingual configuration tool

Example

For example, business type records include English and French names for business types: 

  • Name EN=University
  • Name FR=Université

The English "university" displays if the user has English language option, and as "Université" if a user selects French language option.

Notes:

  • The plugin is designed to translate fields that have a pipe symbol | delimiting English and French information. If there is no pipe in the data it will not translate it. This can be especially tricky because the multilingual plugin triggers every time that entity and field are loaded - you won’t see the other language at all if it is present. However this does not change the underlying data.
  • The plugin only translates when the entity and field are loaded. The data contained in a lookup for example are that entity loading that field when you are on a form (say province when editing a contact record). To make that translate you need to configure it on the PROVINCE not the contact. The opposite is true however when viewing a set of records (active contacts to use the above example) that is only the contact loading so to translate the province there you need to configure translation on the province field on the contact.

Tip: The safest approach is to do a configuration for each and every entity where the field to translate occurs. A good example of that is to look at the account translation since it has so many fields that are lookups.
