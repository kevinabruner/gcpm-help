Configure a Word document template for a snapshot

A snapshot of data (created with the CFG Snapshot) can be inserted into a Word document. The Word document must be configured using XML to display the GCPM data.

To configure an XML Word document : 
Note: You must have Microsoft Windows 2016 or higher installed to do these steps.

  1. Display the Developer tab in Microsoft Word.

    If you do not see the Developer tab, open File > Options, select Customize RibbonUnder Main Tabs, check Developer box, then click OK. The Developer option appears in the Word ribbon.
  2. Configure a snapshot for the contribution agreement, including all necessary data sources.
  3. Once the snapshot has been configured, in the CFG Snapshot record, click WORD TEMPLATE, then choose Download blank template.

  4. Save the file. The blank document template contains the XML data that you can insert and reference in your document template.
    Now you can format the template with the appropriate text and fields.
  5. To view the fields available in the template, choose Developer - XML mapping. The XML mapping pane opens on the right. Only the entity data specified in the snapshot's data source records will be available in the document template.
  6. Add the text for this template, and format to suit your needs.
  7. To insert a field value, move your cursor to the location where you want to insert it. Find the field in the XML mapping pane, right-click the field, and select Insert as plain text.
    See tips below for more ideas on mapping fields.

  8. Save the template locally.

  9. Create a new CFG Document Template record, and upload the configured template.
    The template appears in the Templates tab of the CFG Document Template record.

  10. Associate the CFG document template record to a program (or appropriate entity).
Tips on inserting fields: 
  • The field can be formatted however way you want – bold, italic, underline, etc
  • For paragraph-like fields, such as a field with a lot of text, it may be best to insert as Rich Text instead of Plain Text.
  • Some standard fields can be inserted to the template. For example: 
    • @currentdate inserts the date the document is created. This creates a time stamp on the document created from the template.
    • @currenttime inserts the time the document is created.
  • The entities and fields are displayed with their logical names, not their display names. It is important to be familiar with the logical names to be sure you are choosing the correct field.
  • Inserting a “folder” as data will produce nothing in the document. e.g. referencing / inserting the account name associated with a project: 
  • Field data do not display a navigation arrow on the left side, because they are a single piece of data, not a “folder” that contains data.
