Configure CFG Locking Field records

To configure one or more locking fields on an entity:

  1. Using Advanced Find, search for CFG Locking Field.
  2. Create a new CFG Locking Field record.

  3. Fill in the fields using onscreen help and the information below

    • Name: Type the display value value in the locking field that will trigger the lock.
      For example, "Approved" or "Rejected." 

    • Locking field value
      • If the locking field is an option set, type the numeric value of the option.
      • If the locking field is a lookup type, the lock field value is a GUID of the record that will drive the lock.
      • Tip: An easy way to obtain a record's GUID is to use CRM development tools with Chrome. Open the record and hit F12 to open DevTools, and open CRM Dev Tools. Copy and paste the entity Id value without brackets.
    • Locking field selector: Choose the fields that will be locked when the given field value is selected.
    • Best practice: 

      • Do not include the locking field in the locked field list in the root entity. This ensures you will be able to unlock the record.

      • Do include the field(s) that turn on the lock (lock fields) for child and grandchild records. This ensures the lock is managed only by the root entity.
  4. Save the record.
    Once the record has been saved, a Metadata tab appears.
    All of the fields selected in the General tab appear in the locked field list.
  5. Create CFG Lock records for every child and grandchild needed.
To disable a lock:
  • Temporarily: to disable the lock feature for a specific entity, deactivate the CFG Lock configuration record.
  • Permanently: delete the CFG Lock configuration record.
