Configure sequential fields

To create a new CFG Sequence record:

  1. In advanced find, search for CFG Sequence and create a new record.
  2. Use the onscreen help and notes below for guidance in filling in the fields: 
    • Target entity: Type the logical name of the entity where this sequence is applied.
    • Target field: Type the field name in the target entity that stores the generated sequence value.
    • Seed: Displays the current number in the sequence. For non-child sequences, this number is incremented each time a new record is created.
    • Seed type: The type of seed.
      • Alphabetic increments the same as Excel columns (A - Z, AA, AB - AZ, BA...etc.)
      • Numeric: increments by the number in the increment by field.
    • Seed length: The maximum number of characters that the seed can grow to.
    • Increment by: The number to increment the seed after a new record is created. Defaults to 1 if not set.
    • Padding: The alphanumeric character to pad the seed with, up to seed length. If no padding character is specified, padding will not occur. The seed will be padded by this alphanumeric character until the seed length is reached.
    • Formula parameters: Specify a comma delimited list of attributes whose values are to be used in the calculation or text formatting / concatenation.

    • Note - formula

      This field leverages NET Framework’s library to perform the data formatting and concatenation.

      The syntax for formatting and concatenating fields is the same as the String Format(String format, params object[] args) method in the .NET Framework, where this field is the String format parameter and Formula Parameters is the object[] args parameter.

      See Microsoft’s documentation on the Format method for valid data formatting syntax. A cheat sheet is provided in Calculation Syntax.

      The general formula is: To reference the attributes in Formula Parameters, follow the String.Format placeholder convention: {0} to refer to the first attribute, {1} for the second, and so on. The maximum number must be less than or equal to the number of attributes listed in Formula Parameters. Number placeholders can be used more than once in the formula.

  3. Child sequence: 
    • Regarding: If this is a new sequence for child records, set this field to the logical name of the lookup field of the parent.
    • Resequence: If set to yes, the plugin will re-sequence the child records when a child record is deleted.

    • Related entities: Parent entities that are referenced in the formula parameters.
