Deploy GCPM

The current GCPM development environment uses the following components:

  • SQL Server: Microsoft SQL Server 2016 (SP1-GDR) (KB4019089) - 13.0.4206.0 (X64) Copyright (c) Microsoft Corporation Developer Edition (64-bit)
  • CRM Version: Microsoft Dynamics 365 Version 1612 (8.2.2.112) (DB 8.2.2.112) on-premises

Technical environment

Pre-requisites - installation will be successful and supported with the following: 

  • CRM Dynamics 365 v8.2 and above

  • ORG account - the server URL and ORG account where the installation will be applied
  • use a CRM system administrator account to do the installation
  • Microsoft Dynamics 365 Software Development Kit (SDK)
  • Microsoft Dynamics CRM PowerShell module installed.
  • AD User (active directory user, becomes the CRM User) to perform installation

GCPM deployment package (Release 1) contains these folders / files:

Crowe 8.10

  • Crowe Licence v8+.txt (file containing licencing information that currently is used for GCPM Development)
  • CroweCore_8_0_10_managed.zip
  • CroweEditAbleCRMGrid_8_0_05_managed.zip
  • EditAble CRM Grid Release Notes.pdf
  • EditAble CRM Grid Support Documentation.chm

DATA

  • Config_release_data.zip – configuration data for GCPM Release 1

  • Lookup Data.zip – lookup data for GCPM Release 1

GCPM

  • Set of managed solutions to be installed as basic Stakeholder functionality for GCPM

  • Help files (this online help can be deployed locally and modified as desired)

Tools

  • EXCLUDED ( a set of managed solutions that are excluded from Release 1)

  • Set of managed solutions to be installed as tools that are required for configuration of GCPM system

Utilities

  • Contains supplementary PowerShell Script to create CRM users and example of input CSV file

To deploy GCPM: 

  1. Create organization (ORG), with a user that has system admin access.
    This is covered in Microsoft Dynamics Administration and Troubleshooting course.
  2. Update parameters in PowerShell scripts to connect to your ORG.
  3. Configure the ORG using the Dynamics 365 administration interface.
    1. Choose the system language
      Go to Settings > Administration > System Settings and on the Formats tab, select English (Canada) or French (Canada)
    2. Block the solution from sending error messages to Microsoft.
      Go to Settings > Administration > Privacy Preferences and select Never send an error report to Microsoft
  4. Install Part 1 of GCPM system by executing Release1-SetOrg-Settings.ps1 script in the root directory. Connect to the ORG using SDK data migration utility.
    1. Import data from Config_release_data.zip from DATA folder.
    2. Import LookupData.zip from DATA folder
  5. Install part 2 of GCPM system by executing Release1-Load_SH.ps1 PowerShell script.
  6. Connect to ORG through web interface as user with System Admin level of access.
    1. Publish duplicate detection rules.
      Navigate to Settings > Data management > Duplicate Detection Rules and publish the rules
    2. Activate business rules.
      Using Advanced Find, search for Process with Business Rule category = Draft

      Select all business rules this search returns, and activate them.

Check the Microsoft Developer Network for general help on managing deployment.*

Once you deploy GCPM and test the environment, you are ready to move to GCPM Implementation Overview.
