Calculated Field Example

 

 

To demonstrate the setting up a calculated field, we’ll assume we are building an EditAble CRM Grid to calculate the line items of a custom Order entity.  The EditAble CRM Grid Columns of note will be the following:

 

·Quantity – A CRM Integer field.  The quantity of the item being ordered.  We will want to add an EditAble CRM Grid On-Change Event to this column.
 

·Cost – A CRM Money field.  The cost per individual item.  We will want to add an EditAble CRM Grid On-Change Event to this column.
 

·Extended Cost – A CRM Money field.  The total cost of the line item, this field will be calculated using the simple formula of Quantity * Cost.  Because this is value is automatically calculated, we’ll want the Extended Cost EditAble CRM Grid Column to be read-only so users cannot overwrite the value.

This function is pretty straightforward.  One thing to notice, though, is the line where we are calculating Extended Cost.  The value we get back from the context may be a string, since it is retrieving the current value of an HTML text box.  This means we need to add the parseInt and parseFloat JavaScript commands to make sure we are correctly calculating numeric values.

 

Another important option of the SetFieldValue function is the fourth parameter.  This is a boolean value that turn on or off validation for the field.  Field validation does a few things:

·Performs data validation of the value.  Since Extended Cost is a Currency field, this will range check the value to make sure it falls between the minimum and maximum allowed values.

·Applies field specific formatting.  Extended Cost is a Currency field, so this will format the value we are setting to the qualified format of the row’s currency.

·Triggers any On Change events of the field.  If Extended Cost had its own On Change event, this would trigger that as well when we call this function.

 

Basically, specifying true is making the field behave the same way it would if it were editable and the user had just manually changed its value.

This shows how important the context parameter can be when working with EditAble CRM Grid Events.  We’re using the context object to find out the new value of the Quantity field as well as what row the user is currently working with.  The RowIndex is important because we need to know which Cost field to retrieve the value of and which Extended Cost field to update.

 

The other thing to note here is that we are sending a true value as the third parameter of the GetFieldValue function.  By default, GetFieldValue will return formatted text – in this case, a formatted currency string (i.e $200.00).  The third parameter toggles the option to return unformatted values, so by sending a true value, we will get 200.00 instead of $200.00.
