Columns

 

 

EditAble CRM Grid supports two types of columns.  In addition to showing standard CRM attributes as columns, you can also create custom columns.  Custom columns are "virtual" columns that display data based on user-defined JavaScript functions.  For details on creating custom columns, see the section in the Advanced Development reference.

 

 

Adding Columns

 

Columns can be added to your EditAble CRM Grid configuration using the configuration wizard or by clicking the New button on the Columns grid.  When added using the Columns grid, you will receive the following dialog to select the basic information for the column:

 

 

 

Select the type of column to add -

 

·CRM Attribute - The column will be based on an attribute of the selected entity.
 

·String Substitution - The column will be a custom column that will concatenate the values of other columns in the grid configuration.
 

·Custom Function - The column will be a custom column that will use a custom JavaScript function to supply its value.

 

See Also

·Configuring String Substitution Columns

·Configuring Custom Columns

 

 

Select the entity that contains the attribute - You can add attributes from the entity your grid configuration is based or related entities.

 

Select the attribute - Select the CRM attribute you want to add.

 

Enter a label for the column - Enter a column label.  You will be able to change this later if needed.

 

Note: You will not be able to change the attribute you selected once you press the OK button.  To change the attribute of an existing column, you will need to delete the column and create a new one.

 

Column Configuration Options

 

General

 

 

Entity Name

The CRM entity that the select attribute is part of.  This is set during the initial column configuration and cannot be changed.

 

Attribute Name

The CRM attribute that the column is based on.  This is set during the initial column configuration and cannot be changed.

 

Label

This is the column heading value that appears in the header row of the EditAble CRM Grid.

Visible

This option enables the visibility of the column.

Column Type

The type of EditAble CRM Grid Column.   This is set during the initial column configuration and cannot be changed.

Column Width

This option specifies the width, in pixels, of the column.

Column Alignment

This option specifies the text alignment of the column.  This will affect all standard EditAble CRM Grid cells, as well as footer and summary row cells.

Header Alignment

This option specifies the text alignment of the column header.

 

 

Behavior

 

 

 

Read Only

Specifies that the column will be read-only when the EditAble CRM Grid is in inline edit mode.

Sortable

Specifies if the EditAble CRM Grid can be dynamically sorted on the column by clicking on the column header.  This option is enabled by default.

Summary Type

Specifies if the column will be summarized.  The summaries will be added to the EditAble CRM Grid’s footer, if Show Footer is enabled, as well as group rows, if Summary Rows are enabled.

·None – The EditAble CRM Grid Column will not be summarized.

·Sum – The contents in the EditAble CRM Grid Column will be summed.

·Count – The contents in the EditAble CRM Grid Column will be counted.  This will count only cells that have a non-blank value.

·Average – The contents of the EditAble CRM Grid Column will be averaged, using Sum / Count.

Groupable

Enables the EditAble CRM Grid Column to be available for Dynamic Grouping.  This option is only available if Allow Dynamic Grouping has been enabled for the EditAble CRM Grid Configuration.

Quick Search Field

Enables the EditAble CRM Grid Column to be used when filtering results using the Quick Filter functionality.

Show Link to Row Record

This option will add a hyperlink to the values of the EditAble CRM Grid Column that, when clicked on, will open the CRM form of the record.

Lookup Options

These options are only available for Lookup-type attributes.

Control Type

Specifies which input control type to use.

·Standard – Uses EditAble CRM Grid’s default Lookup control, which opens a dialog that allows the user to search and select the record.

·Picklist – Uses a pick list control where the primary field of the Lookup entity is used to populate the pick list options.  This control is useful for Lookup fields that have a relatively small number of records to select from, such as reference data.

Disable Hyperlink

By default, a hyperlink is added to EditAble CRM Grid Columns showing Lookup field values that allow the user to open the source record of the Lookup value.  Selecting this option will remove the hyperlink functionality from the current EditAble CRM Grid Column.

Lookup Window Style

This option specifies the style of window the Lookup control will use.

·Dialog – The Lookup is opened as an inline dialog, similar to CRM 2013’s default Lookup window.  Since the dialog is opened within the EditAble CRM Grid frame, it is bound by the height and width of the frame.  This may cause issues in smaller EditAble CRM Grids where the dialog is not fully visible.

·Window – The Lookup is opened as a modal window.  This option does not have the dependencies on the EditAble CRM Grid’s frame size that the Dialog has.

Disable Inline Lookup

This option will disable the inline lookup functionality of the Lookup control, instead immediately opening the full Lookup window.

Advanced Lookup Options

Refer to the Advanced Lookup Options section for more information.

 

Boolean Options

These options are only available to boolean type (two option) attributes.

Control Type

Specifies which input control type to use.

·Checkbox

·Picklist

Custom Field Details

Field Name

Specify a name for the custom field.  The name should be unique within the EditAble CRM Grid configuration.

Function

This option specifies the type of function to use to calculate the value of the custom field.  Refer to the section on Custom Fields in the Advanced Configuration Guide for more information.

Data Type

This option specifies the data type of the custom field.  Refer to the section on Custom Fields in the Advanced Configuration Guide for more information.

Parameters

Specify the parameters to pass to the function.  Refer to the section on Custom Fields in the Advanced Configuration Guide for more information.

Function Name

The name of the JavaScript function to call.  Refer to the section on Custom Fields in the Advanced Configuration Guide for more information.

 

 

Advanced Options - Formatting

 

 

Display Mask

This option is used to control the output style of some custom column data types.
 

·Default – No additional or alternative formatting of the values.
 

·Percentage – The value is formatted with a “%” symbol.  For example, a decimal attribute with a value of 99.99 in CRM could be represented as “99.99%” on the EditAble CRM Grid.

·Currency – The value is formatted as a currency/money field.  This will use the current default currency of the EditAble CRM Grid.

·Decimal – The value is formatted as a decimal field.  The default precision of your CRM organization will be used.

 

Precision

This option allows you to override the precision for numeric attributes.

 

 

Advanced Options - Miscellaneous

 

 

Empty Value Text

You can use this setting to specify a substitution value if the field retrieved from CRM is blank.  This is useful if you are going to use the field for Grouping.  For example, you could enter “[No Value]” here and records that don’t have a value for the field will be grouped under a heading of “[No Value]” instead of a blank string (“”).

Primary Proxy Sort Column

EditAble CRM Grid’s Proxy Sort function allows you to specify an alternative field to sort on instead of the actual field you are configuring.  Basically, this allows to create custom sorting definitions for any field.  For example, you may want to sort records by Opportunity Stage but you want the sort to be based on sequencing of your Stages as they related to your Sales Process, and not just sorted alphabetically.

Secondary Proxy Sort Column

The Secondary Proxy Sort Column is provided as additional functionality if you have more complex sorting requirements.

Let’s say you have a Custom Field you want to sort on.  It is a calculated field that combines a date value with a string value (i.e. “1/15/2013 – EditAble CRM Grid Documentation”).  If specify the source date field as the Primary Proxy Sort Column, it would not sort items with the same date correctly.  In this case, you want to add a Secondary Proxy Sort Column that references the source field for the string (“EditAble CRM Grid Documentation”).
