Grid criteria

 

 

Specifying criteria for your EditAble CRM Grid configuration allows you to filter the records it shows, and is a similar process to creating an Advanced Find or view in Dynamics CRM.  Specifying criteria for your configuration is not required.  However, if you selected "Based on specific criteria" as the grid's Criteria Type, not specifying any criteria will result in all records of that entity type being loaded.

 

Note: If your grid's Criteria Type is "Related to a parent record", the link to the parent record is automatically added as a criteria condition.  You can still specify additional criteria to filter the records further.  For example, you may only want to show Active records.

 

Adding and modifying criteria is done using the Criteria Builder window, which can be opened from the Criteria button on the command bar:

 

 

 

If you have not added criteria yet, the Criteria Builder window will be empty:

 

 

 

If you have used CRM's Advanced Find, the process of adding criteria should be familiar.  When you click "SELECT",  you will get a list of all of the entity's fields that are available to use:

 

 

 

If you scroll down to the bottom of this list, you will also see a list of related entities.  Fields from related entities can also be used as criteria conditions. The EditAble Grid supports N:1 and 1:N Relationships.

 

 

 

Once you select the field you want to use, you will be able to define the rest of the criteria condition.

 

 

In this example, I've selected the "Status" field because I want the EditAble CRM Grid to only show records with a Status of "Active".  In addition to the field, there are three other components to the criteria condition:

 

 

Operator

The operator is where you specify how the selected field should be filtered against.  Depending on the data type of the select field, the list of available operators will be different.  Refer the list of available operators for details.

 

 

Type of Value & Value

 

Type of value:

 

Value:

 

Together, the "Type of Value" and "Value" determine what the comparison value is for the condition.  You will typically leave the default option ("Literal / Static Value") selected, unless you are creating a Dynamic Filter or advanced EditAble CRM Grid configuration.

 

The Value has different meaning depending on what option is selected for Type of Value.  The following options are available:

          

Literal/Static Value

This option will use the specific value that is entered.  This is the default option, and the most typical for EditAble CRM Grid configurations.

 

For most types of fields, the value that is entered will be included in the condition as-is.  However, fields with complex data types allow for some additional flexibility.  Lookup-type fields (e.g. Entity Reference, Owner, Regarding) and Option Set-type fields (e.g. Option Set, Status, Status Reason) allow you to specify either the actual value or the display value to use in the condition.

 

In my example of showing only "Active" records, I could enter either "0" or "Active".

 

Note: If you have an Option Set field that uses integer/whole number values as the display value, entering the display value in the condition may not be reliable.

Note: To support multi-format environments, Date values should be entered in the 'yyyy-MM-dd' format.

 

 

Dynamic Filter

This option uses the value selected from a dynamic filter on the EditAble CRM Grid filter panel.  See configuring dynamic filters for more information.

 

 

URL Data Argument

This is an advanced option.  Using this, EditAble CRM Grid will look for the value in the "data" segment of its URL.  This allows you to create a grid configuration that uses condition values that are more dynamic and/or automatically specified.  When this option is specified, the "Value" entered is the name of the argument in the URL's data segment.  If the argument is not found in the URL, the condition is ignored.

 

See the the section in the Advanced Development sections for more information.

 

 

 

Advanced Criteria Options

 

You can access advanced options for applicable conditions by clicking the "..." button at the end of the row, if available:

 

 

·Substitution Variable - Allows you to specify a name to use to refer to the condition in advanced filters.
 

·Allow Multiple Values - If selected, allows you to specify a delimited list of values.  Each value in the list will be evaluated depending on the selected operator.  For example, a condition with an operator of "Equal To" and a literal/static value of "1,2,4" will be evaluated as "is one of the following: 1, 2, or 4".
 

·Delimiter - Allows you to specify the delimiter character that separates each value in multiple value type conditions.  If a delimiter is not specified, comma (",") will be used.
