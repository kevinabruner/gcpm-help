Advanced settings

 

 

There are some advanced settings that can be added to EditAble CRM Grid via the URL "data" argument.  Note that depending on where you are implementing EditAble CRM Grid, the values of the data argument may need to be URL encoded.

 

Each setting you are using gets added to the URL "data" argument.  You must use "&" between each setting when you have more than one.  Note that the EditAble CRM Grid configuration name counts as a setting, and should always be the first one listed.

 

URL Settings

 

config

Specifies the EditAble CRM Grid configuration to use.  Omitting the "config" keyword and just entering the configuration name is a valid shortcut.

Example:  config=My%20Accout%Grid


 

from

Specifies where the EditAble CRM Grid configuration should be retrieved from.  Valid "from" values are:
 

·crm - Retrieves the published version of the EditAble CRM Grid configuration.  This is the default option and does not need to be specified.  To use the published grid configuration, you can simply leave out the "from" setting.
 

·live - Retrieves the real-time/unpublished version of the EditAble CRM Grid configuration

 

Example: from=live

 

 

wait

Specifies an amount of time, in seconds, for EditAble CRM Grid to wait before loading.  One potential use of "wait" is to stagger the initialization of EditAble CRM Grids on a CRM form.  For example, you could may have grids on the form that are hidden or at the bottom of the page.  You could use "wait" to delay when those grids are loaded so they are not taking up valuable browser resources that your visible grids need.

 

Example (waits 1.5 seconds):  wait=1.5

 

 

options

Allows you to override some of the EditAble CRM Grid configuration options. The following options are currently available:

 

·readonly - Sets whether the EditAble CRM Grid will be read-only (true/false, or 1/0).
 

·toolbar - Sets the Show Toolbar option (true/false, or 1/0).

 

 

The individual options are expressed in a name:value pair.  To specify multiple options, use a comma to separate the options.

 

Example:   options=readonly:1,toolbar:0

 

Note that boolean options will correctly translate both true/false values and 1/0 values.
