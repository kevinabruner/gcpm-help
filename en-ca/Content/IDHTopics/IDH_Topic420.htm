Configuring dependent EditAble CRM Grids

 

 

There may be instances where you'd like to setup a second EditAble Grid where the results displayed are filtered based on the record selected in the first Grid.  We call this second Grid a Dependent Grid. 

 

For example, you could have an EditAble CRM Grid on your Account form that shows Orders for the Account:

 

 

 

 

You could then create a second EditAble CRM Grid that shows Order Detail records, and place it on the Account form next to the Orders EditAble CRM Grid:

 

 

 

 

 

Then, by using the EditAble CRM Grid API, load the Order Details EditAble CRM Grid based on the Order record the user selects.  This is achieved by using the LoadDependentGrid API function call, executed from a On Row Select event.

 

 

 

 

 

Let's walk through the process of configuring a Dependent EditAble CRM Grid.  In this example we will reference our first, or parent, grid as Grid1 and our child, or dependent, grid as Grid2.  When a record is selected in Grid1 we want the records in Grid2 to be filtered on the selected row.

 

Create and configure Grid1.  This grid can be configured as either Based on a specific filter or Related to a parent record.  Use whichever option suits your needs best.  Save, Publish, and add Grid1 to the CRM form.

 

Next, create and configure Grid2.  This grid must be configured as Related to a parent record.  Also, be sure to set Is Dependent Grid option to Yes.  The rest of the grid may be configured as needed.  Save and Publish Grid2.

 

 

 

 

Add Grid2 to the CRM Form.  Make note of the 'Name' of the Web Resource, as this will be referenced in JavaScript. 

 

 

 

 

 

Also, do not check the option to pass the unique identifier.  We will pass this in JavaScript code when the user selects a row in Grid1.

 

 

 

 

Create, or use an existing, JavaScript Web Resource.  If you are unfamiliar with this process, see the section Extending EditAble CRM Grids - Adding Custom Javascript Functions.  In the JavaScript Web Resource we will add an On Row Select function that loads the dependent grid. In Grid1, add an On Row Select EditAble CRM Grid Event.  Select the JavaScript Web Resource for Grid1 and specify the On Row Select function we created.  Save and Publish Grid1.

 

 

 

You've now configured an EditAble CRM Grid with a Dependent Grid.  By default Grid2 will display the message "To enable this content, save the record" before a record in Grid1 has been selected.  In the Grid2 Configuration under Additional Options - Advanced Options & Overrides, you can specify an Initial Message that is more relevant to the Dependent Grid.  The EditAble CRM Grid will display the defined message when a record has not been selected in Grid1.
