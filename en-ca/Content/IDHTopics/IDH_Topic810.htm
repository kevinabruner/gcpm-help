Configuration Options

 

 

The EditAble CRM Grid configuration options are grouped into several areas.

 

 

General

 

 

Name

The name of the EditAble CRM Grid configuration.  This is the how you will refer to the grid when adding it to CRM.  The Name should be unique to ensure that CRM loads the correct configuration.

 

 

Description

An optional description of the EditAble CRM Grid configuration. 

 

 

Entity

The CRM entity that the EditAble CRM Grid will represent.  This gets set during the initial configuration wizard and cannot be changed on an existing configuration.

 

Criteria Type
The criteria type allows you to specify how you want the EditAble CRM Grid to query its results.  To change the type of an existing configuration, use the Grid Definition button on the command bar to open the configuration wizard.

 

Parent Entity Name
Available only when Show Related Records is selected as the Criteria Type.  This is the parent entity to use for related records and can be set or changed using the Grid Definition button on the command bar to open the configuration wizard.

 

Parent Entity Key
Available only when Show Related Records is selected as the Criteria Type.  This is the lookup field to the parent entity being used for related records and can be set or changed using the Grid Definition button on the command bar to open the configuration wizard.

 

Row Functionality

 

New Rows

This option specifies how you want the EditAble CRM Grid to handle “New” record requests.

·Create Inline – New records are created inline in the EditAble CRM Grid.

·Use CRM Form – New records are created using the entity’s standard CRM form.

·Allow Both – Users will have the option of creating new records using inline editing or the entity’s standard CRM form.  This is accomplished using a drop down menu when clicking the New button on the toolbar.

·Disabled – This option will hide the “New” button and disable the ability to create new records directly from the EditAble CRM Grid.

 

Enable Quick Create

This option enables the use of the CRM Quick Create form. This option is only available and displayed in CRM 2016 and later. Enabling this option will override the default behavior of opening the standard CRM form for New Rows when New Rows is set to Use CRM Form or Allow Both, and utilize the Quick Create form instead. A Quick Create form must be available for the entity, otherwise this option should be set to Disabled.

 

Edit Rows

This option specifies how you want the EditAble CRM Grid to handle “Edit” record requests.

·Edit Inline – Records are able to be edited inline in the EditAble CRM Grid.

·Use CRM Form – The entity’s standard CRM form will open to allow the user to update the record.

·Allow Both – Users will have the option of editing records using inline editing or the entity’s standard CRM form.  This is accomplished using a drop down menu when clicking the Edit button on the toolbar.

·Disabled – This option will hide the “Edit” button and can be used to disable the ability to edit records in the EditAble CRM Grid, depending on the Double Click Action option selected.

 

Delete Rows

This option specifies how you want the EditAble CRM Grid to handle “Delete” record requests.

·Use Toolbar Button – Shows a “Delete” button on the EditAble CRM Grid toolbar.

·Disabled – This option will disable the ability to delete records directly from the EditAble CRM Grid.

 

Double Click Action

This option allows you to specify what the EditAble CRM Grid will do when a user double clicks a row.

·Edit Inline – The row will enter inline edit mode.

·Open CRM Form – The records CRM form will be opened.  This would be similar to how a standard CRM view works.

·Disabled – The double-click event will be disabled.

 

New Row Location

Available only when the New Rows option is set to Create Inline.  This option specifies where you want the new blank row to be positioned in the grid.

 

New Row Attribute Mappings

Available only when the New Rows option is set to Create Inline or Allow Both, and the Grid Configuration Criteria Type is set to Show Related Records.  This option, when enabled, automatically retrieves and applies the CRM Attribute Mappings for the parent-child relationship when a new record is added Inline. Previously, the use of the EditAble CRM Grid API and CRM SDK libraries were required to populate these values during the On New Event.

 

Row Multi-Select

Enable or disable the ability for users to select multiple rows in the EditAble CRM Grid.

 

Display Options

Row Checkboxes

Toggles row checkboxes on or off.  If turned on, an additional checkbox is added to the column headers that allows users to select/unselect all rows (if Row Multi-Select is enabled).

 

Show Header

Toggles the EditAble CRM Grid column headings on or off.

 

Show Footer

Toggles the EditAble CRM Grid footer row on or off.

 

Show Title

Toggles a title area for the EditAble CRM Grid on or off.  Use the Grid Title option to specify the title you want to show.

 

Grid Title

Available only when Show Title is enabled.  Use this to specify the title you want to appear on the EditAble CRM Grid.

 

Border

This option allows you to specify a border style for the EditAble CRM Grid control.  Note that these options do not relate to the “Display Border” setting of CRM form’s Web Resource frames.  It is recommended to always set that option to not show a border and let EditAble CRM Grid manage its own border styles.

·None – The EditAble CRM Grid will display without a border.  This matches CRM 2013’s border-less styles.

·Normal – A single line border

·Shadow – A single line border with a shadow effect, similar to some CRM 2013 control styles.

·Shadow on Focus – A combination of the None and Shadow options.  The EditAble CRM Grid will have no border until a user start using it, at which point it will switch to the Shadow border style.

 

Show Toolbar

Toggles the EditAble CRM Grid toolbar on or off.

 

Show Status Bar

Toggles the EditAble CRM Grid status bar on or off.  The status bar is the area at the bottom of the grid that shows information like the number of records returned.

The status bar is also where the paging controls will appear if paging is enabled on the EditAble CRM Grid.

 

Columns

The Columns area is where you can add new columns to your EditAble CRM Grid configuration or modify existing ones.  See the Columns section for more information.

 

Sort by (Primary Sort Field & Sort Direction)

Specifies the primary EditAble CRM Grid Column to sort on and direction of the sort (ascending or descending).

 

Then by (Secondary Sort Field & Sort Direction)

Specifies the secondary EditAble CRM Grid Column to sort on and direction of the sort (ascending or descending).

 

Grouping

Enable Grouping

Enables grouping functionality in the EditAble CRM Grid.  Turning this option on will enable several other grouping related options.

 

Dynamic Grouping

This option will enable dynamic grouping on the EditAble CRM Grid, which allows the user to change the field(s) that the results are grouped by.  You can specify which columns of your grid to allow for dynamic grouping in the EditAble CRM Grid Column definition.

 

Group Behavior

This option specifies how you want the groups to work.

·Collapsible, Expanded by Default – Enables collapsible groups and shows them expanded initially.

·Collapsible, Collapsed by Default – Enables collapsible groups and shows them collapsed initially.

·Not Collapsible – Disables collapsing groups.  As such, the groups will always show as expanded

·Only Show Groups (Hide Details) – Only shows the group rows while hiding all the detail rows.

 

Group Options

Use the Group Options area to specify details about the each group level.

Column

Specifies the column to use as the group value.

 

Sort Order

Specifies the sort order of the group value.

 

Summary Row

Turning this option on will enable support of summary rows (i.e. sub-totals) on the group rows.

 

Show Count

Turning this option on will append a count of the grouped records to the group value.

 

Features

 

Quick Copy

This option will enable a “Copy Record” button on the EditAble CRM Grid toolbar that will create a new inline row and pre-populate it with the values of the selected row.

 

Quick Search

This option will display a search box on the toolbar that can be used to quickly filter the current result set.  You can define which columns gets searched against in the EditAble CRM Grid Column definition.

 


Save and New

This option will add a “Save and New” button to the EditAble CRM Grid that will allow the user to save the current row and immediately show a new row.

 

Currency Conversion

This option will enable a toolbar button that allows users to change the default currency of the EditAble CRM Grid.  Any Currency that has been added to your CRM organization will be available.

 

Multi-Currency Support

Use this option to specify how the EditAble CRM Grid handles displaying multiple currencies.

·Normalize Currency for Display – This option will automatically normalize all currency values to the user’s default currency, or the CRM organization’s default currency if the user has not specified one.

·Show Actual Currencies – This option displays the actual currency for each row.  However, if you have configured summary totals for any Money fields, the totals will be normalized to your default currency.

 

Paging

This option specifies if you want the grid to page results.  This can be useful for large result sets, however since all of the records are not loaded at the same time it can have adverse effects when combined with other features of the EditAble CRM Grid, such as grouping and totaling columns.

 

Records Per Page

If Paging is enabled, you will also have the ability to specify how many records are displayed on each page of results.

·Based on User Setting - This option will use the user's "Records Per Page" setting that is defined in their personal options in Dynamics CRM:

·Specific Amount - This option allows you to specify a specific number of records per page, regardless of what the user's personal "Records Per Page" setting is.

 

Pop Out

This option will enable a button on the EditAble CRM Grid toolbar that will allow the grid to be “popped out” to a separate window.

 

Required Fields

This option specifies how the EditAble CRM Grid will denote required fields during inline editing.

 

Enable Export

This option will enable an Export button on the toolbar that allows the user to export the results in the EditAble CRM Grid to Excel. 

 

Auto-Read Only Mode

This option will automatically set the grid to read only mode if it is on a CRM form that is read only.

 

Advanced Options & Overrides

 

Auto-Publish

Enabling Auto Publish will cause the EditAble CRM Grid configuration to be published automatically whenever there is a change to the configuration.  Note that some larger or more complex EditAble CRM Grid configurations can take several seconds to publish and it is not recommended to enable Auto Publish in those scenarios.

 

Privilege Handling

This option specifies if you want the EditAble CRM Grid to do proactive user privilege validation during the initial load of the grid.

By default, the EditAble CRM Grid uses the Security Roles in your CRM organization to determine the privileges of the current user.  This information is used to control various elements of the EditAble CRM Grid interface.  For example, if a user does not have create privileges on the entity they will not see a “New” record button on the EditAble CRM Grid toolbar.
In some scenarios, loading this information can add several seconds to the initialization time of the EditAble CRM Grid and this option is useful for debugging those cases.  Turning this option off does not compromise or bypass CRM’s built-in security model in any way.  Rather, the EditAble CRM Grid will handle security passively.  In the example above, a user that does not have create privileges on an entity would still see the New button on the toolbar.  However, if they try to add a new record they will receive an error message explaining that they do not have privileges to do so.

 

Initial Query

This option specifies when the EditAble CRM Grid should execute its initial query.

·Load Immediately – The EditAble CRM Grid will execute its initial query as soon as all required dependencies have been initialized.

·Partial Delay – The EditAble CRM Grid will delay briefly before executing its initial query.  This can allow other non-EditAble CRM Grid resources to load.

·Wait for Filter – If a Filter Panel has been configured, the EditAble CRM Grid will not execute a query until the filter has been committed.

 

Initial Message

This allows you to change the default message in an empty inactive EditAble CRM Grid.

 

Empty Results Message

This allows you to change the message that appears in the EditAble CRM Grid when no records are returned from its query.

 

Is Dependent Grid

Set this to Yes if the current EditAble CRM Grid is intended to be dependent on another EditAble CRM Grid.

 

Open Forms In

This option specifies how the EditAble CRM Grid should open CRM forms.

·New Window – CRM forms launched from the EditAble CRM Grid will open in a new/separate browser window. This option also enables the “Refresh After Forms Close” option.

·Same Window – CRM forms launched from the EditAble CRM Grid will open in the same browser window.

 

Refresh After Forms Close

This option will enable or disable the EditAble CRM Grid from automatically refreshing when a CRM form that was launched from the EditAble CRM Grid is closed.  For example, if you have New Rows set to Use CRM Form, when a new record is created the EditAble CRM Grid will automatically refresh to show the new record that was created.

This option only works when the “Open Forms In” option is set to New Window.

Note: This option has no effect in the CRM for Outlook client, due to differences in how the client works.

 

Keyboard Shortcuts

This option enables and disables support for keyboard shortcuts when the EditAble CRM Grid is inline edit mode. Currently, this applies to the use of arrow keys to navigate between records in edit mode. In a future release, additional keyboard shortcuts will be added to augment and improve those inherent to the web browser's capabilities.

Note: Out of the box Web Browser (IE, Chrome, Firefox) behavior when using the keyboard up and down arrows while an option set / select control has focus is to change the value of the option set. When Keyboard Shortcuts are enabled, if a select control has focus grid will navigate between records, however, this browser behavior will continue to occur, changing the value of the control. This is currently outside the scope of the EditAble Grid. If affected, we recommend disabling Keyboard Shortcuts.

 

Filter Panel

 

Show Filter Panel

Enables the use of the EditAble CRM Grid Filter Panel.  The Filter Panel can be used to add real-time filters to your EditAble CRM Grid.  The Filter Panel will be shown above the EditAble CRM Grid.

See Also:

·Grid Criteria: Filters

 

Filter Panel Visibility

This option specifies the initial visibility of the Filter Panel.

·Show by Default – The Filter Panel will be collapsible, but visible/expanded initially.

·Hide by Default – The Filter Panel will be collapsible, but hidden/collapsed initially.

 

Custom Events

Custom Events

You can use this option to disable any custom EditAble CRM Grid Events you have added to the EditAble CRM Grid.  This can be useful when troubleshooting issues that may be caused by your custom events.

 

Fire On Load Event

This option specifies when the EditAble Grid will fire the On Load Event.

·          As Needed (Default) – The EditAble Grid will determine after which events an On Load Event is required.

·After Every Transaction – This will fire the On Load Event after every transaction occurs. This can be useful for code that modifies non-persistent grid elements e.g. code that sets the styles of EditAble Grid cells to ensure the style is applied after every save.
