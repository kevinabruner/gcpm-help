About assessment questions

Assessment questions ensure an officer's due diligence and assessment of a project is captured, and can be incorporated into communications with the client, in project approval forms, and during the monitoring phase of the project, if applicable. Assessment templates ensure consistent assessment across projects: assessment questions are copied to projects with the clone tool.

Assessment questions can be configured to appear in appropriate locations on the project form (e.g. cost questions appear in a grid below the cost grid, etc.). There are 5 default assessment question grids: project financing, financial analysis, project costs, previous assistance and repayment considerations.

Assessment questions can be re-used in different programs.

Scoring assessments

Assessment questions can have scores associated with each response option. This facilitates providing an overall score to an assessment, and scores can be used to rank projects from highest to lowest score.

For example, a "yes" response could score higher than a "no" response.

An overall assessment score facilitates comparing one project to another, or choosing the highest scoring projects. Questions can be required or optional, and comments can be required based on chosen responses. A manual override is available when the scoring does not reflect certain factors that are not built into the assessment.

Automation

  • Questions are automatically copied from program template when an assessment is created, using the clone tool.

Design considerations

  • Assessment questions have been separated into 5 grids to showcase the potential of having multiple grids based on one list of questions.

Backlog / future development:

  • Consider adding a field (e.g. repayable\ non-repayable. Use this field to act as a filter to display assessment questions related to repayability only to for-profit organizations.
