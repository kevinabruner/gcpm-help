Configure assessment values

Assessment values are the answers available for assessment questions.

To configure assessment values: 

  1. In an Assessment question | A, click + in the Assessment values grid.

  2. Create a new assessment value. Complete the fields using onscreen help and the notes below.

    • Name: Provide a name for the assessment value (EN and FR).
    • Text: (optional) provide additional text, if needed.
    • The value can either have a Numeric value or a binary option Met / not met.
      • A numeric value can be leveraged in these ways:
        • Can be used in document generation. e.g. Are advances allowed for client? If "yes", easier to reference value instead of text.
          e.g. CEAA - can have 4 outcomes. Each combination of responses triggers different set of clauses in CA. - why you need numeric value. can conbmine with met/not met in order to require comments, send on different BPF, etc.
        • Can be used to trigger validation actions.
        • Can provide a ranking (e.g. score individual questions, and score overall project assessment)
      • Met / Not met: Indicates if the assessment question is met or not met (e.g. yes / no).
        This field can be used to trigger additional questions, e.g if Question A is not met, must answer Question A.1 and A.2.
    • Comment required: choose Yes if the officer must enter a comment when this value is chosen. Typically use this option if the "Not Met" response is selected.
    • Assessment question: Displays the question associated with this assessment value.
  3. Save and close the assessment value record.
  4. Continue creating assessment values for the question.

Notes: 

  • An assessment value can only be associated with one Assessment Question | A. Each question requires a unique assessment value. For example, for the assessment question "Is the applicant an eligible recipient?" the values could be Yes and No. For each question that has Yes / No options, a Yes and No value must be created.
  • If you enter a numeric value for a question, you can use the number for these purposes: 
    • Configuring a document (e.g. defines CA clauses, project approval form, letters to clients...)
    • Reporting (e.g. advanced pipeline ranking, impact in conjunction with risk, etc.)
    • Influencing the next step of a business process flow
    • Use assessment values when you want a high level of control over the responses
