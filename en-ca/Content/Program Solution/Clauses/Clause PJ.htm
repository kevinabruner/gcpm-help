Modify project clauses

The clauses configured on the default program clause template are copied to new projects created in the program. These clauses may be viewed and edited by officers, depending on clause template configuration settings.

The project clauses can be viewed on the project form, or in the associated Clause view.

If officers have been granted appropriate rights, they can add new clauses to the project (either from the Clauses | A list that are not in the template, or completely new clauses). These clauses can appear in the contribution agreement (where is this permission granted?). Why would they add clauses that are not in the CA?

Give example of when an officer might add a clause during assessment - either modify or create new.

To add a clause to a project during assessment that will appear in the contribution agreement:

  1. Open the Clauses associated view on the project.
    The CA clauses grid appears.

  2. Click + Add New Clause | PJ to add a new clause. Use the help below to configure the fields.
    • Parent project clause: Choose a clause from the drop down list (Clause | PJ), or create a new one.
      Note that sometimes a parent clause is created to act as a header for clauses related to a given topic.
    • Clause: Choose a clause from the drop down list (Clause | A), or create a new one.
    • Free text clause: ??
    • Name: Provide a name for the clause
    • Order: Indicate the order of the clause (read here for more detail)
    • Add to CA: Indicates if clause should be added to CA (contribution agreement).
      If No is selected, provide a rationale.
    • Rationale for no: This field only appears if Add to CA = No. Provide a reason why the clause should not appear in the CA.
    • Wording: Provide the wording for the clause (English and French).
      • If you choose an existing clause in the Clause field, the text of the field appears under the Clause heading. You can copy and paste this text into to the Wording field, and modify as needed.
  3. Save and close the record.

Note: Duplicate detection is configured on projects to prevent a clause being added more than once to a project.
