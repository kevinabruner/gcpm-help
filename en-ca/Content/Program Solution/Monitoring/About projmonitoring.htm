About monitoring

Client and project monitoring are an important activity in successfully managing programs and achieving expected results. The GCPM monitoring feature provides oversight of client and project monitoring activities. As officers conduct risk assessments, review financial statements, and track performance, these activities can be tracked as monitoring items in a monitoring activity, to provide a single reference for reporting and audit requirements.

Activities such as risk assessments, review of financial statements, performance measurement and internal controls can be added to a project monitoring record as a review item. A project monitoring record can contain multiple review items.

 

Officers can switch between project-specific reviews and client-centric oversight. GCPM monitoring brings together several GCPM features (risk, financial review, performance results) and can include internal controls through the use of checklists.

  • GC Client monitoring A client monitoring activity records reviews of a client’s financial statements, and client risk assessments.
  • Project monitoring A project monitoring activity can record multiple review items related to project monitoring. For example, a monitoring record for a project tracks the frequency for assessing risk, and the date of the next scheduled assessment. Each risk assessment that is completed is added as a review item on the monitoring record. Other monitoring items may include financial statement due date, annual reporting on performance indicators.

While review items are attached to a monitoring activity, the officer would create the individual assessments or reviews independently in GCPM for project items. The monitoring activity and review items are meant to provide RDA’s with a way of capturing their monitoring practices in one area to demonstrate due diligence and provide an audit trail.

Your organization can locally configure checklist questions to accompany each monitoring activity. This way, you can capture records for officer attestations, or compliance with departmental policies / procedures.

Design considerations

 

Automation: None

Associated tools

Project Review Items – Open ended tool for officers to create individual records related to the activities they are undertaken as part of a project monitoring. Examples for required monitoring on an annual basis would be: review item related to a risk assessment, review item for required performance reports, review item to ensure the audited financial statements are received, and review item related to a required annual site visit. As each of these are completed the officer would indicate the completion date for each item before closing/completing the monitoring activity. These review items also appear on the client’s social pane as outstanding items allowing anyone viewing the clients profile to see that these activities are underway.
