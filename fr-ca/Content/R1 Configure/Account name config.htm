Configurer le nom du compte

Le nom du compte par défaut est configuré de sorte à afficher le nom d'exploitation (EN). Bien qu'il soit possible de configurer la plupart des noms des entités dans GPSC de manière à les afficher dans la langue de préférence de l'utilisateur (p. ex., le nom en FR dans le cas des utilisateurs qui choisissent le français et le nom EN dans celui des utilisateurs qui optent pour l'anglais), le nom du compte ne suit pas cette approche. Voir "Pourquoi?" ci-dessous pour de plus amples renseignements.

Pour modifier la manière dont s'affiche le nom de compte, modifier la configuration du nom du compte : 
L'exemple indique comment configurer le nom du compte de sorte à employer le nom d'exploitation (FR).

  1. À l'aide de Recherche avancée, cherchez l'entité CFG Calculs.
    Autrement, furetez vers la vignette Configuration et choisissez l'icône CFG Calculs.
  2. Ouvrez l'enregistrement Compte - Nom.
  3. Permutez les paramètres de formule pour qu'ils coïncident avec vos exigences.
    Par exemple, pour modifier le nom par défaut de sorte à utiliser le nom d'exploitation (FR), au lieu de l'anglais, modifiez les paramètres de formule à : egcs_operatingname_fr, egcs_operatingname_en.
  4. Sauvegardez et fermer l'enregistrement de la configuration de Compte - Nom.
  5. Vérifiez si le nom du compte paraît dorénavant de la manière souhaitée.
Pourquoi le nom du compte est-il configuré autrement que d'autres entités?

Les rapports et les tableaux de bord affichent les noms français et anglais d'une entité. Si le nom du compte est configuré de sorte à afficher le nom en EN et en FR, les étiquettes sont très longues et compliquées.

Plus au sujet des paramètres

Le champ calculé lit les paramètres en tant que Paramètre 0 et Paramètre 1 pour le nom_en et le nom_fr respectivement. Si vous permutez l'ordre des paramètres qui change qui change la façon dont le paramètre est employé dans la formule. Si les paramètres ne sont pas vides et ne sont pas identiques, il mémorisera le nom avec la structure en pipeline “Anglais | Français”.

Puis, il vérifie pour voir si le nom anglais est vide et si le nom français est inscrit. Si le nom anglais est vide, seul le nom français sera mis en mémoire; sinon, c'est le nom anglais qui sera mémorisé.

La permutation de l'ordre des paramètres

Le fait de mettre le nom_fr en premier indique simplement au programme que le paramètre 0 est le nom_fr au lieu du nom_en, comme c'était le cas à l'origine. Le nom français sera affiché avant le pipeline au lieu du nom anglais.

Nom_en, suivi du nom _fr = “Anglais | Français”

Nom_fr, suivi du nom _en = “Français | Anglais”
