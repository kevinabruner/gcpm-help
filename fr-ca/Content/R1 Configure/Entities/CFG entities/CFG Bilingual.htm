Configurer les enregistrements CFG Bilingue

Les champs GPSC peuvent configurées afin que l'affichage soit anglais ou français selon la langue de choix de l'utilisateur (défini dans les paramètres de l’utilisateur*). Voir exemple ci-dessous.

Pour configurer les champs afin d'afficher la langue de choix de l'utilisateur :

  1. Dans recherche avancée, chercher CFG Bilingue et créer un nouvel enregistrement pour le champ qui devrait afficher l'anglais ou le français.

    • Le Nom n'a pas de format, mais il doit être unique.
      Conseil: utiliser une norme, comme le nom d'une entité à traduire suivi par le mot traduction. Dans l'exemple ci-dessus, Nom = Traduction de type d'entreprise
    • Entité cible: le nom logique de l'entité dont vous souhaitez mettre sur pied la traduction.
      Dans notre exemple, Entité cible = egcs_sh_businessstructure
      Note: ce champs est sensible à la casse.
    • Champs à traduire : une liste de champs à traduire délimitée par des virgules (utiliser un nom logique).
  2. Enregistrer et fermer l'enregistrement.

CFG Bilingue travaille de concert avec le module d'extension CFG Calcul afin d'afficher la langue de l'utilisateur pour les champs bilingues, au besoin. Voir À propos du plugin CFG Bilingue

Exemple

Par exemple, les enregistrements de type d'entreprise comprennent les noms anglais et français pour les types d'entreprise:

  • Nom AN=University
  • Nom FR=Université

Le mot « university » s'affiche si l'utilisateur a l'option de langue anglaise et « Université » s'affiche si l'utilisateur sélectionne l'option de langue française.

Remarques :

  • Le module d'extension est conçu pour traduire les champs qui ont une barre verticale | délimitant les renseignements anglais et français. S'il n' y a pas de barre dans les données, il ne les traduira pas. Cela peut être particulièrement difficile puisque le plugin se déclenchera à chaque fois que l'entité et le champ sont téléchargés - vous ne verrez pas du tout l'autre langue s'il est présent. Par contre, cela ne change pas les données sous-jacentes.
  • Le module d'extension ne traduit seulement quand l'entité et le champ sont téléchargées. Les données contenues dans une recherche sont, par exemple, cette entité qui télécharge ce champ lorsque vous êtes dans un formulaire (disons une province qui modifie un enregistrement de contacts). Afin de faire cette traduction, vous avez besoin de le configurer dans la PROVINCE et non dans le contact. Par contre, le contraire est vrai lorsque vous consultez un ensemble d'enregistrements (contacts actifs pour utiliser l'exemple ci-dessus) qui est seulement le contact qui est téléchargé afin de traduire la province dont vous avez besoin de configurer la traduction du champ de la province sur le contact.

Conseil : L'approche la plus sûre est de faire la configuration pour chaque entité où le champ à traduire se produit. Un bon exemple est de consulter la traduction su compte puisqu'il a de nombreux champs qui sont des recherches.
