Déployer GPSC

L’environnement de déploiement de la GPSC utilise actuellement les composantes suivantes :

  • SQL Server : Microsoft SQL Server 2016 (SP1-GDR) (KB4019089) - 13.0.4206.0 (X64) Copyright (c) Microsoft Corporation Developer Edition (64-bit)
  • Version CRM : Microsoft Dynamics 365 Version 1612 (8.2.2.112) (DB 8.2.2.112) sur place

Afin de déployer la version 1 de la GPSC, vous aurez besoin d'installer :

  • Microsoft Dynamics 365 trousse de développement logiciel (SDK)
  • Microsoft Dynamics CRM module PowerShell.
  • Utilisateur AD (utilisateur d'active directory, deviens l'utilisateur CRM) afin d'effectuer l'installation

Trousse de déploiement de la GPSC (version 1) contient ces dossiers et fichiers :

Crowe 8.10

  • Licence Crowe v8+.txt (fichier qui contient les renseignements de la licence est utilisé pour le développement de la GPSC)
  • CroweCore_8_0_10_managed.zip
  • CroweEditAbleCRMGrid_8_0_05_managed.zip
  • EditAble CRM Grid Release Notes.pdf
  • EditAble CRM Grid Support Documentation.chm

DONNÉES

  • Config_release_data.zip – Données de configuration pour la version 1 de GPSC

  • Lookup Data.zip – Données de recherche pour la version 1 de la GPSC

GPSC

  • Ensemble de solutions gérées à être installé en tant que fonctionnalité de base pour les intervenants pour la GPSC.

  • Dossiers d'aide (cette aide en ligne peut être déployée localement et modifiée au choix)

Outils

  • « EXCLUDED » (EXCLUS) (un ensemble de solutions gérées qui sont exclues de la version 1)

  • Ensemble de solutions gérées à être installés en tant qu'outils qui sont nécessaires à la configuration du système de la GPSC

Services

  • Contient le Script PowerShell complémentaire pour créer des utilisateurs CRM et des exemples de fichier d'entrée CSV

Pour déployer la GPSC :

  1. Créez organisation (ORG).
    Couvert dans Microsoft Dynamics Administration et dans le cours de résolution des problèmes.
  2. Mettre à jour les paramètres dans les scripts PowerShell pour vous connectez à votre ORG.
  3. Créez un utilisateur dans ORG qui a un niveau d’accès égal à l'administrateur central.
  4. Configurez l'ORG par l’utilisation de l'interface administrative de Dynamics 365.
    1. Choisir la langue du système
      Cliquez sur Paramètres > Administration > « System Settings » (Paramètres système) et sous l'onglet Formats, sélectionnez Anglais (Canada) ou Français (Canada)
    2. Empêchez la solution d'envoyer des messages d'erreur à Microsoft.
      Cliquez sur Paramètres > Administration > « Privacy Preferences » (Préférences de confidentialité) et sélectionnez « Never send an error report to Microsoft » (Ne jamias envoyer de message d'erreur à Microsoft)
  5. Installez la partie 1 du système de GPSC en lançant le script Release1-SetOrg-Settings.ps1 dans le répertoire racine. Connectez-vous à l'ORG par l'utilisation du service de migration des données du SDK.
    1. Importez des données de Config_release_data.zip à partir du dossier DONNÉES.
    2. Importez LookupData.zip à partir du dossier DONNÉES
  6. Installez la partie 2 du système de la GPSC en lançant le script PowerShell Release1-Load_SH.ps1.
  7. Connectez vous à l'ORG par l'interface Web en tant qu'utilisateur avec un niveau d’accès égal à l’administrateur système.
    1. Publiez les règles de détection des doublons.
      Naviguez de Paramètres > Gestion des données > « Duplicate Detection Rules » (Règles de détection des doublons) et publiez les règles
    2. Activez les « business rules » (règles d'entreprise).
      Par l'utilisation de la Recherche avancée, cherchez Processus avec la catégorie Règle d'entreprise = « Draft » (Ébauche)

      Sélectionnez toutes les règles opérationnelles qui ressortent de cette recherche et activez-les.

Vérifiez le Microsoft Developer Network pour une aide générale sur lagestion du déploiement.*

Une fois que vous avez déployé la GPSC et mis à l'essai l'environnement, vous êtes prêt à procéder à Aperçu de la mise en œuvre de GPSC.
