À propos du plugin CFG Bilingue

Voir également : Configurer les enregistrements CFG Bilingue

Modification

Seuls plugins “PostMultiLingualRetrieveMultipleHelper” et “PostMultilingualRetrieveHelper” ayant été modifiés ou reconstruits de la source originale dans le fichier “MultiLingual.cs”.

Configuration

Séparateur

Le séparateur est le caractère qui sépare le texte français et le texte anglais.

Le symbole du double croisillon (##) est le séparateur utilisé dans le plugin. On y fait référence dans la méthode GetNameByLanguageId de MultiLingual.cs.

Donné

Utiliser un flux de production ou le plugin du champ calculé pour configurer un enregistrement en plusieurs langues.

Par exemple : TexteFrançais##TexteAnglais. Au moment de la création ou de la mise à jour de l'enregistrement, le flux de production ou le module d'extension affichera le texte de configuration.

Entité en plusieurs langues (fp_multilingualentity)

Les enregistrements créés dans l'entité CFG Bilingual (CFG Bilingue) comportent l'enregistrement parent et la recherche connexe qui doit faire l'objet d'un traitement.

Consulter : Configurer les enregistrements CFG Bilingue.

Module d'extension

Les messages “Retrieve” (récupérer) et “RetrieveMulitple” (récupérer de multiples entrées) devraient tous deux être enregistrés relativement à l'entité (enregistrement bilingue).

Remarque :

  • Si l'enregistrement bilingue est associé (recherche) à une autre entité, cette dernière devrait également être enregistrée relativement à “Retrieve” et “RetrieveMulitple”.

Les captures d'écran ci-dessous indique que "Salutation" est l'entité et que "Contact" est l'entité connexe (enregistrement parent de Salutation).

Action et fonctionnalité

Pour ce qui est de la récupération ou de la récupération multiple d'un enregistrement, dans le cas présent "Salutation", seul le texte français ou anglais sera affiché en fonction des préférences linguistiques de l'utilisateur.

Limites du plugin à plusieurs langues

Dans certains emplacements (tableaux, tableaux de bord, grilles) le champ bilingue ne figure pas dans la langue de préférence de l'utilisateur, mais affiche plutôt le nom concaténé (anglais | français). Cela entraîne la formation d'un long nom qui est souvent abrégé à l'écran.
