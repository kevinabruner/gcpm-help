À propos du module d'extension de calculs CFG

La fonction du champ calculé prêt à l'emploi de Dynamics 365 est si limitée qu'il est nécessaire d'utiliser un plugin permettant de reproduire ses fonctions afin de contourner ses limites. Ces limites incluent :

  • Nombre maximal de champs calculés connexes établi à 5.
  • Impossibilité de déclencher des flux de production ou des plugin dans les champs calculés.
  • Impossibilité d'utiliser relativement aux règles de détection dédoublées
  • Impossibilité de classer les champs calculés qui comportent un champ ou un enregistrement parent, un autre champ calculé ou un champ logique.
  • Impossibilité de couvrir plus de deux entités

Mise à jour Pré-opération Image pré-traitement
Mise à jour Asynchrone post-opération Image pré-traitement et post-traitement

La mise à jour supplémentaire du mode de l'asynchrone post-opération vise à supporter les champs calculés qui font référence aux champs tirés de leurs enregistrements parents. Lorsqu'un enregistrement parent est mis à jour, il recalculera de manière asynchrone les champs enfants.

Le plugin créera également les étapes du plugin ci-dessus pour l'entité elle-même et pour chacun des entités auxquelles on fait référence si ces dernières n'ont pas déjà été crées dans le cadre d'une autre configuration du champ de calcul. Les étapes à jour ne se déclencheront que pour les attributs auxquels on fait référence dans la configuration du champ de calculé

Définir un champ calculé

Définir un champ calculé

  1. Dans la recherche avancée, ouvrir l'entité du CFG calculated field (champ calculé du CFG).
  2. Ajouter un enregistrement en utilisant le service d'aide à l'écran pour vous orienter.
    Remarque : Les paramètres de la formule sont énumérés ci-dessous

Paramètres de la formule : Établir une liste d'attributs délimités par des virgules dont la valeur sera utilisée pour le formatage ou la concaténation des calculs ou du texte. La syntaxe est la suivante : { .∶ ; }

{ lookup.attribute : format ; default value }
  • Le nom de recherche n'est nécessaire que lorsque l'on fait référence aux attributs parents
  • Les accolades ne sont nécessaires que lorsque l'on précise le format ou la valeur par défaut.
  • Formater l'attribut dans le champ de la formule et non dans ce champ, sauf si la valeur doit être formatée deux fois (beta).
  • Utiliser l'attribut modifiedon pour la date d'aujourd'hui.

  • Si un champ n'est pas associé à une valeur, on utilisera la valeur par défaut. Si aucune valeur par défaut n'est précisée ou si le calcul est invalide, le champ ciblé sera effacé.

Étant donné qu'il n'est possible de préciser une valeur temps (supposons que vous vouliez enlever ou ajouter des jours, des mois ou des années à une date), une syntaxe spéciale est fournie afin de saisir les valeurs temps. Les valeurs temps

Syntaxe Description Exemple Résultat
ff … fffffff (2 à 7 chiffres) Millisecondes 10fff 00.00:00:00.010

mm minutes 10mm 00.00:10:00.000

Formule

Voici la formule de calculs ou la formule de formatage ou de concaténation de texte.

Ce champ met à profit les deux bibliothèques du cadre. NET afin d'analyser la formule et d'effectuer le formatage ou la concaténation des données ou des calculs.

  1. La syntaxe en vue du formatage et de la concaténation des champs est identique à la méthode du String Format(String format, params object[] args) du cadre .NET, dans le cadre de laquelle le champ constitue le paramètre régissant le format de la chaîne et les paramètres de formule constituent le paramètre d'args des objets[].

    Consultez les documents de Microsoft sur la méthode de format pour connaître la syntaxe de format de données valide. Un aide-mémoire est fourni à l'Annexe A – Syntaxe de formattage. La formule générale est :

    { valeur ,alignement ∶ format }

  2. La syntaxe en vue des calculs utilise la classe DataTable du cadre de .NET, et plus particulièrement la propriété DataColumn.Expression. Les valeurs de calcul sont rassemblées, tout comme les champs du formatage ou de la concaténation, de façon à ce que toutes les valeurs puissent être formatées avant les calculés réels.

    Consulter les documents de Microsoft sur la propriété DataColumn.Expression pour connaître la syntaxe de calculs valide. On utilise la même syntaxe que SQL. Un aide-mémoire est fourni.

  3. Pour faire référence aux attributs des paramètres de la formule, respecter la convention d'espace réservé String.Format dans le cadre de laquelle {0} fait référence au premier attribut, {1} au deuxième attribut et ainsi de suite. Le nombre maximal doit être inférieur ou égal au nombre d'attributs énumérés dans les paramètres de la formule. Les espaces réservés pour les nombres peut être utilisé plus d'une fois dans la formule.

Condition

Il s'agit de la condition en vertu de laquelle le formatage et la concaténation des calculs et des textes doivent être effectués. La valeur de ce champ est une chaîne de requête FetchXML. Il est possible d'utiliser les recherches avancées de CRM pour établir la chaîne de requête FetchXML. Dans le cas des requêtes plus complexes, il est possible d'utiliser un outil tiers appelé FetchXML Builder.

Avant de coller la chaîne de requête dans ce champ, il convient de s'assurer que toutes les accolades ont été supprimées, exception faite des espaces réservés.
