À propos du plugin du cumul CFG

Le plugin de cumul GPSC (egcs_rollup) surmonte les restrictions associées aux capacités de cumul de Dynamics 365, à savoir : 

  • Un maximum de 10 champs de cumul par enregistrement
  • Un maximum de 100 champs de cumul par organisation.

Le plugin du cumul de GPSC n'a pas ces limites.

La création, la mise à jour ou la suppression d'une entité enfant peut avoir une incidence sur les cumuls, il est donc nécessaire d'enregistrer toute étape de création, de mise à jour ou de suppression pour chaque champ cumul ayant été défini. Il ne faut enregistrer ces étapes qu'une fois pour chacune des entités et non pas pour chacun des champ cumul. Le plugin effectuera tous les cumuls d'une seule entité en réponse à une demande de créer, de mettre à jour ou de supprimer. Il sera toutefois nécessaire de définir de façon individuelle chaque champ cumul de l'entité de configuration.

Configurer le plugin de cumul CFG :

  1. Créer une enregistrement cumul CFG au moyen de l'entité egcs_rollup.
    Consulter Configurer un enregistrement de CFG Cumul  :
  2. Valider ces paramètres : 

    Établissement

    de la valeur

    Nom d'affichage

    Cumul / Cumul

    Nom du schéma

    cfg_rollup

    Propriété

    Organisation (supprime l'exigence du propriétaire)

    Options

    Tout décocher sauf la détection des doubles

    Note : Le code du plugin utilise FetchXML pour effectuer les calculs de cumul et le type de calculs est établi dans FetchXML au-moyen des étiquettes ci-dessus. Les ensembles d'option n'affichent que la valeur et non pas l'étiquette, c'est pourquoi le plugin définit les valeurs ci-dessus en fonction d'un nom de fonction d'agrégation, p. ex. la somme, le décompte, la moyenne, etc.

    Voilà pourquoi il est essentiel que les étiquettes et les valeurs soient identiques aux valeurs ci-dessus. Il est toujours possible de vérifier le code source des valeurs réelles au cas où l'ensemble d'options engendrerait des erreurs. La valeur entière réelle s'affichera dans le protocole d'erreurs si l'erreur est attribuable à l'ensemble d'options.

    Pour de plus amples renseignements sur FetchXML, veuillez consulter les articles suivants de la base de connaissances Microsoft : 

    • Utiliser FetchXML pour établir la requête*
    • Utiliser l'agrégation FetchXML*
